# 案例教程 10：类别不平衡问题与样本重采样

## 一、前序知识要求

### 1.1 核心概念

| 概念 | 说明 | 与不平衡数据的关系 |
|------|------|-----------------|
| **混淆矩阵** | TP / TN / FP / FN | 分清这四个值是理解不平衡评价的基础 |
| **Accuracy** | (TP + TN) / (TP + TN + FP + FN) | 在不平衡数据中会严重误导 |
| **Recall / Sensitivity** | TP / (TP + FN) | 医学场景的核心指标 |
| **Precision / PPV** | TP / (TP + FP) | 阳性预测值，衡量"猜对的概率" |
| **PR Curve** | Precision vs Recall | 不平衡数据下比 ROC 更敏感 |
| **过拟合** | 模型记住了训练噪声 | 过采样（尤其是 Random OverSampling）会加剧过拟合 |

### 1.2 本数据集的不平衡概况

| 类别 | 样本数 | 比例 | 含义 |
|------|--------|------|------|
| MORTO (0, 死亡) | 11,770 | 90.91% | 多数类 |
| VIVO (1, 存活) | 1,177 | 9.09% | **少数类** |
| **Imbalance Ratio** | **10.0 : 1** | — | 每 1 个存活患者对应 10 个死亡患者 |

> **采样方法说明**：原始数据 IR≈1.4:1（轻度不平衡），重采样效果不明显。为充分展示重采样策略的差异，本教程通过**欠采样少数类 VIVO** 构造 IR=10:1（严重不平衡）的教学数据集。总样本数 12,947。

### 1.3 前序教程回顾

- **案例 3（缺失值插补）**：介绍了数据质量对模型的影响，缺失+不平衡是医学数据的"双重挑战"
- **案例 7（数据泄漏）**：任何使用测试集信息的操作都构成泄漏。本教程中 SMOTE 泄漏是最容易忽视的泄漏类型之一
- **案例 9（建模对比）**：7 个模型中 Recall 最高的是 LR（class_weight='balanced'），这正是"处理不平衡"的结果

## 三、各模块详解

### 模块 1：医学代价分析

#### 临床场景：癌症筛查

> 一个 50 岁患者接受癌症筛查，模型给出两种可能：

```
真实情况: 患者有癌症 (Positive)      真实情况: 患者健康 (Negative)
模型判定: 健康 (Negative)            模型判定: 癌症 (Positive)
           ↓                                   ↓
       False Negative (FN)                False Positive (FP)
           ↓                                   ↓
    错过最佳治疗窗口                         追加活检/CT
    患者几个月后确诊晚期                      增加焦虑和费用
           ↓                                   ↓
    代价: 可能致命                        代价: 可控的医疗支出
```

**结论：在癌症筛查中，FN 的代价远高于 FP。**

#### 对模型评价的影响

| 场景 | 优先指标 | 理由 |
|------|---------|------|
| 癌症筛查 | **Recall (Sensitivity)** | "宁可错判一千，不可漏过一个" |
| 确诊治疗 | **Precision (PPV)** | "诊断了就要化疗，不能乱诊断" |
| 术后随访 | **F1 (Recall + Precision 的平衡)** | 两种错误都要控制 |

### 模块 2：不平衡数据 EDA

#### Imbalance Ratio (IR)

$$
\text{IR} = \frac{N_{\text{多数类}}}{N_{\text{少数类}}} = \frac{11,770}{1,177} = 10.0
$$

| IR 大小 | 不平衡程度 | 处理建议 |
|---------|-----------|---------|
| **IR < 2** | 轻微 | 可不处理 |
| **2 ≤ IR < 10** | 中等 | 建议重采样 |
| **10 ≤ IR < 50** | 严重 | 必须重采样 |
| **IR ≥ 50** | 极严重 | 需考虑异常检测思路 |

**本数据集 IR=10.0，属于"严重不平衡"，必须重采样。**

### 模块 3：Accuracy Paradox（准确率悖论）

#### 实验：两个模型的极端对比

| 指标 | "全预测死亡"模型 | 加权 Logistic Regression |
|------|----------------|------------------------|
| **Accuracy** | 0.9091 | **0.7614** ↑ |
| **Recall** | **0.0000** | **0.8782** ↑ |
| Precision | 0.0000 | 0.2596 |
| F1 | 0.0000 | 0.4008 |
| **ROC-AUC** | **0.5000** | **0.8974** ↑ |
| PR-AUC | — | 0.4177 |

#### 核心教学点

```
"总是预测死亡" 这个模型的本质:

        预测: 死亡 (0)        预测: 存活 (1)
实际: 死亡  TN = 3,531         FP = 0     → 多数类准确率 100%
实际: 存活  FN = 353           TP = 0     → 少数类准确率 0%

Accuracy = (3531 + 0) / (3531 + 0 + 0 + 353) = 3531 / 3885 = 90.91%

但 Recall = 0 / (0 + 353) = 0%
```

**结论：Accuracy 在不平衡数据中是一个危险的指标。不到 30 秒就能构建出一个 Accuracy=90.91% 但完全没有临床价值（Recall=0）的模型。**

### 模块 4：重采样策略比较

#### 四种方法对比

| 方法 | AUC | Recall | Precision | F1 | 训练样本数 | 特点 |
|------|-----|--------|-----------|-----|-----------|------|
| **No Resampling** | **0.9007** | 0.1813 | **0.6214** | 0.2807 | 9,062 | 基线 Recall 极低 (0.18)，正类仅 9.1% (824) |
| Random UnderSampling | 0.8971 | 0.8810 | 0.2620 | 0.4039 | **1,648** | 丢弃多数类，正类比例升至 50% (824) |
| Random OverSampling | 0.8976 | 0.8754 | 0.2619 | 0.4031 | 16,476 | 复制少数类，正类比例 50% (8,238) |
| **SMOTE** | 0.8976 | 0.8640 | 0.2694 | **0.4108** | 16,476 | 全面均衡的表现，正类比例 50% (8,238) |
| ADASYN | 0.8947 | **0.8952** | 0.2555 | 0.3975 | 16,494 | Recall 最高，正类比例 50.1% (8,256) |

#### 关键发现

| 发现 | 数据 | 解读 |
|------|------|------|
| 不重采样：Recall=0.18（极低！） | 9,062 训练样本中仅 824 个 VIVO (9.1%) | 模型几乎无法检出少数类 |
| Under 只用 1,648 样本 | 丢弃了大量多数类数据 | 信息损失但 AUC 尚可 (0.8971) |
| SMOTE vs ADASYN: 接近 | AUC 0.8976 vs 0.8947 | 本数据集上差异可忽略 |
| 所有重采样方法 AUC 接近 | 0.8947-0.9007 | 重采样主要影响 Recall (0.18→0.90), 对排序能力 (AUC) 影响小 |

```
核心观点:
  重采样不提升模型的"排序能力"(AUC), 而是改变模型的"决策阈值"(Recall)。
  如果你需要一个能找到少数类的模型, 重采样是必须的。
  如果你只需要一个排序准确的概率估计, 可能不需要重采样。
```

### 模块 5：数据泄漏案例（医学论文最常见错误之一）

#### 错误做法 vs 正确做法

```
❌ 错误做法:                             ✅ 正确做法:

全数据                                   训练集     测试集
  │                                       │
  ├── SMOTE (生成合成样本)                 ├── SMOTE (仅在训练集上)
  │                                       │
  └── Train/Test Split                    └── 训练模型

  测试集包含了 SMOTE 生成的样本             测试集完全来自真实数据
  → AUC 被虚高                              → 真实的泛化能力
```

#### 实验证明

| 做法 | AUC | Recall | 说明 |
|------|-----|--------|------|
| **泄漏版 (SMOTE→Split)** | **0.8990** | 0.8779 | 在泄漏的测试集上评估 → 虚高 (Acc=0.8207) |
| 泄漏版 (在原始测试集) | 0.8981 | 0.8640 | 在原始测试集上揭穿真相 |
| **正确版 (Split→SMOTE)** | **0.8976** | 0.8640 | 这才是真实的泛化能力 (Acc=0.7748) |
| Δ (虚高量) | +0.0014 | +0.0139 | 看似小，但在大样本和高维场景下会放大 |

#### 为什么测试集上的 SMOTE 样本会导致泄漏？

```python
# SMOTE 的原理: 在少数类样本之间插值
# 如果测试集中也有"被插值"的样本:
#   测试集中的样本 B = α × 训练集中的样本 A + (1-α) × 训练集中的样本 C
#   模型已经"见过"样本 A 和 C → 对 B 的预测自然更准确
#   → AUC 虚高

# 等价于: 测试集的一部分是训练集的"线性组合"
#         → 这已经不是"未见过的数据"了
```

#### 教学格言

> **在数据划分之前做的任何事情，如果使用了整个数据集的信息，都是泄漏。**

### 模块 6：交叉验证中的重采样

#### 正确 CV 流程

```
Stratified 5-Fold CV:
           ┌─────────────────────────────────────┐
Fold 1:    │  [训练折 + SMOTE]  →  验证折         │  AUC₁
           └─────────────────────────────────────┘
           ┌─────────────────────────────────────┐
Fold 2:    │  [训练折 + SMOTE]  →  验证折         │  AUC₂
           └─────────────────────────────────────┘
                ... (共 5 轮)

每次循环:
  1. 从原始数据中拆分出训练折和验证折
  2. 仅在训练折上做 SMOTE
  3. 用 SMOTE 后的训练折训练模型
  4. 在原始验证折上评估
```

#### 实验对比

| 方法 | Mean AUC | σ | 说明 |
|------|---------|------|------|
| **正确 CV (SMOTE per fold)** | **0.8902** | **0.0090** | 真实泛化能力 |
| 泄漏 CV (全数据 SMOTE → CV) | 0.8942 | 0.0022 | AUC 虚高 +0.0040 |
| Δ | **0.0040** | — | AUC 虚高且方差被人为减小 |

> **泄漏 CV 的 AUC 虚高（Δ=0.0040），且 σ 更小（0.0022 vs 0.0090）。** 本数据集为严重不平衡（IR=10:1），泄漏 CV 的 σ=0.0022 比正确 CV 的 σ=0.0090 更小 → 泄漏让结果"看起来更稳定"但这是危险的假象，会让审稿人误以为模型"非常稳定"。在严重不平衡场景下，泄漏效应比轻度不平衡更明显，必须警惕。

### 4.1 重采样方法选择指南

```
┌──────────────────────────────────────────────────────────────┐
│                     数据不平衡 → 怎么办?                       │

     class_weight            Random UnderSampling     异常检测思路
     'balanced'              (计算受限时)              (Isolation Forest)
```

### 4.2 SMOTE vs ADASYN

| 对比 | SMOTE | ADASYN |
|------|-------|--------|
| 年份 | 2002 | 2008 |
| 核心思想 | 随机选择 k 近邻插值 | 在**难分类**样本周围生成更多合成样本 |
| 生成策略 | 所有少数类样本一视同仁 | 密度越低、"越难"的区域生成越多 |
| 本实验 AUC | 0.8976 | 0.8947 |
| 何时 ADASYN 更好 | — | 少数类分布不均匀时 (簇状分布) |
| 风险 | 可能生成边界样本 | 更可能生成噪声 (因为专注于难分类区域) |

### 4.3 不平衡数据的 7 种评价指标

| 指标 | 公式 | 关注点 | 不权衡衡下的行为 |
|------|------|--------|-----------------|
| **Accuracy** | (TP+TN)/N | 全局正确率 | 高 (被多数类主导) |
| **Recall** | TP/(TP+FN) | 少数类检出率 | 低 (未加权的模型) |
| **Precision** | TP/(TP+FP) | 预测少数类的精度 | 高 (只预测最确定的) |
| **F1-score** | 2×P×R/(P+R) | Recall + Precision 的调和 | 低 (两者都不高) |
| **ROC-AUC** | — | 排序能力 | 偏高 (多数类 AUC 贡献大) |
| **PR-AUC** | — | 少数类排序能力 | 低 (对少数类数量敏感) |
| **Brier Score** | Σ(y-p)²/N | 概率校准 | 偏低 (预测死亡有高置信度) |

> **教学要点**：在不平衡数据中，**PR-AUC 比 ROC-AUC 更诚实**。PR-AUC 只关注正类的 Precision 和 Recall，不会被多数类 (占 91%) 的 TN 数量所稀释。

## 五、扩展内容

### 5.1 重采样为什么几乎不改变 AUC？

本实验显示：所有重采样方法的 AUC 都在 0.8947-0.9007 之间，差异 <= 0.006。但 Recall 从 0.18 提升到 0.90。

**原因**：AUC 衡量的是排序能力——正类的预测概率是否普遍高于负类。重采样**不改变**样本在特征空间中的分布 (只是增加了少数类的频次)，所以排序能力几乎不变。

**类比**：重采样不改变"谁是老师眼中的好学生"，它改变的是"老师给多少学生发奖学金"。

### 5.2 为什么不直接用 class_weight='balanced'？

本教程中使用的是 `class_weight=None`，就是故意展示不加权时的基线表现。

添加 `class_weight='balanced'` 等价于在损失函数中给少数类错误加权：

```python
# balanced 权重的计算方式
weight_for_VIVO = n_samples / (2 * n_VIVO)   = 12947 / (2 * 1177) ≈ 5.50
weight_for_MORTO = n_samples / (2 * n_MORTO) = 12947 / (2 * 11770) ≈ 0.55
```

这个权重在优化过程中告诉模型："一个 VIVO 的误判 = 5.50/0.55 ≈ 10.0 个 MORTO 的误判"（等于 IR）。这正是"医学代价分析"的数学实现。

### 5.3 如果 IR 更高，什么方法最有效？

| IR 范围 | 本实验 | 推荐方法 | 原因 |
|---------|--------|---------|------|
| 10:1 | ✅ 当前值 | 必须重采样/SMOTE | 严重不平衡 |
| 50:1 | 更极端 | SMOTE + Repeated CV | 合成数据 + 稳定评估 |
| 100:1 | 非常极端 | **SMOTEENN** (过采样 + 清洗) | 防止 SMOTE 生成边界噪声 |
| 500:1 | 天文数字 | **异常检测** (Isolation Forest) | 分类器已完全不适用 |

**SMOTEENN**：先用 SMOTE 生成少数类，再用 Edited Nearest Neighbors 清洗边界区域中的噪声样本。是 `imbalanced-learn` 中 `SMOTEENN` 类。

### 5.4 阈值移动 (Threshold Moving)：不重采样的另一种方案

除了重采样，还可以**不改变数据分布，但调整决策阈值**：

```python
# 默认阈值: 0.5 (认为概率 > 0.5 即为正类)
# 在不平衡数据中，正类的概率通常远低于 0.5

# 调整阈值 = 多数类比例
threshold = n_neg / (n_neg + n_pos)  # ≈ 0.9091

# 但更科学的方法是: 在验证集上搜索最佳阈值
from sklearn.metrics import precision_recall_curve
precisions, recalls, thresholds = precision_recall_curve(y_val, y_prob)
# 选择 Recall 最高的 threshold
```

| 方法 | 数据 | 改变什么 | 优点 | 缺点 |
|------|------|---------|------|------|
| **重采样** | 训练集 | 数据分布 | 直观、通用 | 可能引入噪声 |
| **阈值移动** | 测试集 | 决策边界 | 不改变数据 | 需要另外确定阈值 |

### 5.5 合成数据 (DeepSMOTE / GAN-based)

对于高级课程，可以介绍基于深度学习的合成数据生成：

| 方法 | 原理 | 使用场景 |
|------|------|---------|
| **DeepSMOTE** | 用自编码器提取特征，在隐空间做 SMOTE | 高维表格数据 |
| **CTGAN** | 条件 GAN 生成表格数据 | 隐私保护 + 数据增强 |
| **VAE** | 变分自编码器生成新样本 | 连续性特征为主的数据 |

> **教学定位**：这些方法在医学数据中正逐渐增加，但 SMOTE 仍然是最广泛使用的"黄金标准"。

### 5.6 重采样 + 树模型

```python
# 树模型对不平衡的鲁棒性比线性模型强
# 但重采样仍然有帮助

# XGBoost: 用 scale_pos_weight 而不是重采样
xgb = XGBClassifier(scale_pos_weight=10.0, ...)

# Random Forest: 用 class_weight + 重采样
rf = RandomForestClassifier(class_weight='balanced_subsample', ...)
# 'balanced_subsample' 在每棵树的 Bootstrap 采样中自动加权
```

## 六、总结与思考题

### 6.1 核心收获

1. **高准确率 ≠ 好模型**："全预测死亡"模型 Accuracy=90.91% 但 Recall=0，在临床上毫无价值
2. **Imbalance Ratio 是重要的数据诊断指标**：IR=10.0 → 严重不平衡，必须重采样
3. **SMOTE 是最推荐的基线方法**：保持 AUC 的同时将 Recall 从 0.18 提升到 0.86（4.8 倍提升）
4. **SMOTE 泄漏是医学论文中最危险且最易犯的错误之一**：AUC 虚高 +0.0014，Recall 虚高 +0.0139
5. **CV 中的重采样必须在每折训练集上独立做**：泄漏 CV 的 σ 更小（0.0022 vs 0.0090），这是危险的假象；Pipeline + SMOTE 是最安全的方案
6. **评价指标的选择决定结论**：不平衡数据中 PR-AUC 比 ROC-AUC 更诚实

### 6.2 思考题

1. **（基础）** 本实验中，"全预测死亡"模型的 Accuracy 是 90.91%，但 Recall=0。如果一个学生拿到了这个模型，他报告说"我的模型有 90.91% 准确率"。你会如何向他解释这个指标为什么是误导的？

2. **（进阶）** No Resampling 的 Precision=0.7507，而 SMOTE 的 Precision=0.7093。为什么不做重采样时 Precision 反而更高？（提示：模型只预测最有把握的样本时会怎样？）

3. **（进阶）** 泄漏版 SMOTE 在泄漏的测试集上 AUC=0.8979，但在原始测试集上 AUC=0.8948（接近正确版 0.8946）。为什么差异很小？在什么情况下这个差异会大很多？（提示：考虑本数据集的特征数只有 6 个，而基因组数据有 10,000+ 个特征）

4. **（拓展）** 泄漏版 CV 的 σ=0.0053，正确版 σ=0.0029。本数据集为轻度不平衡，泄漏效应不明显。在严重不平衡场景下，泄漏版 CV 通常方差更小且 AUC 虚高，这是否意味着"泄漏版看起来更稳定"？审稿人可能因此被误导吗？
