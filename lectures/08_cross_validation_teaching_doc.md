# 案例教程 8：数据划分与交叉验证

## 一、前序知识要求

### 1.1 机器学习基础

| 概念                  | 说明                    | 与交叉验证的关系                |
| ------------------- | --------------------- | ----------------------- |
| **训练集 / 测试集 / 验证集** | 训练集拟合模型，验证集调参，测试集最终评估 | 交叉验证将同一份数据轮流用作"训练"和"验证" |
| **过拟合**             | 模型记住训练集噪声，在测试集表现差     | CV 通过多次验证检视过拟合          |
| **泛化能力**            | 模型在未见数据上的表现           | CV 的目标就是更准确地估计泛化能力      |
| **偏差-方差权衡**         | K 越大偏差越小方差越大（反之亦然）    | 选择 K 值就是在平衡偏差和方差        |
| **超参数**             | 需要人为设定的参数（如 C、K 值）    | Nested CV 在选参的同时无偏评估    |
| **Pipeline**        | 将预处理 + 建模串联           | 确保每折独立做预处理，防止泄漏         |

### 1.2 统计学基础

| 概念       | 说明                    |
| -------- | --------------------- |
| **均值**   | AUC 的集中趋势，越接近真实泛化能力越好 |
| **标准差**  | AUC 的波动范围，越小说明评估越稳定   |
| **分层抽样** | 按类别比例抽样，保持样本分布一致      |

### 1.3 前序教程回顾

- **案例 7（数据泄漏）**：任何使用测试集信息的操作都构成泄漏
- **交叉验证与泄漏的关系**：在 CV 的每折中，当前验证集就是"测试集"，预处理（标准化、插补）必须在每折的训练折上独立完成

## 二、本次实践内容与目的

### 2.1 核心目的

> **理解模型泛化能力评估——为什么单次划分不靠谱？K 折交叉验证如何给出更可信的评估？**

### 2.2 七个部分一览

| 部分      | 方法                | 核心特征       | 教学要点              |
| ------- | ----------------- | ---------- | ----------------- |
| **1**   | Train-Test Split  | 单次划分       | 结果不稳定性（极差 0.0151） |
| **2**   | K-Fold CV         | 5/10 折轮流验证 | 为什么比单次划分可靠        |
| **3**   | Stratified K-Fold | 保持每折类别比例   | 医学数据的标准做法         |
| **4-5** | Repeated K-Fold   | 多次重复取平均    | 进一步降低方差           |
| **6**   | LOOCV             | 每次留一个样本    | 极端情况下的优缺点         |
| **7**   | Nested CV         | 内外双层 CV    | 避免调参偏倚的"金标准"      |

### 2.3 核心框架

```
评估方法的选择是在三个约束之间做权衡:

        ┌───────────────────┐
        │    偏差 (Bias)    │ ← 评估值与真实值的差距
        └───────────────────┘
                    │
                    ▼
    ┌───────────────────────────────┐
    │        评估方法选择            │
    └───────────────────────────────┘
                    ▲
        ┌───────────────────┐
        │    方差 (Variance) │ ← 评估值自身的稳定性
        └───────────────────┘
                    ▲
        ┌───────────────────┐
        │  计算成本 (Cost)   │ ← 训练模型的次数
        └───────────────────┘
```

## 三、各模块详解

### 第一部分：Train-Test Split 为什么不可靠？

#### 实验数据

20 次不同随机种子的 80/20 划分，每次用逻辑回归训练评估：

| 统计量         | 值          | 解读            |
| ----------- | ---------- | ------------- |
| Mean AUC    | 0.8920     | 中心趋势          |
| **Std (σ)** | **0.0040** | **单次划分的标准误**  |
| Min AUC     | 0.8839     | "最差"的一次划分     |
| **Max AUC** | **0.8990** | **"最好"的一次划分** |
| **极差**      | **0.0151** | **你运气好坏的差距**  |

> **核心教学点**：如果你的论文只看一次划分的结果，你可能是"运气好"（AUC=0.8990）也可能是"运气差"（AUC=0.8839）。**0.0151 的极差意味着：同样的模型、同样的数据，仅仅因为随机种子不同就可以改变"是否达到审稿人要求"的结论。**

#### 不稳定的来源

```
一次 80/20 划分 = 单个训练集 + 单个测试集

问题 1: 测试集大小                     问题 2: 测试集的组成
20% 的 20,000 样本 = 4,000 个         不同的随机种子 = 不同的测试集
其中 VIVO (41.15%) ≈ 1,646 个         测试集中 VIVO 的比例 ~ 38-44%
少数类的 AUC 对 1,646 个样本高度敏感   "好"测试集 ≈ 更容易得到好 AUC
```

### 第二部分：K-Fold Cross Validation

#### 流程

```
K-Fold CV 的完整流程 (以 5-Fold 为例):

第 1 轮:  [Fold 1][Fold 2][Fold 3][Fold 4]Fold 5 → AUC₁
第 2 轮:  [Fold 1][Fold 2][Fold 3]Fold 4[Fold 5] → AUC₂
第 3 轮:  [Fold 1][Fold 2]Fold 3[Fold 4][Fold 5] → AUC₃
第 4 轮:  [Fold 1]Fold 2[Fold 3][Fold 4][Fold 5] → AUC₄
第 5 轮:  Fold 1[Fold 2][Fold 3][Fold 4][Fold 5] → AUC₅

        训练集                   验证集
最终: AUC = (AUC₁ + AUC₂ + AUC₃ + AUC₄ + AUC₅) / 5
```

#### 为什么 K-Fold 比单次划分可靠？

| 对比维度          | Train-Test Split | K-Fold CV            |
| ------------- | ---------------- | -------------------- |
| **验证次数**      | 1 次              | K 次                  |
| **数据利用率**     | 80% 训练，20% 验证    | 每折 (K-1)/K 训练，1/K 验证 |
| **每样本参与验证次数** | 0 或 1 次          | 恰好 1 次               |
| **AUC 方差来源**  | 单次划分的偶然性         | K 个不同划分的平均           |

**关键洞察**：K-Fold 不依赖于任何单次划分的好坏。每个样本恰好被验证一次，平均了 K 种不同训练集大小下的表现。

#### 5-Fold vs 10-Fold

| 对比                | 5-Fold            | 10-Fold           |
| ----------------- | ----------------- | ----------------- |
| 每折训练集占比           | 80%               | 90%               |
| 偏差 (Bias)         | 略高（训练集更小）         | 略低                |
| **方差 (Variance)** | **更低 (σ=0.0042)** | **更高 (σ=0.0048)** |
| 计算成本              | 5 个模型             | 10 个模型            |
| 经验推荐              | 大多数场景             | 数据量小时             |

> **反直觉发现**：本实验中 5-Fold 比 10-Fold 的方差更低！这是因为每折的测试集更大（4,000 vs 2,000 样本），AUC 估计更稳定。**不是折数越多越好。**

### 第三部分：Stratified K-Fold

#### 为什么医学数据必须分层抽样？

医学数据集的特点决定了 Stratified K-Fold 的必要性：

| 场景               | 非分层                 | 分层              |
| ---------------- | ------------------- | --------------- |
| 总体 VIVO 比例       | 41.15%              | 41.15%          |
| Fold 1 的 VIVO 比例 | ≈ 38% \~ 44% (随机波动) | ≈ 41.15% (精确保持) |
| Fold 2           | 完全不同的分布             | 相同的分布           |
| 后果               | **AUC 在不同折之间剧烈波动**  | **每折 AUC 更一致**  |

**本数据集**：VIVO 占 41.15%（轻度不平衡），Stratified 仍能保持每折比例一致。在严重不平衡的数据中（如 VIVO < 10%），Stratified 是必须的，否则某折可能只包含几个 VIVO 样本，该折的 AUC 几乎没有意义。

#### 实验对比

| 方法                | σ      | 极端折差异              |
| ----------------- | ------ | ------------------ |
| 5-Fold            | 0.0042 | 折 1 vs 折 3: 0.0052 |
| Stratified 5-Fold | 0.0027 | 折 1 vs 折 5: 0.0082 |

**实验发现**：本实验中 Stratified 5-Fold 的方差更低（σ=0.0027 vs 0.0042），符合分层抽样的预期。在 20,000 样本下分层抽样保持了类别比例一致，使评估更稳定。在小样本量下（如 1,000 样本），Stratified 的优势会更加明显。

**教学结论**：**Stratified 是医学数据的安全做法。它不会伤害结果（和普通 K-Fold 一样好），但在小样本或不平衡场景下是唯一可靠的选择。**

### 第四部分 & 第五部分：Repeated K-Fold

#### 原理

```
Repeated 5-Fold CV (10 次重复):

                  ┌─ 第 1 次 5-Fold: [AUC₁, AUC₂, AUC₃, AUC₄, AUC₅]  ← 不同随机洗牌
                  ├─ 第 2 次 5-Fold: [AUC₆, AUC₇, AUC₈, AUC₉, AUC₁₀] ← 不同划分
                  ├─ ...
                  └─ 第 10 次 5-Fold: [AUC₄₆, ..., AUC₅₀]

最终: AUC = 50 个值的平均
      标准差 = 50 个值的标准差
```

#### 与普通 K-Fold 对比

| 方法                         | Mean AUC | σ          | 模型数 |
| -------------------------- | -------- | ---------- | --- |
| 5-Fold                     | 0.8914   | **0.0042** | 5   |
| Repeated 5-Fold            | 0.8915   | **0.0046** | 50  |
| Repeated Stratified 5-Fold | 0.8914   | **0.0040** | 50  |

#### 教学结论

- **Repeated CV 的 Mean 更可靠**（基于更多数据点）
- **Repeated CV 的 Std 更大**（因为包含了不同随机洗牌的变异性，而单次 CV 只反映单个洗牌的内部变异性）
- **建议同时报告 Mean ± Std**，不要只看一个

### 第六部分：Leave-One-Out (LOOCV)

#### 优缺点对比

```
LOOCV:
  训练集: n-1 个样本    验证集: 1 个样本
  循环: n 次 (每个样本留一次)
  最终 AUC: n 个预测结果

优点:
  ✅ 几乎无偏 (训练集用了 n-1 个样本)
  ✅ 确定性结果 (没有随机划分)
  ✅ 每个样本都参与评估

缺点:
  ❌ 计算成本 O(n) — 大数据不可行
  ❌ 方差大 — 每次只验证 1 个样本
  ❌ 模型之间高度相似 — 评估不独立
```

#### 本实验的计算成本分析

| 样本量       | LOOCV 耗时       | 对应 K-Fold 耗时    |
| --------- | -------------- | --------------- |
| 500       | 0.6s           | < 0.1s          |
| 20,000    | **\~24s (估算)** | \~0.6s (5-fold) |
| 50,000    | \~60s          | \~1.5s          |
| 1,000,000 | \~20min        | \~30s           |

#### 分类讨论

> **你会在什么场景下用 LOOCV？**

| 场景            | 适合？ | 原因                     |
| ------------- | --- | ---------------------- |
| 样本量 < 100     | ✅   | 计算可接受，K-Fold 每折样本太少    |
| 样本量 100-1,000 | 可能  | 取决于计算资源                |
| 样本量 > 1,000   | ❌   | 5-Fold 或 10-Fold 足够好   |
| 需要确定性结果       | 可能  | 但 Repeated K-Fold 也可做到 |
| 深度神经网络        | ❌   | 训练一次几个小时，LOOCV 完全不现实   |

### 第七部分：Nested Cross Validation

#### 结构

```
Nested CV 的双层结构:

外层 CV (5-Fold) → 评估泛化能力
│
├── 外折 1:
│   ├── 训练集 (80%) ──→ 内层 CV (3-Fold) ──→ 选最佳参数 C=0.01
│   └── 测试集 (20%) ←── 用 C=0.01 训练模型 → AUC₁
│
├── 外折 2:
│   ├── 训练集 (80%) ──→ 内层 CV (3-Fold) ──→ 选最佳参数 C=0.01
│   └── 测试集 (20%) ←── 用 C=0.01 训练模型 → AUC₂
│
├── ...
│
最终: AUC = (AUC₁ + AUC₂ + ... + AUC₅) / 5
```

#### 为什么 Nested CV 是高水平论文标配？

```
不嵌套 CV:                         Nested CV:
                               
┌── 全数据 ──┐                   ┌── 外折 1 ──┐
│ 训练 80%   │ ← (用了测试集)    │  训练 64%  │
│ 测试 20%   │   信息来选参)     │  内部 16%  │ ← 仅在训练折内选参
└────────────┘                   │  测试 20%  │
      │                          └────────────┘
      ▼                                ▼
  AUC 被乐观偏倚                   AUC 更准确
  (optimism bias)              (无偏估计泛化能力)
```

#### 本实验的关键发现

| 方法        | AUC                             | 解读               |
| --------- | ------------------------------- | ---------------- |
| 非嵌套 CV    | 0.8952                          | 调参偏倚（但很小，因参数网格窄） |
| Nested CV | 0.8952                          | 真正无偏的泛化能力估计      |
| Fixed C=1 | 0.8941                          | 不调参的基线           |
| 外折选择的 C   | \[0.01, 0.01, 0.01, 0.01, 0.01] | 所有外折选相同参数        |

**教学要点**：本实验中 Nested 和 Non-Nested 的差异很小，因为参数网格只有 4 个值（C=0.01, 0.1, 1, 10）。在更复杂的调参中（如 XGBoost 的 100+ 参数组合），非嵌套 CV 的偏倚可能会显著增大。

## 五、扩展

### 5.1 K 值选择：5、10 还是 20？

```
K 越小 → 训练集越小 → 偏差越大 → 但方差越小 (训练集差异大)
K 越大 → 训练集越大 → 偏差越小 → 但方差越大 (训练集差异小)

经验法则:
  - K=5: 大多数场景的好默认值 (本实验验证)
  - K=10: 样本量小 (< 2,000) 时更好
  - K=∞ (LOOCV): 仅 n < 100 时合理
```

### 5.2 分层 vs 不分层

| 数据集特征            | 建议       | 原因           |
| ---------------- | -------- | ------------ |
| 目标分布均匀 (40-60%)  | 均可       | 分层影响小        |
| 目标分布不平衡 (≤10%)   | **必须分层** | 否则某折可能 0 个正类 |
| 小样本 (n < 500)    | **必须分层** | 随机波动可能导致类别缺失 |
| 大样本 (n > 10,000) | 建议分层     | 几乎没有成本，但更安全  |

### 5.3 Repeated CV：重复多少次？

```
重复次数越多 → 均值越精确 → 计算成本线性增加

经验规则:
  - 5 次重复 = 25 个模型 (5-Fold) → 够用
  - 10 次重复 = 50 个模型 → 推荐 (本实验用)
  - 20 次重复 = 100 个模型 → 追求高精度
  - 50 次重复 → 通常过度
```

### 5.4 CV 与数据泄漏

> 交叉验证本身**不防止**数据泄漏。如果你在全数据上做了预处理再传给 CV，泄漏已经发生了。

**正确做法**：将整个预处理 + 建模流程放在一个 Pipeline 中，让 CV 的每折独立运行。

```python
# ✅ 正确: Pipeline 在每折中独立 fit_transform
pipe = Pipeline([
    ('imputer', SimpleImputer()),
    ('scaler', StandardScaler()),
    ('select', SelectKBest(k=5)),
    ('lr', LogisticRegression())
])
scores = cross_val_score(pipe, X, y, cv=5)

# ❌ 错误: 预处理泄漏
X_processed = scaler.fit_transform(X)  # 用了全数据!
scores = cross_val_score(lr, X_processed, y, cv=5)
```

## 六、总结与思考题

### 6.1 核心收获

1. **单次 Train-Test Split 不可靠**：极差 0.0151，相当于模型好坏的声明取决于运气
2. **5-Fold CV 是好的默认选择**：本实验中 σ=0.0042（Stratified 5-Fold 的 σ=0.0027 更小），成本适中
3. **Stratified 对医学数据是必须的**：防止类别失衡导致某折评估失效
4. **Repeated CV 给出更精确的均值**：更多数据点，但要结合标准差一起看
5. **LOOCV 在计算成本方面的代价巨大**：500 样本 0.6 秒 → 20,000 样本 24 秒
6. **Nested CV 是论文级标准**：避免调参偏倚，更被审稿人信任
7. **核心公式**：评估的可信度 ≈ 平均 AUC ± 标准差 / √估计次数

### 6.2 思考题

> 以下问题覆盖从理解到批判性思维的全层次：

1. **（基础）** 20 次 Train-Test Split 的 AUC 极差为 0.0151（0.8839-0.8990）。如果一个审稿人要求你"只用一次划分"，你该如何回应？如果这种情况真的发生了，你会怎么做来让你的结果更可信？
2. **（进阶）** 在本实验中，5-Fold（σ=0.0042）比 10-Fold（σ=0.0048）更稳定。为什么 10 折的方差反而更大？这是否违背了"更多数据 → 更稳定"的直觉？（提示：考虑每折的测试集大小）
3. **（进阶）** Stratified 5-Fold 的标准差（σ=0.0027）低于普通 5-Fold（σ=0.0042）。为什么分层抽样能降低方差？这是否意味着在任何情况下 Stratified 都是最优选择？
4. **（拓展）** Nested CV 的结果（0.8952）与非嵌套 CV（0.8952）相同，差异为 0。但这是否说明 Nested CV 不重要？如果你在调 XGBoost 的 5 个超参数，每个有 5 种取值（总共 5⁵=3125 种组合），Nested 和非 Nested 的差异会有多大？
5. **（拓展）** Repeated Stratified 5-Fold（10 次重复）训练了 50 个模型，耗时 0.7s。如果在深度神经网络上做同样的事，每个模型训练需要 1 小时。在深度学习场景下，你会如何平衡评估的可靠性和计算成本？
6. **（开放）** 有人说"交叉验证只是一种评估方法，不是模型的一部分"。你同意吗？如果 CV 的结果告诉你某个特征对模型很重要，这个信息本身是否也会影响你的建模决策？
7. **（实践）** 修改本实验代码，将 n\_repeats 从 10 改为 3 和 30，观察 Repeated CV 的均值和标准差如何变化。多少次的重复开始"收益递减"？
