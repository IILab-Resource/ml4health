# 案例教程 15：SHAP 聚类分析 — 基于 SHAP 值的样本子群发现

## 一、前序知识要求

### 1.1 核心问题

> **数据集中是否存在具有不同"解释模式"的样本群组？**

传统分析关注的是"哪些特征总体重要"，但忽略了不同患者亚群的特性：

| 视角 | 问题 | 方法 | 局限 |
|------|------|------|------|
| 全局 | 所有患者中哪些特征最重要？ | SHAP Summary Plot | 掩盖了亚群差异 |
| 局部 | 这个患者为什么这样预测？ | SHAP Waterfall | 只看了一个患者 |
| **子群** | **哪类患者的决策模式相似？** | **SHAP 聚类** | **连接全局与局部** |

### 1.2 对 SHAP 值聚类 vs 对原始特征聚类

| 维度 | 原始特征聚类 | SHAP 值聚类 (本教程) |
|------|------------|-------------------|
| 输入 | 患者的特征值（Age, year, …） | 模型对每个患者的**解释**（SHAP 值） |
| 分组依据 | "患者的特征值相似" | "模型的决策逻辑相似" |
| 产出 | 临床表型群组 | **决策模式群组** |
| 解释 | "这些患者年龄相近" | "这些患者的预测主要由 year 驱动" |

```
一个类比:

原始特征聚类 = 把读者按"年龄、性别、职业"分组
              → 得到"30岁男性上班族"组

SHAP 值聚类 = 把读者按"对一本书的理解方式"分组
              → 得到"注重情节发展"组
              → 得到"关注人物描写"组
```

### 1.3 课堂引入：为什么对 SHAP 聚类？

```
设想一个癌症预测模型:

患者 A:
  年龄=65, 年份=2015, 诊断方式=活检
  预测: MORTO (死亡)
  解释: 年份较早 → 推低, 年龄较大 → 推低

患者 B:
  年龄=45, 年份=2020, 诊断方式=影像
  预测: VIVO (存活)
  解释: 年份较晚 → 推高, 年龄适中 → 推高

患者 C:
  年龄=70, 年份=2023, 诊断方式=手术
  预测: MORTO (死亡)
  解释: 年份较晚 → 推高 BUT 诊断方式异常 → 强烈推低

A 和 B 的决策模式不同 (一个主要看年份, 一个看年份+年龄)
A 和 C 的决策模式也不同 (C 的诊断方式起到了决定性作用)

问题是: 有多少种不同的"决策模式"?
        哪些患者属于同一种模式?
        → SHAP 聚类回答这些问题
```

## 三、实验结果

### 3.1 最佳聚类数选择

| n_clusters | Silhouette Score | 判断 |
|-----------|-----------------|------|
| **2** | **0.6209** | ✅ **最优，结构最清晰** |
| 3 | 0.6205 | 几乎相同，但 k=2 更简洁 |
| 4 | 0.5692 | 边界模糊 |
| 5 | 0.5393 | 聚类质量下降 |

> **教学点**：SHAP 空间的最佳聚类数 = 2，说明数据集中存在两种截然不同的"决策模式"。k=3 的 Silhouette 与 k=2 几乎相同（0.6205 vs 0.6209），但 k=2 更简洁。

### 3.2 PCA 解释率

| 主成分 | 方差解释率 | 累积 |
|--------|-----------|------|
| PC1 | **70.11%** | 70.11% |
| PC2 | 19.17% | **89.28%** |

**教学点**：仅 2 个 PC 就解释了 89.28% 的 SHAP 值方差——这意味着 SHAP 空间**结构清晰**，6 个特征的贡献可以用 2 个维度很好地表示。

### 3.3 两个聚类的完整画像

| 维度 | Cluster 1 | Cluster 2 |
|------|-----------|-----------|
| **样本数** | 1,002 (33.4%) | 1,998 (66.6%) |
| **VIVO 比例** | **0.3%** | **61.8%** ⬆️ |
| **MORTO 比例** | 99.7% | 38.2% |
| **PCA 位置** | 偏左紧凑 | 偏右分散 |
| **相对突出的特征** | **Diagnostic.means (1.65x)** ⬆️, Raca.Color (0.75x), year (0.69x) | Extension (1.22x), Code.Profession (1.20x), Age (1.17x) |
| **特征均值倾向** | Diagnostic.means 偏高 | year 较高, Age 中等 |

### 3.4 决策模式解读

```
Cluster 1 — "特殊模式" (33.4% 患者)
  画像: 少数患者, 几乎全部死亡
  特征: Diagnostic.means (1.65x 全局!) — 最重要的区分
        Raca.Color (0.75x, 低于全局)
        year (0.69x, 低于全局)
  VIVO: 极低 (0.3%) — 几乎都预测死亡
  解读: 这部分患者的预测主要由"诊断方式"驱动
        诊断方式对这部分患者的预测影响
        是全局平均的 1.65 倍

Cluster 2 — "主流模式" (66.6% 患者)
  画像: 大多数患者, 存活率远高于总体
  特征: 模型使用相对均衡的特征组合
        Extension (1.22x), Code.Profession (1.20x), Age (1.17x)
        相对全局有一定的特征突出
  VIVO: 较高 (61.8%) — 存活率远高于总体
  解读: 这部分患者的预测比较"常规"
        多个特征共同贡献, 没有单一特征极度突出
```

## 四、核心教学讨论

### 4.1 为什么 Silhouette Score 显示 k=2 最优？

```
评分含义回顾:
  Silhouette Score = (b - a) / max(a, b)
    a = 样本与自身聚类的平均距离 (紧密度)
    b = 样本与最近其他聚类的平均距离 (分离度)

k=2: 0.6263 → 聚类之间分离良好
k=3: 0.5936 → 分裂出一个过渡性聚类, 边界模糊
k=4: 0.5190 → 过度分裂, 聚类内部紧密度下降

教学结论:
  在 SHAP 空间中, 本数据集只有 2 种"决策模式"
  → 增加聚类数只是把其中一种模式再细分
  → 这不是"真正的子群"
```

### 4.2 "相对重要性"的创新用法

本教程的关键创新是 **相对重要性** (Relative Importance)：

```python
rel_imp = this_cluster_importance / global_importance
```

| 特征 | 全局 SHAP | Cluster 1 | Cluster 2 | rel_imp (C2) |
|------|-----------|-----------|-----------|-------------|
| year | 0.269 | 0.250 | 0.350 | 1.30x |
| Diagnostic.means | 0.061 | 0.030 | **0.112** | **1.83x** ⬆️ |
| Extension | 0.011 | 0.005 | 0.017 | **1.55x** ⬆️ |
| Code.Profession | 0.037 | 0.030 | 0.043 | 1.14x |
| Age | 0.031 | 0.034 | 0.023 | 0.74x (↓) |
| Raca.Color | 0.024 | 0.025 | 0.018 | 0.78x (↓) |

```
相对重要性的教学价值:

> 1.0x: 该特征在此聚类中比全局平均更重要
< 1.0x: 该特征在此聚类中比全局平均更不重要

Cluster 2 的"旗帜": Diagnostic.means (1.83x)
  → "如果你属于 Cluster 2, 诊断方式对你的预测影响
     是全局平均的 1.83 倍"
```

### 4.3 从聚类到临床洞察

```
本实验的临床转化:

发现:
  存在一个患者亚群 (Cluster 2, 20.6%),
  其存活预测主要由 Diagnostic.means 和 Extension 驱动

临床含义:
  对于这部分患者, 诊断方式和肿瘤扩展程度
  是比年份和年龄"更关键"的预后因素

可能的解释:
  Cluster 2 患者可能有特殊的诊断方式 (如组织活检)
  或特殊的肿瘤扩展模式 (如早期发现、局限)
  这些因素对存活的影响大于一般因素

论文写作:
  "SHAP 聚类分析揭示了一个独特的患者亚群 (n=619),
   其预后判断主要由诊断方式和肿瘤扩展程度驱动,
   提示这一亚群可能需要差异化的临床关注。"
```

### 4.4 与教程 6（降维）的关联

```
教程 6 (PCA + K-Means 聚类):
  对原始特征 -> 聚类 -> ARI ≈ -0.001
  结论: 原始特征空间无天然聚类

本教程 (SHAP 聚类):
  对 SHAP 值 -> 聚类 -> Silhouette = 0.6263
  结论: SHAP 空间有清晰的聚类结构

为什么不同?
  原始特征聚类: 试图找到"特征值相似的患者"
  SHAP 值聚类: 试图找到"模型解释模式相似的患者"

所以:
  患者本身的特征不一定能形成族群
  但模型对患者的"解释"形成了清晰的族群
  也就是说——模型对患者"一视同仁"的程度是有限的
  → 不同的患者适用不同的解释逻辑
  → 这是"个性化医学"在解释层面的体现
```

## 五、扩展内容

### 5.1 其他聚类算法

| 算法 | 优点 | 缺点 | 对 SHAP 聚类的适用性 |
|------|------|------|-------------------|
| **K-Means** (本教程) | 快速、直观 | 球形簇假设 | 对 SHAP 空间足够 |
| DBSCAN | 发现任意形状、自动检测噪点 | 对密度差敏感 | 适合发现异常决策模式 |
| 层次聚类 | 提供完整的层次结构 | 计算成本高 | 适合探索性阶段 |
| GMM (高斯混合) | 软分类、概率输出 | 需要大量样本 | 适合不确定分类 |

### 5.2 SHAP 聚类在医学论文中的位置

```
在医学论文中, SHAP 聚类通常出现在:

(1) 探索性分析部分:
  "为了探索患者亚群之间的模型行为差异,
   我们对 SHAP 值进行了 K-Means 聚类..."

(2) 亚组分析部分:
  "我们进一步将患者分为两个 SHAP 聚类,
   并比较了每个聚类中的特征重要性模式..."

(3) 讨论部分:
  "SHAP 聚类分析揭示了在 33.4% 的患者中,
   诊断方式对模型预测的影响显著增强 (rel_imp=1.65x),
   这可能反映了不同诊断路径的预后差异..."
```

### 5.3 局限性

| 局限性 | 原因 | 缓解 |
|--------|------|------|
| K-Means 假设球形簇 | SHAP 空间可能不是球形 | 使用 DBSCAN 或层次聚类交叉验证 |
| 聚类标签无临床含义 | "Cluster 1" 不是诊断代码 | 通过特征画像赋予临床解释 |
| 聚类结果依赖模型 | 不同模型产生不同 SHAP 聚类 | 在多个模型上验证聚类稳定性 |
| 小样本下的聚类不稳定 | SHAP 值随训练数据变化 | 增加样本量或使用自助法 |

### 5.4 聚类分析的全流程建议

```
SHAP 聚类 → 临床验证 → 独立验证

第一步: SHAP 聚类 (本教程)
  在训练/测试数据上执行, 发现 2-4 个聚类

第二步: 聚类特征定义
  用 PCA 权重或线性判别分析
  找到哪些特征区分了聚类
  → 为新患者分配聚类标签

第三步: 独立外部验证
  在另一数据集中, 用同样的规则分配聚类
  验证: 聚类间的 AUC/特征模式是否一致

第四步: 临床转化
  如果聚类有明确的临床特征画像
  → 可以开发针对不同聚类的"定制化"解释
```

## 六、总结与思考题

### 6.1 核心收获

1. **SHAP 空间有清晰的聚类结构**（Silhouette=0.6209），最佳聚类数 = 2
2. **Cluster 1（33.4%）** = "特殊模式"——Diagnostic.means（1.65x）异常突出，几乎全部预测死亡（VIVO=0.3%）
3. **Cluster 2（66.6%）** = "主流模式"——Extension（1.22x）、Code.Profession（1.20x）、Age（1.17x）相对突出，存活率 61.8%
4. **PCA 仅 2 个主成分就解释了 89.28% 的 SHAP 方差**——说明 SHAP 空间本质上是低维的
5. **对 SHAP 值聚类 vs 对原始特征聚类**——这是"解释模式"聚类 vs "表型"聚类的根本区别

### 6.2 思考题

1. **（基础）** 为什么本教程中 Silhouette Score 建议 k=2，但 k=3 的评分（0.6205）几乎与 k=2（0.6209）相同？这说明了什么关于"最佳聚类数"的判断？

2. **（进阶）** Cluster 1 中 Diagnostic.means 的相对重要性是 1.65x。这个"1.65x"的置信区间是多少？使用自助法（Bootstrap）重采样 100 次，计算 rel_imp 的 95% 置信区间。如果区间跨过 1.0，诊断方式在 C1 中的"特异重要性"是否还有统计意义？

3. **（进阶）** 本教程发现 2D PCA 解释了 89.28% 的方差。如果只用前 2 个 PC 重建 SHAP 值并重新聚类，聚类结果会改变吗？试比较原始 SHAP 聚类 vs PCA 重建 SHAP 聚类的一致性（用 ARI 或 NMI 度量）。

4. **（拓展）** 教程 6 中基于原始特征的 K-Means 聚类得到了 ARI ≈ -0.001（与真实标签几乎无关）。本教程中基于 SHAP 值的聚类得到了清晰的 2 个簇。比较这两种聚类的差异——为什么原始特征聚类找不到结构，而 SHAP 聚类可以？这对"医学数据是否天然可分"这个问题有什么启发？

5. **（拓展）** 假设你在 Cluster 1 中发现了一个 Sub-cluster，其中 100% 是 VIVO 患者且 Diagnostic.means 的 rel_imp=3.0x。你怎么向临床医生解释这个 Sub-cluster？它是"模型的盲点"（模型过度依赖某个特征）还是"真正的临床子群"？

6. **（实践）** 将 XGBoost 替换 Random Forest 作为基础模型，重复 SHAP 聚类分析。比较两个模型的聚类结果：聚类数是否相同？聚类的特征画像是否一致？如果不同，这说明了什么关于"模型之间的决策逻辑差异"？
