## 三、各模块详解

### 模块 A：全局解释

#### A1. Feature Importance (Gini)

```
Feature           Gini Importance    含义
───────────────────────────────────────────
year                   0.7009     中轴特征，RF 200棵树的首次分裂首选
Diagnostic.means       0.1071     诊断方式的判别力
Age                    0.0725     年龄有贡献但远低于 year
Code.Profession        0.0655     职业编码
Extension              0.0311     肿瘤扩展
Raca.Color             0.0228     种族颜色（最低）
```

Gini Importance 有**高基数偏倚**——year 因为是连续特征，在树分裂时有更多分裂点可选，容易被选为分裂特征。但这不意味着其他特征不重要，只是 Gini 的"偏好"。

#### A2. Permutation Importance

本实验的核心教学发现：

| 特征 | Permutation (ΔAUC) | 解读 |
|------|-------------------|------|
| **year** | **+0.0527** | 打乱 year → AUC 显著下降 → 关键特征 |
| Code.Profession | +0.0142 | 中等重要 |
| Diagnostic.means | +0.0137 | 中等重要 |
| Raca.Color | +0.0066 | 轻微贡献 |
| Extension | +0.0026 | 几乎无贡献 |
| **Age** | **-0.0034** | ⚠️ 打乱后 AUC 反而上升？ |

> **教学陷阱**：Age 的 Permutation Importance 为负值（-0.0034）。这并不意味着"Age 是噪声"——而是 Age 和 year 之间存在**部分冗余**。打乱 Age 后，模型可以依赖 year 来补偿，AUC 反而因为降低冗余而略微上升。

#### A3. SHAP 全局分析

SHAP 的核心思想：**每个特征对预测的贡献是"在存在其他特征的情况下"计算的。**

```
SHAP 工作的三句话:

1. base value (基准值) = 数据集中平均预测概率
   → 本实验中 = 4.93% (VIVO 比例)

2. 对于每个样本:
   预测概率 = base value + Σ(各特征的 SHAP 值)
   
3. SHAP 值可正可负:
   + → 推高预测概率 (→ VIVO)
   - → 推低预测概率 (→ MORTO)
```

**蜂群图 (Bee Swarm)** 的关键解读：

```

                        ●●●●●●●●●○○○○○○○○○○

Diagnostic.means        ■■■■■■■■■                
                        ●●●●●●●○○○○○○○○○○○○○○
Raca.Color              ■■■■■■■             
                        ●●●●●●○○○○○○○○○○○○○○○○○
Extension               ■■■■■              
                        ●●●●○○○○○○○○○○○○○○○○○○○○○

**依赖图 (Dependence Plot)** 的教学价值：

```

### 模块 B：局部解释

#### B1. SHAP Waterfall Plot

以预测概率最高的 VIVO 样本为例：

```
base value: 0.049 (平均 VIVO 概率)
                              ↓
year = +0.3271   ────────────→ 大幅推高
Code.Profession = +0.0638   ──→ 推高
Diagnostic.means = +0.0350  ──→ 推高
Raca.Color = +0.0138        ──→ 推高
Extension = +0.0051         ──→ 轻微推高
Age = -0.0135               ──→ 轻微推低
                              ↓
最终预测概率: 0.049 + 0.3271 + 0.0638 + ... ≈ 0.94
```

**教学要点**：Waterfall Plot 的本质是"从基准概率出发，各特征一步步增减，最终到达预测概率"的叙事过程。这是向临床医生解释模型最直觉的方式。

#### B2. LIME

LIME 的思路与 SHAP 不同：

```
LIME 的三步:

1. 在你关心的样本附近，随机生成扰动样本
2. 用黑箱模型对这些扰动样本预测
3. 在扰动样本上训练一个"简单模型" (线性回归)
   → 简单模型的系数 = 局部特征贡献
```

本实验的 VIVO 高概率样本的 LIME 结果：

| 特征 | 条件 | 贡献 | 方向 |
|------|------|------|------|
| year | > 0.82 | +0.2806 | → VIVO |
| Diagnostic.means | ≤ -0.11 | +0.2048 | → VIVO |
| Raca.Color | ≤ -0.45 | +0.0995 | → VIVO |
| Extension | ≤ -0.21 | +0.0781 | → VIVO |
| Age | -0.58 到 0.13 之间 | -0.0085 | → MORTO |

**注意**：LIME 不是给出具体的特征值，而是将特征值**离散化为区间/阈值**。这是 LIME 和 SHAP 的关键区别之一。

### 模块 C：SHAP vs LIME 深度对比

#### C1. 同一患者的解释对比

| 维度 | SHAP | LIME |
|------|------|------|
| 最重要的特征 | year (+0.3271) | year (> 0.82, +0.2806) |
| 排名 #2 | Code.Profession (+0.0638) | Diagnostic.means (≤ -0.11, +0.2048) |
| 排名 #3 | Diagnostic.means (+0.0350) | Raca.Color (≤ -0.45, +0.0995) |
| 排名 #4 | Raca.Color (+0.0138) | Extension (≤ -0.21, +0.0781) |
| 排名 #5 | Extension (+0.0051) | Code.Profession (> -0.42, +0.0078) |
| Age 方向 | 负 (-0.0135) | 负 (-0.0085) |

**一致性**：两个方法都认为 year 是最重要特征，Age 负贡献——**在最重要的判断上一致**。
**差异**：Code.Profession 和 Diagnostic.means 的排名互换——这是方法差异导致的正常现象。

#### C2. 核心差异表

| 维度 | SHAP | LIME | 教学含义 |
|------|------|------|---------|
| **理论基础** | Shapley Value (博弈论) | 局部线性近似 | SHAP 有严谨数学保证，LIME 是近似 |
| **计算速度** | TreeExplainer 快 | 需要大量扰动采样 | SHAP 做一次，LIME 每次 query 都要重新采样 |
| **输出类型** | 原始 SHAP 值 | 离散化阈值 | SHAP："year=1.9 贡献 +0.33" |
| | | | LIME："year>0.82 贡献 +0.28" |
| **稳定性** | ✅ 确定性的 | ❌ 有随机性 (扰动采样) | 同一样本跑两次 LIME 可能不同 |
| **可微调** | 不支持 | 可调整扰动幅度和核宽度 | LIME 可调参数多，但也是双刃剑 |
| **医学论文** | 95% 以上使用 SHAP | 较少 | SHAP 已成为医学 XAI 标准 |

#### C3. 什么时候用 SHAP，什么时候用 LIME？

```
决策树:

模型类型:
  树模型 (RF/GBDT/XGB/LGB)  →  SHAP TreeExplainer (快)
  深度学习/SVM/任意模型      →  SHAP KernelExplainer (慢) 或 LIME

应用场景:
  论文/汇报                 →  SHAP (标准、严谨)
  临床决策支持系统            →  SHAP + LIME 互补
  快速原型/调试              →  LIME (轻量、灵活)

性能要求:
  需要确定性解释             →  SHAP
  可以接受随机波动            →  LIME 也够用
```

### 模块 D：LIME 稳定性实验

#### D1. 实验结果

| 样本 | 预测概率 | 真实标签 | Top 5 LIME 特征 |
|------|---------|---------|-----------------|
| **A** (idx=229) | **0.9400** | VIVO (1) | year > 0.82, Diagnostic.means ≤ -0.11, Raca.Color ≤ -0.45, Extension ≤ -0.21, Age (-0.58~0.13) |
| **B** (最近邻) | **0.8988** | VIVO (1) | year > 0.82, Diagnostic.means ≤ -0.11, Raca.Color ≤ -0.45, Extension ≤ -0.21, Age (0.13~0.72) |

**一致性**：Top 4 特征完全相同！仅有 Age 的区间阈值不同（样本 A: -0.58~0.13, 样本 B: 0.13~0.72）。

> **教学发现**：在这个案例中 LIME 的稳定性出奇地好。但这取决于数据密度（本实验有 7,000 个训练样本，扰动采样空间足够丰富）。在**小样本**或**稀疏区域**，LIME 的稳定性会显著下降。

#### D2. LIME 不稳定的场景

```
LIME 不稳定的常见原因:

1. 样本在特征空间的"边缘":
   扰动样本可能进入"不现实"的区域
   → 线性近似失效

2. 训练样本太少:
   模型决策边界附近的样本空间稀疏
   → 不同采样导致不同线性模型

3. 扰动幅度 (kernel width):
   幅度太大 → 线性近似粗糙
   幅度太小 → 局部过拟合

对策:
   - 固定 random_state (本实验的做法)
   - 增大采样数 (num_samples > 5000)
   - 与 SHAP 结果交叉验证
```

## 五、扩展内容

### 5.1 SHAP 的理论基础：Shapley Value

SHAP 基于博弈论中的 Shapley Value：

| 概念 | 类比 | SHAP 对应 |
|------|------|----------|
| 玩家 | 特征 | year, Age, Code.Profession... |
| 联盟 | 特征子集 | 用部分特征预测 |
| 收益 | 预测结果 | 预测概率 |
| 贡献分配 | Shapley Value | SHAP 值 |

Shapley Value 是**唯一**满足以下四个公理的特征贡献分配方法：

| 公理 | 含义 | 为什么重要 |
|------|------|-----------|
| **效率** | 所有特征贡献之和 = 总预测 - 基准值 | 保证解释是完整的 |
| **对称性** | 两个特征贡献相同 → 分配相同 | 公平 |
| **虚拟性** | 不贡献的特征 → 分配 0 | 不引入噪声 |
| **可加性** | 合并两个模型的 SHAP = 分别计算的和 | 一致性保证 |

### 5.2 SHAP 的局限性

| 局限性 | 说明 | 缓解方案 |
|--------|------|---------|
| **计算成本** | KernelExplainer 极慢 | 用 TreeExplainer (仅限于树模型) |
| **特征独立性假设** | Shapley Value 假设特征可以自由组合 | 实际数据中特征高度相关时解释不稳定 |
| **因果关系 ≠ 关联** | SHAP 是相关性不是因果 | 与因果推断方法结合 |
| **对异常值敏感** | 单个异常样本的 SHAP 值可能被扭曲 | 多个数据集/折叠验证 |

### 5.3 LIME 的扩展：SP-LIME

LIME 有个叫 SP-LIME (Submodular Pick LIME) 的扩展，用来选择 **最有代表性** 的一组局部解释：

```
问题: LIME 只能解释单个样本
      → 如果要向医生解释模型，选哪几个样本？

SP-LIME 的方案:
  1. 对所有 (或随机) 样本计算 LIME
  2. 用子模选择选出一组"典型解释"
  3. 每个解释代表一种"决策模式"

示例:
  模式 1: year高 + D.means异常 → VIVO (20%的样本)
  模式 2: year低 + Age高 → MORTO (35%的样本)
  ...
```

### 5.4 XAI 可解释性谱系

```
可解释性从低到高:

  黑箱 → 局部近似 → 全局近似 → 内在可解释
   │         │          │          │
  DNN     SHAP/LIME   GAMs/SP-LIME   Logistic回归
   │         │          │          │
  无法解释  逐样本解释  全局模式   系数直接解读
```

### 5.5 其他解释方法概览

| 方法 | 类型 | 适用模型 | 特点 |
|------|------|---------|------|
| **PDP (Partial Dependence)** | 全局 | 任意 | 特征对预测的平均边际效应 |
| **ICE (Individual Conditional Expectation)** | 局部 | 任意 | 每个样本的边际效应 |
| **Anchors** | 局部 | 任意 | 决策"锚点" — 等价于决策规则 |
| **Grad-CAM** | 局部 | CNN | 图像中哪些区域对分类重要 |
| **Integrated Gradients** | 局部 | DNN | 沿着梯度路径累积 |

## 六、总结与思考题

### 6.1 核心收获

1. **三种全局重要性方法高度一致**：year 是最重要特征，Gini/Permutation/SHAP 的排名相似
2. **SHAP 的理论基础最严谨**：基于博弈论的 Shapley Value，满足效率、对称性、虚拟性、可加性四条公理
3. **SHAP Waterfall Plot 是向医生解释模型的最佳工具**：从基准概率出发，各特征"增减"的叙事最自然
4. **SHAP 与 LIME 的本质区别**：SHAP = 博弈论完备解，LIME = 局部线性近似。在最重要的判断上一致，但细节有差异
5. **LIME 的稳定性是双刃剑**：本实验中稳定性好，但在样本稀疏区域可能失效
6. **Permutation Importance 的特殊发现**：Age 的负值提示与 year 存在冗余，这是其他方法无法揭示的
7. **模型可解释性不是选做题**：在医学 AI 论文中，SHAP 已经成为审稿人的标准要求

### 6.2 思考题

1. **（基础）** 本实验中 Gini Importance 显示 year=0.7009，而 Diagnostic.means=0.1071。Gini Importance 的"高基数偏倚"是指什么？如果所有特征都是二分变量，Gini Importance 的分布会改变吗？

2. **（进阶）** Permutation Importance 发现 Age 的 ΔAUC 为负值（-0.0034）。解释为什么打乱一个特征后 AUC 反而可能上升？这是否意味着 Age 是"无用特征"？

3. **（进阶）** SHAP 的 base value = 0.049 (平均 VIVO 比例)。如果一个患者的预测概率是 0.01，但他的所有特征 SHAP 值都是正的（推高），可能吗？为什么？（提示：特征间的非线性交互）

4. **（拓展）** SHAP Waterfall 给出 year = +0.3271，LIME 给出 year > 0.82 = +0.2806。这两个值代表同一个概念吗？哪个更精确？为什么 LIME 会给出"阈值"条件而不是具体数值？

5. **（拓展）** LIME 稳定性实验中，两个欧氏距离 0.93 的样本的 Top 5 特征高度一致。但在某些情况下 LIME 可能给出完全不同的解释。请你设计一个实验，系统性地评估 LIME 在小样本（n < 500）下的稳定性——哪些因素（样本密度、扰动幅度、模型复杂度）最影响稳定性？

6. **（实践）** 尝试针对 Logistic Regression 模型（不是 RF）运行 LIME，观察 LIME 的局部解释与 LR 系数是否一致。如果 LR 是线性模型，LIME 的"局部线性近似"还有什么价值？
