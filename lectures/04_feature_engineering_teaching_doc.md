# 案例教程 4：特征工程

## 一、前序知识要求

### 1.1 机器学习基础

| 概念                         | 简要说明                             | 与本教程的关系                              |
| -------------------------- | -------------------------------- | ------------------------------------ |
| **特征（Feature）**            | 输入模型的变量，是模型"看到"的唯一信息             | 特征质量直接决定模型性能上限                       |
| **特征缩放（Feature Scaling）**  | 改变特征的量纲和取值范围                     | StandardScaler vs RobustScaler 的比较核心 |
| **梯度下降（Gradient Descent）** | 逻辑回归的参数优化算法                      | 特征尺度影响梯度下降的收敛速度和路径                   |
| **逻辑回归系数**                 | 每个特征对应一个权重 w，表示该特征对 log-odds 的贡献 | 标准化后系数可比，可解释为特征重要性                   |
| **非线性关系**                  | 特征与目标之间不是简单的直线关系                 | Age\_Sq（年龄平方）用于捕捉 U 型效应              |
| **交互效应（Interaction）**      | 两个特征联合起来对目标的影响 ≠ 各自影响之和          | Gender × Age\_Group 构造交互特征           |
| **过拟合（Overfitting）**       | 模型学到噪声而非信号，在训练集好、测试集差            | 特征构造过多时需警惕过拟合                        |

### 1.2 统计学基础

| 概念               | 说明                                          |
| ---------------- | ------------------------------------------- |
| **均值（μ）**        | StandardScaler 的中心化基准                       |
| **标准差（σ）**       | StandardScaler 的缩放基准，受离群值影响                 |
| **中位数（Median）**  | RobustScaler 的中心化基准，不受离群值影响                 |
| **四分位距（IQR）**    | RobustScaler 的缩放基准，不受离群值影响                  |
| **偏度（Skewness）** | 判断是否需要 RobustScaler（偏态分布时 RobustScaler 更稳健） |

### 1.3 前序教程回顾

- EDA 阶段：Age 近似正态（偏度 -0.68），Code.Profession 右偏（偏度 0.94）
- 数据预处理：均值插补保留样本量，但不一定能保留分布形态
- 统计学分析：year 的 Rank-biserial r = 0.44，属于大效应量（量纲差异巨大的表现之一）

## 二、本次实践内容与目的

### 2.1 核心目的

> **理解特征质量决定模型上限——通过标准化加速模型收敛、通过特征构造注入领域知识，从而提升模型性能。**

特征工程是机器学习中最具"性价比"的环节。调参可能带来 1-2% 的提升，而一个好的特征构造可以带来 5-10% 甚至更高的提升。正如业界常说：**"垃圾进，垃圾出"（Garbage In, Garbage Out）**。

### 2.2 实践内容全景

#### 第一部分：数据标准化比较

| 方法                 | 公式                 | 中心   | 尺度         | 对离群值的敏感性                  |
| ------------------ | ------------------ | ---- | ---------- | ------------------------- |
| **Raw（原始数据）**      | x                  | —    | —          | —                         |
| **StandardScaler** | (x - μ) / σ        | 均值 μ | 标准差 σ      | **敏感**（μ 和 σ 都被离群值拉偏）     |
| **RobustScaler**   | (x - median) / IQR | 中位数  | IQR（Q3-Q1） | **不敏感**（中位数和 IQR 不受离群值影响） |

#### 第二部分：特征构造

| 新特征                     | 基于                  | 领域知识动机            |
| ----------------------- | ------------------- | ----------------- |
| **Age\_Group**          | Age                 | 不同年龄段的癌症类型和预后截然不同 |
| **Age\_Sq**             | Age                 | 年龄与死亡率可能存在 U 型关系  |
| **Year\_Decade**        | year                | 医疗技术进步使诊断年代影响存活率  |
| **Year\_From\_2000**    | year                | 存活率随时间的线性改善趋势     |
| **Gender\_x\_AgeGroup** | Gender × Age\_Group | 性别的效应在不同年龄段表现不同   |
| **Is\_Child**           | Age                 | 儿童癌症在病理和治疗上具有特殊性  |
| **Age\_Centered**       | Age                 | 提高截距项的可解释性        |

### 2.3 关键发现摘要

#### 标准化对比

| 指标         | Raw        | StandardScaler | RobustScaler | <br /> | <br /> |
| ---------- | ---------- | -------------- | ------------ | :----- | :----- |
| **AUC**    | 0.8698     | 0.8698         | 0.8698       | <br /> | <br /> |
| **Recall** | 0.9026     | 0.9026         | 0.9026       | <br /> | <br /> |
| **Brier**  | 0.1437     | 0.1437         | 0.1436       | <br /> | <br /> |
| **迭代次数**   | **174**    | **10**         | **11**       | <br /> | <br /> |
| **耗时**     | **0.257s** | **0.021s**     | **0.022s**   | <br /> | <br /> |
| \*\*       | 系数         | 均值\*\*         | 0.3482       | 0.6672 | 1.1986 |

> **核心发现**：标准化**不改变模型性能**（AUC/Recall/Brier 几乎一致），但**极大加速收敛**——StandardScaler 只需 10 次迭代（Raw 需要 174 次），迭代速度提升约 **17 倍**！

#### 特征构造对比

| 指标         | Base（6 特征） | Engineered（13 特征） | 变化      |
| ---------- | ---------- | ----------------- | ------- |
| **AUC**    | 0.8698     | **0.8737**        | +0.0039 |
| **Recall** | **0.9026** | 0.8886            | -0.0140 |
| **Brier**  | 0.1437     | **0.1409**        | -0.0028 |
| **PR-AUC** | 0.7412     | **0.7530**        | +0.0118 |

> **核心发现**：特征构造提升了 AUC (+0.39%) 和校准度（Brier -1.9%），但 Recall 有所下降。这说明新特征带来的信息并非均衡改善所有指标——在轻度不平衡数据中，非线性特征的加入可能使决策边界变得更加复杂，反而降低了对少数类的召回。

## 三、扩展内容

### 3.1 为什么标准化不影响 AUC 但影响系数？

这是一个关键的理解点，揭示了逻辑回归的数学本质：

```
逻辑回归的决策边界:  w·x + b = 0

标准化:  x' = (x - μ) / σ
        w' = w × σ  (经过缩放)
        
决策边界不变:  (w·σ) × [(x - μ) / σ] + b' = w·x - w·μ + b' = 0
        
→ 标准化只是"重新参数化"，不改变模型的预测能力
→ 但影响系数的数值范围和优化路径
```

**系数可比性的实际意义**：

| 场景                 | 未标准化        | 标准化后        |
| ------------------ | ----------- | ----------- |
| Age 系数             | -0.024      | -0.402      |
| year 系数            | 0.194       | 0.762       |
| Code.Profession 系数 | -0.001      | -0.348      |
| **结论**             | 系数不可比（量纲不同） | 系数可比较（量纲统一） |

标准化后，我们可以直接从系数大小判断特征重要性：**year > Age ≈ Code.Profession**。而未标准化时，year 的系数 0.194 看起来最大，但 year 的取值范围只有 2000-2015（跨度约 15），而 Age 跨度约 0-120——量纲不同导致系数无法直接比较。

### 3.2 StandardScaler vs RobustScaler：怎么选？

| 数据特点   | 推荐方法           | 原因                       |
| ------ | -------------- | ------------------------ |
| 近似正态分布 | StandardScaler | 保留分布形状，均值和标准差具有统计意义      |
| 有少量离群值 | RobustScaler   | 离群值不影响中位数和 IQR           |
| 严重偏态分布 | RobustScaler   | 均值会被拉偏，中位数更稳健            |
| 稀疏数据   | 两者均需谨慎         | 可用 MaxAbsScaler          |
| 需要保留零值 | RobustScaler   | StandardScaler 可能使零值变为负数 |

**本数据集的应用**：

- Age（近似正态，偏度 -0.68）→ StandardScaler 即可
- Code.Profession（右偏，偏度 0.94）→ RobustScaler 更合适
- year（左偏，偏度 -1.01）→ RobustScaler 更合适

### 3.3 特征构造的"黄金法则"

```
好特征 = 领域知识 + 数据形态 + 模型特性
```

| 法则          | 解释            | 反面案例                      |
| ----------- | ------------- | ------------------------- |
| **可解释性**    | 特征要有明确的含义     | 盲目对 Age 做 Box-Cox 变换      |
| **不泄漏未来信息** | 特征不能用到未来的数据   | 用 Date.of.Death 预测存活（作弊）  |
| **避免冗余**    | 高度相关的特征会造成共线性 | Age 和 Age\_Centered 本质上一样 |
| **避免爆炸**    | 不要盲目做所有可能的交互  | 6 个特征两两交互 → 21 个新特征       |
| **验证有效性**   | 用交叉验证验证新特征的价值 | 构造了一堆特征但模型没提升             |

### 3.4 更多特征构造方向

#### 基于本数据集的构造建议

| 方向       | 新特征                        | 领域知识支撑            | 实现难度          |
| -------- | -------------------------- | ----------------- | ------------- |
| **时间窗口** | 诊断到死亡的间隔                   | 生存时间是最直接的预后指标     | 低（需清理日期）      |
| **癌症亚型** | 形态学 + 解剖部位的分组              | 同部位不同病理类型预后不同     | 中（需 ICD 编码知识） |
| **合并诊断** | 多个诊断编码的共现                  | 多原发癌患者的生存率不同      | 中             |
| **社会经济** | Education + Profession 的融合 | 社会经济地位影响就医时机      | 低             |
| **地理区域** | Naturality.State 的区域分组     | 不同区域的医疗资源差异       | 低             |
| **诊断延迟** | Age vs Diagnosis Date 的关系  | 年轻患者可能更晚被发现       | 中             |
| **风险评分** | 多个风险因素的线性组合                | 类似 Charlson 合并症指数 | 高（需医学文献支持）    |

#### 通用特征构造方法

| 方法                        | 公式                 | 适用场景              |
| ------------------------- | ------------------ | ----------------- |
| **多项式特征**                 | x², x³, √x, log(x) | 捕捉非线性关系           |
| **比例特征**                  | x₁ / x₂            | 两个特征的比值有意义时（如腰臀比） |
| **聚合特征**                  | groupby 的均值/中位数    | 按群体统计信息           |
| **分箱（Binning）**           | 连续 → 离散区间          | 医学常用的年龄分箱、BMI 分箱  |
| **计数特征**                  | 某个类别出现的频次          | 罕见类别合并            |
| **目标编码（Target Encoding）** | 用目标变量的均值编码类别       | 高基数类别特征           |
| **差值特征**                  | x₁ - x₂            | 前后对比、时间差          |
| **嵌入特征**                  | 用预训练模型提取的表征        | 文本、图像等非结构化数据      |

### 3.5 为什么标准化后模型收敛更快？

特征尺度对梯度下降的影响可以用"等高线"直观理解：

```
原始尺度差异大时:           标准化后:
    等高线呈"椭圆形"          等高线呈"圆形"
    梯度方向不断振荡           梯度直指最优方向
    收敛路径如"之"字         收敛路径近似直线
```

**数值解释**：

以两个特征为例：

- 特征 1（Age）：范围 0-120，梯度 ≈ -0.2
- 特征 2（year）：范围 2000-2015，梯度 ≈ 0.05

**未标准化**：

- 梯度向量 = \[-0.2, 0.05]
- 对特征 1 的步长远大于特征 2 → 优化路径振荡

**标准化后**：

- 两个特征都在 \[-3, 3] 范围内
- 梯度向量各分量尺度一致 → 优化路径平稳

这就是从 174 次迭代降到 10 次的核心原因。

### 3.6 Recall 下降的反直觉结果

在本实验中，特征构造后 AUC 上升但 Recall 下降——这看似矛盾，实则揭示了类别不平衡下的重要现象：

| 原因              | 说明                                                                   |
| --------------- | -------------------------------------------------------------------- |
| **决策边界复杂度**     | 13 个特征比 6 个特征的决策边界更复杂，对少数类的拟合更困难                                     |
| **逻辑回归的线性限制**   | 逻辑回归是线性模型，非线性特征（如 Age\_Sq）的引入不一定会提升少数类的边界清晰度                         |
| **Brier 改善的代价** | Brier 从 0.1437 降到 0.1409（校准度提升），但这主要是多数类的概率更准确了，少数类的 Recall 可能为此付出代价 |
| **阈值效应**        | 默认阈值 0.5 对不平衡数据不是最优的，Recall 可能对阈值非常敏感                                |

> **教学要点**：不要只看单一指标。AUC 升 + Recall 降 + Brier 降，说明模型在整体区分力和校准度上有提升，但对少数类的敏感度反而下降了。这正是"选择正确的评价指标"的重要性。

### 3.7 更值得探索的构造方向

> 以下内容可以在教学延伸中作为独立实验展开。

#### 方向一：构造特征的"组合爆炸"

- 6 个基础特征两两交互 = C(6,2) = 15 个交互项
- 6 个基础特征二次多项式 = 6 + 6 + 15 = 27 个特征
- 全部交互会显著增加维度 → 需要 L1 正则化（Lasso）或特征选择

#### 方向二：目标编码（Target Encoding）

- 对高基数类别特征（如 Topography.Code 374 个类别）做目标编码
- 用每个类别对应的目标均值替换原始类别
- 优点：将高基数类别压缩为单维数值
- 陷阱：需交叉验证防止目标泄漏

#### 方向三：领域特定的医学特征

```
Charlson 合并症指数:  根据患者合并症评分的加权指数
                       → 预测生存率的经典医学指标

年龄调整:              Age_Adjusted = (Age - Population_Median) / Population_IQR
                       → 消除年龄分布的种群差异

诊断窗口期:            Diagnosis_to_Treatment_Delay
                       → 诊断到治疗的时间间隔（需要日期数据）
```

#### 方向四：PCA / 自动特征提取

- 对高基数类别特征的 One-Hot 编码做 PCA
- 使用自编码器（Autoencoder）提取非线性特征
- 使用特征重要性引导的特征选择

## 四、总结与思考题

### 4.1 核心收获

1. **标准化不改变模型性能，但改变模型效率**：174 次迭代 → 10 次迭代，速度提升约 17 倍
2. **标准化后系数可比较**：未标准化时，Code.Profession 的系数 -0.001（看似不重要）其实是量纲假象
3. **StandardScaler vs RobustScaler**：数据有离群值时选 RobustScaler，否则选 StandardScaler
4. **特征构造 = 编码领域知识**：Age\_Group、Age\_Sq、Gender\_x\_AgeGroup 的意义来自医学认知，而非数据本身
5. **特征质量决定模型上限**：即使简单的逻辑回归，好的特征也能带来性能提升
6. **多指标评估**：特征构造提升了 AUC 和 Brier，但 Recall 下降——没有完美的特征集，需要权衡取舍

### 4.2 思考题

> 以下问题可作为课后思考或课堂讨论：

1. **（基础）** 本实验中 Raw 和 StandardScaler 的 AUC 完全相同。这是巧合还是必然？如果改用树模型（如随机森林），标准化还有影响吗？
2. **（进阶）** StandardScaler 耗时 0.021s（10 次迭代），Raw 耗时 0.257s（174 次迭代）。为什么迭代次数相差 17 倍但耗时只相差 12 倍？（提示：考虑 sklearn 的 solver 实现）
3. **（进阶）** RobustScaler 使用了 IQR，对于偏度为 0.94 的 Code.Profession，RobustScaler 和 StandardScaler 哪个更合适？为什么？
4. **（拓展）** 特征构造后 Recall 从 0.9026 下降到 0.8886。你能解释这个现象吗？如果要改善 Recall，你会怎么做？
5. **（拓展）** 如果让你在 Age\_Group（分箱）和 Age\_Sq（平方项）中只选一个加入模型，你会选哪个？为什么？（提示：考虑它们捕获的信息是否重叠）
6. **（开放）** "特征质量决定模型上限"这句话中，"上限"是指什么？如果特征质量很低，用什么方法可以弥补？
7. **（实践）** 在本实验的基础上，尝试添加一个新的构造特征（如 Age × year 交互），观察它对 AUC、Recall、Brier 的影响。你的新特征是否改进了模型？
