# 案例教程 3：数据预处理与缺失值插补

## 一、前序知识要求

### 1.1 机器学习基础

| 概念                            | 简要说明                       | 与本教程的关系                                       |
| ----------------------------- | -------------------------- | --------------------------------------------- |
| **逻辑回归（Logistic Regression）** | 广义线性模型，输出概率值，适用于二分类        | 本教程的评估模型                                      |
| **训练集 / 测试集划分**               | 将数据分为训练和评估两部分，防止信息泄露       | 插补必须在训练集上拟合，再变换测试集                            |
| **AUC（ROC 曲线下面积）**            | 衡量模型区分正负类能力的指标，不依赖阈值       | 评估插补方法对区分能力的影响                                |
| **Recall（召回率）**               | 正类中被正确识别的比例，TP / (TP + FN) | 评估模型对 VIVO（少数类）的捕捉能力                          |
| **校准度（Calibration）**          | 预测概率与真实概率的一致性              | 评估模型概率输出的可靠性                                  |
| **Brier Score**               | 预测概率与真实标签的均方误差，越低越好        | 综合衡量区分力和校准度                                   |
| **类别不平衡（Class Imbalance）**    | 正负类样本数量不均衡                 | 本数据 VIVO 约 41%，轻度不平衡，设置 class\_weight='balanced' |

### 1.2 数据处理基础

| 知识                       | 说明                                   |
| ------------------------ | ------------------------------------ |
| **标准化（Standardization）** | 减去均值除以标准差，使特征服从 N(0,1)               |
| **Label Encoding**       | 将分类变量的每个类别映射为一个整数                    |
| **缺失值基础**                | MCAR / MAR / MNAR 三种缺失机制（案例教程 1 已介绍） |
| **过拟合 vs 欠拟合**           | 模型在训练集表现好但在测试集表现差 vs 两者都差            |

### 1.3 前序教程回顾

- EDA 阶段发现了 **30/38 列存在缺失**，18 列缺失 ≥ 50%
- 统计检验阶段发现所有特征均与目标显著相关（但效应量差异巨大）
- 已理解"统计显著≠预测力强"的核心论点

## 二、本次实践内容与目的

### 2.1 核心目的

> **理解数据质量对模型性能的影响——不同的缺失值处理方法如何改变模型的表现，以及为什么。**

这不是一个关于"如何填补缺失值"的纯技术教学，而是一个**因果链**的完整演示：

```
缺失值 → 插补方式 → 特征分布改变 → 模型决策边界改变 → AUC/Recall/校准度改变
```

### 2.2 实践内容全景

| 模块            | 方法               | 核心思想             | 教学价值                |
| ------------- | ---------------- | ---------------- | ------------------- |
| **完整案例分析**    | 删除所有含缺失的行        | 最简单的"方案"，只保留完整记录 | 作为基线对照，展示样本量损失的影响   |
| **均值插补**      | 用均值（数值）或众数（分类）填充 | 最简单的插补，快速但粗糙     | 展示"低估方差"的具体含义       |
| **KNN 插补**    | 用 k 个最近邻的特征均值填充  | 利用相似患者信息 "猜" 缺失值 | 展示局部结构的保留与计算代价的权衡   |
| **MICE 多重插补** | 用其他特征迭代预测缺失值     | 利用变量间的联合分布       | 医学研究的黄金标准，展示"严谨"的价值 |

### 2.3 评价指标体系

本教程使用三个维度评估插补方法对模型的影响，而不是孤立地看某一个指标：

```
                     ┌─────────────────────────┐
                     │  插补方法对比评价体系    │
                     └────────────┬────────────┘
                                  │
            ┌─────────────────────┼─────────────────────┐
            ▼                     ▼                     ▼
    ┌───────────────┐    ┌───────────────┐    ┌───────────────┐
    │  区分能力     │    │  少数类捕捉   │    │  概率校准     │
    │  (Discrimination) │  (Sensitivity)  │    │  (Calibration) │
    ├───────────────┤    ├───────────────┤    ├───────────────┤
    │  AUC (ROC)    │    │  Recall       │    │  Brier Score  │
    │  PR-AUC       │    │  (VIVO 类)    │    │  校准曲线     │
    └───────────────┘    └───────────────┘    └───────────────┘
            │                     │                     │
            ▼                     ▼                     ▼
    模型能否区分       模型能否找到          模型预测的概率
    存活与死亡？       真正的存活患者？       是否可靠？
```

**为什么需要三个维度？**

| 单一指标的局限          | 补充说明                      |
| ---------------- | ------------------------- |
| AUC 高 ≠ 校准好      | 模型可能排序正确但概率偏移（如所有预测概率偏高）  |
| Recall 高 ≠ AUC 高 | 可能通过把所有人都预测为正类来获得高召回      |
| 校准好 ≠ 区分好        | 概率准确但无法区分（如所有样本预测为 41%）  |

### 2.4 关键发现摘要

| 方法                  | AUC        | Recall     | Brier      | 训练样本   | 耗时        | 核心发现                       |
| ------------------- | ---------- | ---------- | ---------- | ------ | --------- | -------------------------- |
| **Complete Case**   | **0.8644** | **0.9240** | 0.1493     | 47,270 | 0.1s      | 损失仅 16% 数据，AUC/Recall 略优    |
| **Mean Imputation** | 0.8632     | 0.9236     | 0.1466     | 56,000 | 0.1s      | Brier 优于 Complete Case       |
| **KNN Imputation**  | 0.8628     | 0.9237     | 0.1467     | 56,000 | **20.5s** | 耗时是其他方法的 200+ 倍，性能无优势   |
| **MICE Imputation** | 0.8637     | 0.9223     | **0.1462** | 56,000 | 0.1s      | Brier 最优（校准度最好）           |

> **关键洞察**：四种方法的性能差异极小（AUC 差距 < 0.002），Complete Case 在 AUC 和 Recall 上反而略优，而插补方法在 Brier Score（校准度）上稍好。这告诉我们：**当缺失率较低时（本例最高仅 15%），插补方法的选择对模型性能影响有限**。但随着缺失率升高，保留样本量的优势会逐渐显现。

## 三、扩展内容

### 3.1 均值插补为什么"不好"？—— 低估方差的本质

均值插补是"最危险"的插补方法——它看似合理（用数据自身的均值填充），但实际上**人为地改变了数据的统计特性**：

```
原始分布（Age）:  μ=63.85, σ²=274.41
               
均值插补后:       μ=63.85, σ²=273.99 (方差略微缩小)
                
问题: 所有缺失值都被替换为同一个值 (63.85)
      → 均值不变，但方差被压缩
      → 破坏了特征之间的协方差结构
      → 回归系数被向零压缩（attenuation bias）
```

**教学直观展示**：在生成的 `06e_distribution_impact.png` 图中，可以看到：

- Mean Imputation 在均值点（\~64 岁）上有一个异常高峰——所有缺失的 Age 都被填为均值
- KNN 和 MICE 的分布更接近原始数据的自然形态
- Mean Imputation 的方差最小（条形图最明显）

### 3.2 KNN 插补：为什么这么慢？

| 步骤   | 计算复杂度            | 说明                               |
| ---- | ---------------- | -------------------------------- |
| 距离计算 | O(n² × d)        | 每个缺失样本需要与所有完整样本计算距离              |
| 排序选取 | O(n × k × log n) | 对每个缺失样本的 n 个距离排序                 |
| 总复杂度 | O(n² × d)        | 本实验 n=56,000, d=5 → 约 157 亿次距离计算 |

**KNN 插补的时间复杂度与样本量成平方关系**，这意味着：

- 10,000 样本 → \~0.7s
- 56,000 样本 → \~20.5s（本实验）
- 500,000 样本 → \~27 分钟
- 1,780,000 样本 → \~5.7 小时（几乎不可行）

**优化方向**（可作为拓展内容）：

- 使用 KD-Tree / Ball-Tree 加速最近邻搜索
- 不计算所有距离，只搜索候选集
- 先聚类再插补

### 3.3 MICE 的"魔力"来自哪里？

MICE（Multiple Imputation by Chained Equations）的核心思想是：

```
1. 先用均值/中位数初始化所有缺失值
2. 对每个有缺失的变量：
   a. 用其他变量作为特征，该变量作为目标
   b. 建立回归模型（线性回归、决策树等）
   c. 用模型预测缺失值
3. 重复步骤 2 多次（通常 5-10 轮）
4. 最终收敛到一个稳定的插补值
```

**MICE 的优势来源**：

| 对比   | 均值插补     | MICE            |
| ---- | -------- | --------------- |
| 信息利用 | 只用了该列的均值 | 利用了所有其他列的信息     |
| 方差估计 | 低估方差     | 保留自然方差          |
| 变量关系 | 破坏协方差结构  | 保留变量间关系         |
| 不确定性 | 视为已知值    | 可生成多个插补版本反映不确定性 |

**为什么医学研究偏爱 MICE？**

- 医学数据缺失常见且多为 MAR
- MICE 在 MAR 假设下给出无偏估计
- 多重插补可以在后续分析中反映插补的不确定性
- 临床研究中结果的稳健性比计算效率更重要

### 3.4 其他缺失值处理方法（扩展）

| 方法                      | 描述                      | 适用场景      | 优缺点               |
| ----------------------- | ----------------------- | --------- | ----------------- |
| **中位数插补**               | 用中位数代替均值                | 偏态分布      | 比均值更稳健，但仍低估方差     |
| **众数插补**                | 用出现最频繁的值填充              | 分类变量      | 简单粗暴，可能导致类别偏倚     |
| **回归插补**                | 用线性回归预测缺失值              | 变量间相关性较强  | 单步回归，不如 MICE 迭代精细 |
| **随机森林插补** (missForest) | 用随机森林迭代预测               | 非线性关系     | 非参数，可处理混合类型，但计算量大 |
| **热卡插补（Hot Deck）**      | 从完整样本中随机匹配              | 简单且保留分布   | 随机性导致结果不稳定        |
| **指示变量法**               | 加上"IsMissing"标志         | 缺失本身可能有信息 | 增加维度，可能导致过拟合      |
| **不插补**（树模型原生支持）        | XGBoost/LightGBM 可以处理缺失 | 树模型场景     | 避免插补偏差，但不适用于逻辑回归  |

### 3.5 扩展到其他预处理技术

> 本教程聚焦在缺失值插补，但数据预处理远不止于此。

### 3.6 深入讨论：插补如何影响模型性能？

**路径 1：样本量效应**

```
Complete Case:   47,270 样本 → 仅损失 16% → 信息量充足 → AUC/Recall 略优
插补保留样本:    56,000 样本 → 保留全部样本 → 校准度（Brier）略优
```

在本实验中，缺失率较低（最高 15%），Complete Case 仅损失少量样本，四种方法性能差异极小。**缺失率低时，方法选择影响有限**。

**路径 2：分布失真效应**

```
均值插补:  缺失值 → 全部替换为均值 → 均值点产生"假峰" → 方差压缩
         → 逻辑回归的决策 boundary 偏向均值方向 → 轻微偏差

KNN插补:  缺失值 → 用相似样本填充 → 保留局部分布 → 更自然的分布
         → 决策边界更接近真实边界 → 理论上应该更好

MICE插补: 缺失值 → 用联合分布预测 → 保留变量关系 → 最自然的分布
         → 决策边界最接近真实边界 → 理论上最优
```

**路径 3：校准度效应**

校准度对插补方法的敏感度高于 AUC，因为：

- AUC 只关心排序（正类概率是否高于负类）
- 校准度关心**概率值的绝对大小**（预测的 0.8 是否对应 80% 的真实概率）
- 分布失真会直接破坏概率的校准性

## 四、总结与思考题

### 4.1 核心收获

1. **缺失率低时方法差异小**：本例缺失率最高仅 15%，Complete Case 仅损失 16% 样本，四种方法性能差异极小
2. **均值插补压缩方差**：看似"最安全"的方法，实际改变了数据的统计特性
3. **KNN 插补理论上优但计算代价巨大**：在实际大规模数据中需权衡
4. **MICE 在校准度上最优**：Brier Score 最低，但 AUC/Recall 略低于 Complete Case
5. **模型性能的三个维度**：AUC（区分能力）、Recall（少数类捕捉）、校准度（概率可靠性），三者缺一不可
6. **插补方式通过"样本量"和"分布失真"两条路径影响模型**

### 4.2 思考题

> 以下问题可作为课后思考或课堂讨论：

1. **（基础）** 本实验中，Complete Case 的 AUC 反而最高（0.8644），且仅损失 16% 的样本。如果数据缺失率高达 50%，Complete Case 还能保持优势吗？为什么？
2. **（进阶）** 均值插补后 Age 的方差从 274.41 降到了 273.99。这种方差缩小对逻辑回归的系数有什么具体影响？（提示：考虑回归系数的标准误）
3. **（进阶）** KNN 插补花费了 20.5 秒，但 AUC 并没有显著优于 Mean 或 MICE。在什么情况下 KNN 的优势会变得更加明显？
4. **（拓展）** 本实验对分类变量使用了 Label Encoding。如果改用 One-Hot Encoding，插补结果会不同吗？为什么？
5. **（拓展）** 如果某个特征的缺失率是 80%，而且缺失与目标变量高度相关（MNAR 机制），你会如何处理？三种插补方法在这种场景下谁更可靠？
6. **（开放）** 本实验中三种插补方法的 AUC 差距在 0.005 以内。在什么情况下这个差距会显著扩大？请设计一个实验来验证你的猜想。
7. **（实践）** 试试在本实验的基础上增加一个特征（如 Extension、State.Civil），观察插补方法的性能变化。新增特征对哪种插补方法影响最大？
