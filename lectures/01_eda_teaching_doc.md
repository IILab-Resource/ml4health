### 1.2 统计学基础概念

| 概念               | 简要说明                                         |
| ---------------- | -------------------------------------------- |
| **均值 vs 中位数**    | 均值对极端值敏感，中位数更稳健，两者差异可反映分布偏态                  |
| **标准差与方差**       | 衡量数据的离散程度                                    |
| **偏度（Skewness）** | 衡量分布不对称程度：正偏（右偏）→ 右侧有长尾，负偏（左偏）→ 左侧有长尾        |
| **峰度（Kurtosis）** | 衡量分布"尾部"的厚度，峰度高 → 极端值更多                      |
| **四分位数与 IQR**    | Q1（25%）、Q3（75%）、IQR = Q3 - Q1，用于衡量数据散布和离群值检测 |
| **正态分布**         | 钟形曲线，均值和标准差完全决定其形态，是许多统计方法的前提假设              |
| **缺失机制**         | MCAR（完全随机缺失）、MAR（条件随机缺失）、MNAR（非随机缺失）         |

## 二、本次实践内容与目的

### 2.1 核心目的

> **从零开始对一个真实的医学数据集进行系统的探索性数据分析（EDA），建立"数据驱动"的思维方式。**

在机器学习项目中，EDA 既不是第一步也不是可选项——**它就是第一步，而且是不可跳过的一步**。没有高质量的 EDA，后续的建模就是"盲人摸象"。

### 2.2 实践内容全景

| 模块        | 分析内容                   | 核心问题                 | 教学价值               |
| --------- | ---------------------- | -------------------- | ------------------ |
| **数据概况**  | 样本量、特征数、标签分布、平衡性       | 数据长什么样？目标变量分布是否均衡？   | 建立数据全局观            |
| **缺失值分析** | 缺失比例、缺失模式、缺失机制讨论       | 数据是否完整？缺失是随机的还是有规律的？ | 理解真实数据的"不完美"       |
| **分布分析**  | 均值/中位数/偏度、直方图+KDE、Q-Q图 | 每个数值特征服从什么分布？        | 为后续特征工程和模型选择提供依据   |
| **离群值分析** | IQR 方法、Z-score 方法、对比讨论 | 数据中是否存在异常值？如何检测？     | 理解离群值的不同检测策略及其适用场景 |

### 2.3 数据集背景

- **来源**：巴西坎皮纳斯市癌症登记中心（RCBP CAMPINAS-UNICAMP）
- **规模**：原始约 **178 万条** 记录，38 个原始字段（含人口学信息、诊断信息、肿瘤特征、生存状态等）
- **目标变量**：`Status.Vital` — VIVO（存活）= 1，MORTO（死亡）= 0
- **特点**：大规模、高缺失率、包含丰富的临床信息

> **⚠️ 关于目标变量 `Status.Vital` 缺失值的处理**
>
> `Status.Vital` 列缺失率高达 **88.20%**（1,568,418 条记录无明确生死标签）。在癌症登记数据库中，目标变量出现如此高缺失率的常见原因包括：
>
> 1. **患者刚入库，尚未到随访时间** → Status = Missing
> 2. **患者失访**（Lost to follow-up）
> 3. **数据采集不完整**
> 4. **该变量本来只对部分患者记录**
>
> 因此，这些缺失**并非随机缺失（非 MCAR）**。如果直接将缺失值归为"死亡"（即 `NaN == 'VIVO'` 为 `False` → target=0），会严重低估真实存活率。
>
> **处理方式**：在加载数据后、创建目标变量前，先删除 `Status.Vital` 缺失的样本，仅保留有明确生死标签的 **209,758 条** 记录（删除率 88.20%）。后续所有 EDA 分析均基于这 21 万条有标签数据。

### 2.4 关键发现摘要

| 维度       | 核心发现                                                |
| -------- | --------------------------------------------------- |
| **平衡性**  | 轻度不平衡（存活:死亡 ≈ 41:59），存活 40.95%，死亡 59.05%             |
| **缺失值**  | 24/39 列存在缺失，10 列缺失 ≥ 50%；临床分期字段（TNM 77%、Extension 61%）缺失率仍偏高 |
| **分布形态** | Age 轻度左偏（偏度 -0.68），Code.of.Morphology 显著右偏（偏度 2.63）  |
| **离群值**  | Age 离群占 1.44%（IQR 界限 [21, 109]），医学上属合理范围；Code.Profession 无 IQR 离群 |

### 3.2 数值特征 vs 类别特征：如何区分和处理？

#### 为什么要区分？

在本次数据集中，`pandas` 的 `select_dtypes(include=[np.number])` 只识别出 4 个数值列（Age、Code.Profession、Code.of.Morphology、year）。但 **"数值"不等于"数值型特征"**：

| 类型         | 定义              | 本数据集示例                    | 应如何处理                       |
| ---------- | --------------- | ------------------------- | --------------------------- |
| **连续数值型**  | 可无限细分、有实际数学意义   | Age（年龄）                   | 描述统计、分布拟合、标准化               |
| **离散数值型**  | 整数计数、可枚举但数量大    | Code.Profession（职业编码）     | 可视为数值或类别，取决于语义              |
| **有序类别型**  | 有天然顺序但数值本身无数学意义 | TNM 分期（I、II、III、IV）       | 标签编码或保留有序性                  |
| **名义类别型**  | 无顺序的分类          | Gender（性别）、Raca.Color（肤色） | One-Hot 编码或 Target Encoding |
| **日期/时间型** | 时间戳             | Date.of.Diagnostic        | 提取年/月/季/周作为新特征              |
| **ID 型**   | 唯一标识            | Patient.Code              | 排除，不参与建模                    |

> **关键洞察**：Code.Profession 和 Code.of.Morphology 虽然是数值编码，但本质上是**类别变量**。直接对其计算均值、标准差没有实际意义——这是 EDA 中需要特别留意的"数值陷阱"。

#### 实操建议

在 EDA 阶段，应该**手动分类**每个特征的数据类型：

```python
numerical_features = ['Age', 'year']  # 真正的连续/离散数值
categorical_features = ['Gender', 'Raca.Color', 'Degree.of.Education',
                        'State.Civil', 'Topography.Code', 'Code.of.Morphology',
                        'Diagnostic.means', 'Extension', 'Laterality', 'TNM']
datetime_features = ['Date.of.Birth', 'Date.of.Diagnostic', 
                     'Date.of.Death', 'Date.of.Last.Contact']
id_features = ['Patient.Code']
```

然后对每种类型采用**不同的分析方法**和**不同的可视化策略**。

### 3.3 特征与特征之间的交互分析

#### 为什么需要交互分析？

单个特征的分析（单变量分析）只能告诉我们"这个特征自己长什么样"，但无法回答：

- **高龄 + 特定癌症类型 → 存活率是否更低？**（Age × Topography × target）
- **女性乳腺癌的发现时期是否比男性更早？**（Gender × Extension × Topography）
- **教育程度高的人群是否诊断时分期更早？**（Education × TNM stage）

#### 交互分析的方法矩阵

| 变量类型组合       | 推荐分析方法              | 对应图表                |
| ------------ | ------------------- | ------------------- |
| 数值 × 数值      | 相关性、协方差             | 散点图、Pairplot、相关性热力图 |
| 数值 × 类别      | 分组统计（均值/中位数对比）      | 分组箱线图、小提琴图、分面直方图    |
| 类别 × 类别      | 列联表、卡方检验、Cramér's V | 堆叠柱状图、马赛克图、热力图      |
| 数值 × 类别 × 类别 | 三维交互分析              | 分面箱线图、条件散点图         |

#### 本数据集值得深入探索的交互关系

1. **Age × Gender × target**：不同年龄段的男女存活率差异
2. **Topography × Morphology × Extension**：不同癌症类型在诊断时的扩散程度
3. **Year × Diagnostic.means × target**：随年份变化，诊断方式与存活率的关系
4. **Raca.Color × Degree.of.Education × TNM**：种族和教育程度是否影响诊断时的分期

### 3.4 更高级的 EDA 方向（进阶参考）

> 以下方向可作为后续案例教程的独立选题，或在当前教程中作为头脑风暴素材。

#### 方向一：类别特征的详细分析

- 计算每个类别特征中各类别的频数和占比
- 对低频类别进行合并建议
- 计算类别特征与目标变量的关联强度（Cramér's V、Theil's U）

#### 方向二：时间序列分析视角

- 新增病例数随年份的变化趋势
- 存活率的年度变化
- 从诊断到死亡/最后接触的时间间隔分析

#### 方向三：多维缺失模式诊断

- 使用 `missingno` 库生成缺失值矩阵的层次聚类
- 构建"缺失指示变量"，分析缺失是否与目标变量相关
- 评估不同插补策略的适用性

#### 方向四：数据质量报告自动化

- 使用 `ydata-profiling`（原 pandas-profiling）一键生成完整 EDA 报告
- 使用 `Great Expectations` 构建数据质量断言

#### 方向五：面向建模的 EDA

- 特征与目标变量的单变量关联筛选
- 多重共线性诊断（VIF 计算）
- 特征交互项的建议
- 初步的特征重要性排序（使用随机森林或 LightGBM）

### 3.5 本次 EDA 的"盲区"

| 盲区              | 说明                               | 如何弥补                           |
| --------------- | -------------------------------- | ------------------------------ |
| **类别特征几乎未分析**   | 38 列中仅 4 列是数值型，其余大部分是有价值的类别信息    | 单独做一轮类别特征 EDA                  |
| **未做目标变量分组对比**  | 除 Age 外，其他特征未按 VIVO/MORTO 做差异化分析 | 对每个特征做 groupby(target) 的统计对比   |
| **未考虑特征之间的相关性** | 只做了缺失值的相关性，未做特征值的相关性             | 针对数值特征计算 Pearson/Spearman 相关矩阵 |
| **未探索时间维度**     | year 列仅做了分布分析，未做趋势分析             | 按年份聚合存活率，绘制趋势折线图               |
| **未分群体子集分析**    | 整体 EDA 可能掩盖子群体的特殊模式              | 按 Gender、癌症类型等分群后分别做 EDA       |

## 四、总结与思考题

### 4.1 核心收获

1. **EDA 的本质是"提问-回答"的迭代过程**，不是机械地执行固定函数
2. **"数据质量"比"数据量"更重要**——原始 178 万条记录中目标变量缺失 88%，过滤后仅 21 万条可用
3. **领域知识不可或缺**——不理解 TNM 分期、不理解 Morphology Code 的含义，就无法正确判断离群值是否为错误数据
4. **EDA 要为下游任务服务**——发现轻度不平衡 → 后续可视情况重采样；发现严重缺失 → 后续需处理缺失值

### 4.2 思考题

> 以下问题可作为课堂讨论或课后作业：

1. 本数据集原始 178 万条记录中 `Status.Vital` 缺失 88%，过滤后存活率为 40.95%。这个存活率是否能代表巴西癌症患者的真实生存率？过滤缺失样本是否会引入选择偏差（survivorship bias）？
2. TNM 分期的缺失率仍高达 77.15%，有什么办法可以补充或替代这一关键信息？
3. IQR 方法检测到 Age 的离群值下限为 21 岁——这意味着 21 岁以下的癌症患者被标记为"离群值"。这合理吗？说明了什么？
4. 对比过滤前后 Code.Profession 的离群值情况：过滤前 IQR=0 导致 20.46% 被标记为离群值，过滤后 IQR=513、离群值为 0%。为什么过滤目标变量缺失样本后，IQR 方法的结果会发生如此大的变化？这说明了什么？
5. 如果让你增加一页 EDA 分析内容，你会选择分析什么？为什么？
