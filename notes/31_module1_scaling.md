## 二、均值插补：处理缺失值

### 2.1 完整代码

```python
# 用均值插补处理缺失
imputer = SimpleImputer(strategy='mean')
X_raw_arr = imputer.fit_transform(df_feat[base_features])
```

### 2.2 `SimpleImputer(strategy='mean')` 是什么？

`SimpleImputer` 是 Scikit-learn 提供的**单变量插补器**，用于把缺失值（`NaN`）填充为某个统计量。`strategy='mean'` 表示用**该列的均值**填充缺失值。

| `strategy` 值      | 填充方式                  | 适用数据类型  |
| ----------------- | --------------------- | ------- |
| `'mean'`          | 列均值                   | 数值型     |
| `'median'`        | 列中位数                  | 数值型     |
| `'most_frequent'` | 众数                    | 数值型或类别型 |
| `'constant'`      | 固定值（需指定 `fill_value`） | 任意      |

### 2.3 为什么本模块只用均值插补？

> 📌 **回顾案例 3**：在案例教程 3 中，我们详细对比了 Complete Case、Mean、KNN、MICE 四种插补策略，发现它们的最终性能差异很小。本模块的主题是**标准化**，而不是插补。为了不让插补方法的差异干扰标准化比较，我们统一采用均值插补作为"中性"的预处理步骤，把注意力集中在标准化效果上。

### 2.4 `fit_transform` 一步到位

`imputer.fit_transform(df_feat[base_features])` 做了两件事：

1. **`fit`**：计算每列的均值，存入 `imputer.statistics_`。
2. **`transform`**：把 `NaN` 替换为对应列的均值。

返回值 `X_raw_arr` 是一个 NumPy 数组（不再是 DataFrame），形状为 `(80000, 6)`——80,000 条样本，6 个基础特征（Age、year、Gender、Code.Profession、Diagnostic.means、Raca.Color）。

> ⚠️ **常见问题**：为什么这里对**全量数据**做 `fit_transform`，而不是先划分再插补？
> 严格来说，最佳实践是先划分训练/测试集，再在训练集上 `fit`、在测试集上 `transform`，以避免测试集信息泄露到插补统计量中。本模块为了简化代码、聚焦标准化主题，对全量数据做了插补。在实际生产环境中，建议遵循"先划分后插补"的原则。

## 三、分层划分训练/测试集

### 3.1 完整代码

```python
y = df_feat['target'].values
X_train_raw, X_test_raw, y_train, y_test = train_test_split(
    X_raw_arr, y, test_size=0.3, random_state=RANDOM_STATE, stratify=y
)
```

### 3.2 参数逐个详解

| 参数                          | 值      | 含义                        |
| --------------------------- | ------ | ------------------------- |
| `X_raw_arr`                 | 特征数组   | 待划分的特征                    |
| `y`                         | 目标标签   | 用于分层和作为标签                 |
| `test_size=0.3`             | 0.3    | 30% 作为测试集，70% 作为训练集       |
| `random_state=RANDOM_STATE` | 固定种子   | 保证划分可复现                   |
| `stratify=y`                | 按 y 分层 | **关键参数**，保证训练集和测试集中类别比例一致 |

### 3.3 `stratify=y` 为什么重要？

> 📌 **回顾案例 3**：我们的目标变量 `target` 是不平衡的——VIVO（存活，1）约 41%，MORTO（死亡，0）约 59%。如果用普通随机划分，可能碰巧训练集里 VIVO 占 45%、测试集里 VIVO 占 35%，导致评估偏差。

`stratify=y` 强制让训练集和测试集**保持与原始数据相同的类别比例**：

| 数据集       | 总样本数   | VIVO (1) 比例 | MORTO (0) 比例 |
| --------- | ------ | ----------- | ------------ |
| 原始数据      | 80,000 | \~41%       | \~59%        |
| 训练集 (70%) | 56,000 | \~41%       | \~59%        |
| 测试集 (30%) | 24,000 | \~41%       | \~59%        |

> 💡 **小贴士**：在类别不平衡的分类任务中，**永远使用** **`stratify=y`**。这是防止评估偏差的第一道防线。

## 四、定义标准化方法字典

### 4.1 完整代码

```python
# 定义标准化方法
scalers = {
    'Raw (No Scaling)': None,
    'StandardScaler': StandardScaler(),
    'RobustScaler': RobustScaler(),
}
```

### 4.2 字典结构详解

`scalers` 是一个**简单的字符串→对象字典**：

| 键（方法名）               | 值（标准化器）            | 含义                     |
| -------------------- | ------------------ | ---------------------- |
| `'Raw (No Scaling)'` | `None`             | 不做任何标准化，作为基线（baseline） |
| `'StandardScaler'`   | `StandardScaler()` | Z-score 标准化器实例         |
| `'RobustScaler'`     | `RobustScaler()`   | 稳健标准化器实例               |

### 4.3 为什么用字典组织？

> 📌 **回顾案例 3**：在案例教程 3 中，我们用字典组织了四种插补方法。这里采用相同的"配置与逻辑分离"模式。

用字典组织的好处：

1. **代码简洁**：一个 `for name, scaler in scalers.items():` 循环就能遍历所有方法。
2. **易于扩展**：想加 `MinMaxScaler`、`MaxAbsScaler`，只需在字典里加一行。
3. **结果可追溯**：方法名作为字典的键，会自动出现在结果表格和图表标签中。
4. **统一处理**：循环体内的"标准化 → 训练 → 评估"逻辑只写一遍，避免重复。

### 4.4 `None` 的妙用

`'Raw (No Scaling)': None` 把"不做标准化"也纳入了统一循环。在循环体内，我们用 `if scaler is not None:` 判断是否需要标准化：

```python
if scaler is not None:
    X_train = scaler.fit_transform(X_train_raw)
    X_test = scaler.transform(X_test_raw)
else:
    X_train = X_train_raw.copy()
    X_test = X_test_raw.copy()
```

<br />

> 💡 **小贴士**：注意 `else` 分支用了 `.copy()`。这是因为后续如果对 `X_train` 做原地修改，不会污染原始的 `X_train_raw`（在 NumPy 中，赋值只是引用，不复制数据）。

## 五、三种标准化方法的数学原理（🌟 本模块重点）

这是本模块的核心内容。理解三种方法的数学原理，是解读后续实验结果的基础。

### 5.1 Raw（不做标准化）

#### 公式

$$x' = x$$

即**不做任何变换**，保留原始数值。

#### 输出特性

- 均值：原始均值
- 标准差：原始标准差
- 范围：原始范围

#### 我们的数据长什么样？

我们的 6 个基础特征的量纲差异巨大：

| 特征                 | 典型范围        | 量级    |
| ------------------ | ----------- | ----- |
| `Age`              | 0 – 120     | \~10² |
| `year`             | 2000 – 2023 | \~10³ |
| `Gender`           | 0 / 1       | \~10⁰ |
| `Code.Profession`  | 0 – \~50    | \~10¹ |
| `Diagnostic.means` | 0 – \~10    | \~10¹ |
| `Raca.Color`       | 0 – \~5     | \~10⁰ |

`year` 的数值在 2000+，而 `Gender` 只有 0 和 1，**量纲差异高达 3 个数量级**。这种情况下，逻辑回归的优化会非常困难。

### 5.2 StandardScaler（Z-score 标准化）

#### 公式

$$x' = \frac{x - \mu}{\sigma}$$

其中：

- $\mu$ 是该列的**均值**（mean）
- $\sigma$ 是该列的**标准差**（standard deviation）

#### 输出特性

- 均值：**0**
- 标准差：**1**
- 范围：通常在 \[-3, 3] 之间（如果数据近似正态分布）

#### 直观理解

Z-score 回答的问题是："这个值距离均值有几个标准差？"

- $x' = 0$：恰好等于均值
- $x' = 1$：比均值高 1 个标准差
- $x' = -2$：比均值低 2 个标准差

#### 缺点：受离群值影响

> ⚠️ **关键缺点**：$\mu$ 和 $\sigma$ 都是基于**所有数据点**计算的统计量，**对离群值非常敏感**。
>
> **举例**：假设某列有 99 个值在 \[0, 10] 之间，1 个离群值为 1000。
>
> - 均值 $\mu$ 会被这个 1000 大幅拉高（从 \~5 变成 \~15）
> - 标准差 $\sigma$ 会被大幅放大（从 \~3 变成 \~100）
> - 结果：99 个正常值的 Z-score 都被压缩到 \[-0.15, -0.05] 这个很窄的区间，区分度大幅下降

这就是为什么需要 RobustScaler。

### 5.3 RobustScaler（稳健标准化）

#### 公式

$$x' = \frac{x - \text{median}}{\text{IQR}}$$

其中：

- $\text{median}$ 是该列的**中位数**（median）
- $\text{IQR} = Q\_3 - Q\_1$ 是**四分位距**（Interquartile Range）
  - $Q\_1$：第 25 百分位数（下四分位数）
  - $Q\_3$：第 75 百分位数（上四分位数）

#### 输出特性

- 中位数：**0**
- IQR：**1**
- 范围：通常比 StandardScaler 更分散（因为 IQR 通常比 σ 小）

#### "稳健"二字的含义

> 🌟 **核心优点**：中位数和 IQR 都是**基于排序的统计量**，**不受离群值影响**。
>
> **为什么中位数稳健？**
> 中位数是"中间值"，无论最大值是 100 还是 1,000,000，中位数都不变——因为它只关心"排在中间的那个值"。
>
> **为什么 IQR 稳健？**
> $Q\_1$ 和 $Q\_3$ 分别是第 25 和第 75 百分位数，它们也只关心"排在 25% 和 75% 位置的值"，与极端值无关。
>
> **对比**：均值和标准差是"基于求和"的统计量，一个极端值就能让它们剧烈变化；中位数和 IQR 是"基于排序"的统计量，极端值对它们几乎无影响。

#### 直观理解

RobustScaler 回答的问题是："这个值在中位数之上还是之下？偏离了多少个四分位距？"

- $x' = 0$：恰好等于中位数
- $x' = 1$：比中位数高 1 个 IQR（约等于上四分位）
- $x' = -1$：比中位数低 1 个 IQR（约等于下四分位）

### 5.4 三种方法对比表

| 特性         | Raw      | StandardScaler                | RobustScaler                                |
| ---------- | -------- | ----------------------------- | ------------------------------------------- |
| **公式**     | $x' = x$ | $x' = \frac{x - \mu}{\sigma}$ | $x' = \frac{x - \text{median}}{\text{IQR}}$ |
| **中心**     | 无        | 均值 → 0                        | 中位数 → 0                                     |
| **尺度**     | 原始       | 标准差 → 1                       | IQR → 1                                     |
| **受离群值影响** | N/A      | **是**（严重）                     | **否**（稳健）                                   |
| **输出范围**   | 原始       | 通常 \[-3, 3]                   | 通常比 StandardScaler 更分散                      |
| **保留分布形状** | 是        | 是                             | 是                                           |
| **适用场景**   | 树模型      | 一般线性模型、神经网络                   | 含离群值的数据                                     |
| **计算成本**   | 最低       | 低（需算 μ, σ）                    | 低（需排序算中位数、IQR）                              |

> 💡 **小贴士**：StandardScaler 和 RobustScaler 都**不改变分布的形状**，只是平移和缩放。如果原始分布是右偏的，标准化后仍然是右偏的。要改变分布形状，需要用 `PowerTransformer`（如 Yeo-Johnson）或 `QuantileTransformer`。

### 5.5 为什么标准化能加速梯度下降？

这是本模块最核心的直觉。考虑一个二维特征空间 $(x\_1, x\_2)$，其中 $x\_1$ 的量级远大于 $x\_2$（比如 $x\_1 \in \[2000, 2023]$，$x\_2 \in \[0, 1]$）。

逻辑回归的损失函数是凸函数，其等高线在未标准化时是**狭长的椭圆**（因为两个方向的尺度差异巨大）。梯度下降的方向垂直于等高线，于是：

```
未标准化（椭圆等高线）：
        ┌─────────────────┐
        │  ╲            ╱  │   梯度方向偏离最优方向
        │   ╲    ●    ╱    │   优化路径呈"之"字形
        │    ╲  ↗  ╱       │   收敛缓慢
        │     ╲   ╱        │
        │      ╲ ╱         │
        │       ★          │   ★ 是最优点
        └─────────────────┘

标准化后（圆形等高线）：
        ┌─────────────────┐
        │       ●         │   梯度方向直指最优
        │      ↗          │   优化路径近似直线
        │     ↗           │   收敛迅速
        │    ↗            │
        │   ★             │
        └─────────────────┘
```

**数学解释**：梯度下降的收敛速度取决于损失函数的**条件数**（condition number）= 最大曲率 / 最小曲率。椭圆等高线的条件数大（病态），圆形等高线的条件数 = 1（最优）。标准化把所有特征缩放到相近尺度，让等高线接近圆形，条件数接近 1，收敛速度大幅提升。

> 🌟 **核心结论**：标准化的本质是**改善损失函数的几何性质**，让梯度下降"走直线"而不是"走之字"。

## 六、`fit_transform` vs `transform`：防止数据泄露

### 6.1 完整代码

```python
if scaler is not None:
    X_train = scaler.fit_transform(X_train_raw)
    X_test = scaler.transform(X_test_raw)
```

### 6.2 为什么要分开？

> 📌 **回顾案例 3**：这是机器学习工程中最重要的原则之一——**防止数据泄露（Data Leakage）**。

| 操作                           | 做什么                              | 用在哪个集    |
| ---------------------------- | -------------------------------- | -------- |
| `fit_transform(X_train_raw)` | 计算训练集的 μ/σ（或 median/IQR），并变换训练集  | **仅训练集** |
| `transform(X_test_raw)`      | 用**训练集的** μ/σ（或 median/IQR）变换测试集 | **测试集**  |

### 6.3 为什么测试集只能 `transform`？

如果对测试集也 `fit_transform`，相当于让测试集的统计信息（均值、标准差）参与了标准化过程——**测试集的信息就"泄露"到了预处理中**。这会导致：

1. **评估过于乐观**：模型在测试集上的表现会"虚高"，因为它已经"见过"测试集的分布。
2. **泛化能力下降**：部署到真实新数据时，性能会明显下降。

正确做法是：**测试集必须用训练集的统计量来变换**，模拟"模型面对全新数据"的场景。

> ⚠️ **常见错误**：
>
> ```python
> # ❌ 错误！测试集泄露
> X_test = scaler.fit_transform(X_test_raw)
>
> # ✅ 正确
> X_test = scaler.transform(X_test_raw)
> ```

## 七、LogisticRegression 参数详解

### 7.1 完整代码

```python
lr = LogisticRegression(
    class_weight='balanced',
    max_iter=5000,
    random_state=RANDOM_STATE,
    solver='lbfgs',
    tol=1e-5
)
```

### 7.2 参数逐个详解

#### 🌟 `class_weight='balanced'`（回顾案例 3）

> 📌 **回顾案例 3**：自动调整各类别权重，让少数类（VIVO，约 41%）获得更大权重，避免模型偏向多数类。

权重计算公式：

$$\text{weight}\_i = \frac{\text{总样本数}}{\text{类别数} \times \text{该类样本数}}$$

#### `max_iter=5000`（比案例 3 更大）

> 📌 **回顾案例 3**：案例 3 用的是 `max_iter=2000`。本模块改为 `5000`，原因是 **Raw（未标准化）方法收敛极慢**，需要更多迭代次数才能收敛。

从实际结果看，Raw 方法用了 **174 次迭代**才收敛，而 StandardScaler 只用了 **10 次**。如果 `max_iter` 设得太小（如默认的 100），Raw 方法会触发 `ConvergenceWarning`，导致比较不公平。设为 5000 是为了给 Raw 方法"足够的空间"。

#### 🌟 `tol=1e-5`（**本模块重点参数**）

**作用**：收敛容差（tolerance）。当损失函数的改善小于 `tol` 时，优化算法认为已收敛，停止迭代。

**默认值**：`1e-4`（0.0001）

**为什么设为** **`1e-5`？**

> 🌟 **关键原因**：本模块的核心指标之一是**迭代次数**（`n_iter_`）。如果 `tol` 设得太大（如默认的 1e-4），算法会"过早"停止，三种方法的迭代次数差异可能被掩盖。设为更小的 `1e-5`，让算法"更严格"地判断收敛，能更精确地反映三种方法的收敛速度差异。

| `tol` 值     | 含义                 | 对迭代次数的影响       |
| ----------- | ------------------ | -------------- |
| `1e-4`（默认）  | 损失改善 < 0.0001 即停   | 迭代次数偏少，差异不明显   |
| `1e-5`（本模块） | 损失改善 < 0.00001 即停  | 迭代次数更精确，差异更明显  |
| `1e-6`      | 损失改善 < 0.000001 即停 | 迭代次数更多，但可能过度优化 |

> 💡 **小贴士**：`tol` 和 `max_iter` 是一对"互补"的停止条件。算法在以下任一条件满足时停止：
>
> - 损失改善 < `tol`（收敛）
> - 迭代次数 ≥ `max_iter`（达到上限，触发警告）

#### `solver='lbfgs'`（回顾案例 3）

> 📌 **回顾案例 3**：L-BFGS 是一种拟牛顿法（Quasi-Newton），利用二阶导数信息加速收敛。对于中小型数据集（< 10 万样本），`lbfgs` 通常是最佳选择。其他可选 solver：`'liblinear'`（适合小数据 + L1 正则）、`'sag'`/`'saga'`（适合大数据）。

#### `random_state=RANDOM_STATE`

保证结果可复现。虽然 `lbfgs` 是确定性算法（不依赖随机数），但保留 `random_state` 是好习惯，方便切换其他 solver。

### 7.3 参数总览表

| 参数             | 值              | 与案例 3 的差异   | 原因             |
| -------------- | -------------- | ----------- | -------------- |
| `class_weight` | `'balanced'`   | 相同          | 处理类别不平衡        |
| `max_iter`     | `5000`         | 案例3是 2000   | 给 Raw 方法留足收敛空间 |
| `tol`          | `1e-5`         | 案例3用默认 1e-4 | 精确比较迭代次数       |
| `solver`       | `'lbfgs'`      | 相同          | 适合中小数据         |
| `random_state` | `RANDOM_STATE` | 相同          | 可复现            |

## 八、训练与计时

### 8.1 完整代码

```python
start = time.time()
lr.fit(X_train, y_train)
elapsed = time.time() - start
n_iter = lr.n_iter_[0]
```

### 8.2 `time.time()` 计时

`time.time()` 返回当前时间戳（秒）。在 `lr.fit()` 前后各调用一次，差值就是训练耗时。这是一种简单但有效的计时方法。

> ⚠️ **注意**：`elapsed` 包含了模型拟合的全部时间，但不包括数据标准化和预测的时间。对于本模块的小数据集（56,000 × 6），训练时间在毫秒级，差异主要反映迭代次数。

### 8.3 🌟 `lr.n_iter_[0]`：获取实际收敛迭代次数

**这是本模块的关键指标！**

#### `n_iter_` 是什么？

`n_iter_` 是 LogisticRegression 训练后暴露的一个属性，记录**实际收敛所用的迭代次数**。

- 类型：NumPy 数组，形状 `(n_classes,)` 或 `(1,)`
- 对于二分类，`n_iter_` 通常是一个长度为 1 的数组，所以用 `[0]` 取出标量值

#### 为什么重要？

`n_iter_` 直接反映了**优化算法的收敛速度**，是衡量标准化效果的最直观指标：

| 方法             | `n_iter_` | 含义              |
| -------------- | --------- | --------------- |
| Raw            | 174       | 收敛极慢，需要 174 次迭代 |
| StandardScaler | 10        | 收敛极快，只需 10 次迭代  |
| RobustScaler   | 11        | 收敛极快，只需 11 次迭代  |

**17 倍的差距！** 这就是标准化的威力。

> 💡 **小贴士**：`n_iter_` 只有在使用迭代式 solver（如 `lbfgs`、`sag`、`saga`）时才有意义。对于 `liblinear`（基于坐标下降），`n_iter_` 的含义略有不同。

## 九、预测与评估指标

### 9.1 完整代码

```python
# 预测
y_prob = lr.predict_proba(X_test)[:, 1]
y_pred = lr.predict(X_test)

# 评估
auc = roc_auc_score(y_test, y_prob)
recall = recall_score(y_test, y_pred, pos_label=1)
brier = brier_score_loss(y_test, y_prob)
ap = average_precision_score(y_test, y_prob)
```

### 9.2 两种预测方式

> 📌 **回顾案例 3**：

| 方法                            | 返回值                          | 用途                  |
| ----------------------------- | ---------------------------- | ------------------- |
| `predict_proba(X_test)[:, 1]` | 每个样本属于正类（VIVO）的概率，范围 \[0, 1] | 计算 AUC、Brier、PR-AUC |
| `predict(X_test)`             | 每个样本的预测类别（0 或 1），基于 0.5 阈值   | 计算 Recall、Accuracy  |

`[:, 1]` 是因为 `predict_proba` 返回形状 `(n_samples, 2)` 的数组，第 0 列是负类概率，第 1 列是正类概率。

### 9.3 四个评估指标

> 📌 **回顾案例 3**：

| 指标         | 公式                               | 衡量维度        | 越好越 |
| ---------- | -------------------------------- | ----------- | --- |
| **AUC**    | ROC 曲线下面积                        | 区分能力（排序）    | 大   |
| **Recall** | TP / (TP + FN)                   | 少数类捕捉率      | 大   |
| **Brier**  | $\frac{1}{N}\sum(p\_i - y\_i)^2$ | 概率校准度       | 小   |
| **PR-AUC** | PR 曲线下面积                         | 不平衡数据下的识别能力 | 大   |

#### AUC（ROC-AUC）

衡量模型**区分正负类的能力**。AUC = 0.5 表示随机猜测，AUC = 1.0 表示完美区分。AUC 只关心**排序**，不关心绝对概率值。

#### Recall（召回率，针对 VIVO）

$$\text{Recall} = \frac{TP}{TP + FN}$$

即"真实的 VIVO 样本中，有多少被正确预测为 VIVO"。在医学场景中，**漏诊存活患者**（FN）的代价往往很高，所以 Recall 是关键指标。

#### Brier Score（布里尔分数）

$$\text{Brier} = \frac{1}{N}\sum\_{i=1}^{N}(p\_i - y\_i)^2$$

衡量**预测概率的校准度**——预测概率是否接近真实标签。Brier 越低越好（0 表示完美）。

#### PR-AUC（Precision-Recall AUC）

对**不平衡数据**比 ROC-AUC 更敏感。当负样本很多时，ROC-AUC 可能"虚高"，而 PR-AUC 会"诚实"地反映模型对少数类的识别能力。

> 💡 **小贴士**：本模块的核心发现是——**这四个指标在三种标准化方法下几乎完全相同**。

## 十、系数绝对值均值

### 10.1 完整代码

```python
# 系数幅度
coef_mean_abs = np.abs(lr.coef_[0]).mean()
```

### 10.2 `lr.coef_` 是什么？

`lr.coef_` 是逻辑回归训练后暴露的**系数矩阵**：

- 形状：`(n_classes, n_features)`，对于二分类是 `(1, 6)`
- `lr.coef_[0]` 取出第 0 个类别的系数向量，形状 `(6,)`
- 每个元素对应一个特征的权重

### 10.3 `np.abs(...).mean()` 计算什么？

1. `np.abs(lr.coef_[0])`：取每个系数的绝对值，得到形状 `(6,)` 的数组
2. `.mean()`：求这 6 个绝对值的均值

结果是一个标量，衡量**系数的总体幅度**（magnitude）。

### 10.4 为什么要看系数幅度？

系数幅度反映了**模型对特征的依赖程度**。在标准化前后，系数幅度会发生巨大变化：

| 方法             | 系数均值   | 原因                            |
| -------------- | ------ | ----------------------------- |
| Raw            | 0.3482 | 特征量纲大（year \~2000），系数必须很小才能抵消 |
| StandardScaler | 0.6672 | 特征缩放到 \[-3, 3]，系数中等           |
| RobustScaler   | 1.1986 | IQR 比 σ 小，缩放后数值更大，系数也更大       |

> 🌟 **关键洞察**：**系数的绝对大小没有意义，只有相对大小才有意义**。Raw 方法系数小，不代表特征不重要；RobustScaler 系数大，也不代表特征更重要。**只有标准化后，系数才能横向比较**（详见第十一节）。

#### `x_pos = np.arange(len(feature_names)) + idx * 0.25`

这是**分组柱状图**的关键技巧：

- `x_pos`：柱子的 x 坐标
- `np.abs(lr.coef_[0])`：柱子高度 = 系数绝对值
- `width=0.2`：柱子宽度 0.2（小于偏移量 0.25，留出间隙）
- `label=name`：图例标签为方法名
- `alpha=0.8`：透明度 80%

#### `ax_coef.set_xticks(np.arange(len(feature_names)) + 0.25)`

x 轴刻度位置设在 `[0.25, 1.25, 2.25, 3.25, 4.25, 5.25]`——即三种方法柱子的**中间位置**，让特征名标签居中显示。

### 12.3 🌟 图表解读

![系数对比](../img/07b_coefficient_comparison.png)

#### 三种方法的系数幅度差异

| 方法             | 系数均值   | 视觉表现 |
| -------------- | ------ | ---- |
| Raw            | 0.3482 | 柱子最矮 |
| StandardScaler | 0.6672 | 柱子中等 |
| RobustScaler   | 1.1986 | 柱子最高 |

#### 为什么 RobustScaler 的系数最大？

> 🌟 **关键解释**：IQR 通常比 σ 小（因为 IQR 只覆盖中间 50% 的数据，而 σ 覆盖约 68% 的数据）。
>
> - StandardScaler 除以 σ（较大）→ 缩放后数值较小 → 系数需要较大才能拟合
> - RobustScaler 除以 IQR（较小）→ 缩放后数值较大 → 系数需要更大才能拟合
>
> **数学上**：如果 $x' = x / s$（s 是缩放因子），那么为了让 $w' \cdot x' = w \cdot x$（预测结果不变），必须有 $w' = w \cdot s$。所以缩放因子 s 越小，系数 w' 越大。
>
> - StandardScaler：$s = \sigma$（较大）→ $w'$ 较小
> - RobustScaler：$s = \text{IQR}$（较小）→ $w'$ 较大

#### 重要提醒

> ⚠️ **不要被柱子高度误导**：RobustScaler 的柱子最高，**不代表它的模型最好**。系数大小只反映量纲，不反映模型性能。三种方法的性能几乎完全相同（见第十三节）。

#### 三种方法的迭代次数差异巨大

| 方法             | 迭代次数    | 相对速度      |
| -------------- | ------- | --------- |
| Raw            | **174** | 1×（基线）    |
| StandardScaler | **10**  | **17.4×** |
| RobustScaler   | **11**  | **15.8×** |

**Raw 需要 174 次迭代，StandardScaler 只需 10 次——收敛速度提升约 17 倍！**

#### 为什么标准化能加速收敛？

> 🌟 **核心解释**：回顾第 5.5 节的等高线图。
>
> **未标准化（Raw）**：
>
> - 特征量纲差异巨大（year \~2000 vs Gender \~1）
> - 损失函数的等高线是**狭长椭圆**
> - 梯度方向偏离最优方向，优化路径呈"之"字形
> - 需要 174 次迭代才能"绕"到最优点
>
> **标准化后（StandardScaler / RobustScaler）**：
>
> - 所有特征缩放到相近范围
> - 等高线接近**圆形**
> - 梯度方向直指最优，优化路径近似直线
> - 只需 10–11 次迭代就到达最优点
>
> **数学上**：标准化把损失函数的**条件数**（condition number）从 \~1000 降到 \~1，收敛速度提升约 $\sqrt{\text{条件数}}$ 倍。

#### 为什么 RobustScaler 比 StandardScaler 多 1 次迭代？

RobustScaler 用 IQR 缩放，而 IQR 通常比 σ 小，导致缩放后的数值范围更大、更分散。这会让等高线略微"非圆"，收敛稍慢。但差异很小（11 vs 10），可以忽略。

> 💡 **小贴士**：迭代次数差异在大数据集上会更明显。本模块的数据集（56,000 × 6）较小，差异已经达到 17 倍。如果是 100 万样本 × 100 特征，差异可能达到 100 倍以上。

## 十五、核心讨论：标准化不改变性能但改变效率

这是本模块最重要的理论部分。我们需要回答三个问题：

### 15.1 为什么 AUC/Recall/Brier 完全相同？

> 🌟 **核心论点**：标准化只是对特征的**重新参数化（reparameterization）**，不改变模型的决策边界。

#### 数学证明

逻辑回归的决策函数是：

$$\hat{y} = \sigma(w\_1 x\_1 + w\_2 x\_2 + \dots + w\_n x\_n + b)$$

假设我们对特征做标准化 $x'\_i = (x\_i - \mu\_i) / \sigma\_i$，那么：

$$w\_i x\_i = w\_i (\sigma\_i x'\_i + \mu\_i) = (w\_i \sigma\_i) x'\_i + w\_i \mu\_i$$

令 $w'\_i = w\_i \sigma\_i$，$b' = b + \sum\_i w\_i \mu\_i$，则：

$$w\_1 x\_1 + \dots + w\_n x\_n + b = w'\_1 x'\_1 + \dots + w'\_n x'\_n + b'$$

**决策函数完全等价！** 所以：

- 预测概率 $\hat{y}$ 不变
- AUC、Recall、Brier 都不变
- 决策边界在原始空间中是同一个超平面

#### 直观理解

标准化只是"换了把尺子"来度量特征，模型学到的**决策边界**（在原始空间中）是同一个。所以预测结果完全相同，性能指标自然相同。

### 15.2 为什么迭代次数差异巨大？

> 🌟 **核心论点**：标准化改变了损失函数的**几何形状**，影响梯度下降的收敛路径。

#### 损失函数的几何

逻辑回归的损失函数是凸函数，其等高线（level sets）的形状取决于特征的尺度：

- **未标准化**：等高线是**狭长椭圆**（因为不同特征的尺度差异巨大）
- **标准化后**：等高线接近**圆形**（所有特征尺度相近）

#### 梯度下降的路径

梯度下降的方向是**等高线的法线方向**。在椭圆等高线上，法线方向往往偏离最优点；在圆形等高线上，法线方向直指圆心（最优点）。

```
椭圆等高线（Raw）：
        ╱──────╲
       ╱  ╲    ╱╲
      ╱   ╲↗  ╱  ╲     梯度方向偏离最优
      │    ●  │   │     路径呈"之"字形
      ╲    ╲  ╱   ╱     需要多次迭代
       ╲   ╲╱   ╱
        ╲────╱

圆形等高线（StandardScaler）：
         ┌───┐
        ╱     ╲
       │  ●↗   │         梯度方向直指最优
       │   ↓   │         路径近似直线
        ╲     ╱          迭代次数少
         └───┘
```

#### 条件数（Condition Number）

数学上，收敛速度取决于损失函数 Hessian 矩阵的**条件数** $\kappa = \lambda\_{\max} / \lambda\_{\min}$（最大特征值 / 最小特征值）：

- Raw：$\kappa \approx 10^6$（year 的方差 / Gender 的方差）
- StandardScaler：$\kappa \approx 1$（所有特征方差都是 1）

梯度下降的迭代次数大致与 $\sqrt{\kappa}$ 成正比，所以标准化后收敛速度提升约 $\sqrt{10^6} = 1000$ 倍（理论上）。实际提升 17 倍，是因为 `tol=1e-5` 已经比较严格，Raw 方法在 174 次就达到了这个精度。

### 15.3 为什么系数不同？

> 🌟 **核心论点**：量纲变化导致系数等比例缩放，但系数的**相对大小**（在标准化后）才反映特征重要性。

#### 系数与缩放因子的关系

如果 $x'\_i = x\_i / s\_i$（$s\_i$ 是缩放因子），那么为了让 $w'\_i x'\_i = w\_i x\_i$，必须有：

$$w'\_i = w\_i \cdot s\_i$$

| 方法             | 缩放因子 $s\_i$           | 系数 $w'\_i$                 |
| -------------- | --------------------- | -------------------------- |
| Raw            | 1（不缩放）                | $w\_i$（原始）                 |
| StandardScaler | $\sigma\_i$（标准差）      | $w\_i \cdot \sigma\_i$     |
| RobustScaler   | $\text{IQR}\_i$（四分位距） | $w\_i \cdot \text{IQR}\_i$ |

由于 $\text{IQR} < \sigma$（通常），所以 RobustScaler 的系数最大。

#### 系数可比性的意义

> 🌟 **关键洞察**：**只有标准化后，系数才能横向比较**。

**为什么？**

在 Raw 方法中，`year` 的系数非常小（因为 year 的数值很大，系数必须很小才能抵消），`Gender` 的系数可能很大（因为 Gender 只有 0/1）。如果直接比较系数大小，会得出"Gender 比 year 重要"的错误结论——实际上这只是量纲差异造成的假象。

标准化后，所有特征都在相近的尺度上，系数大小才真正反映**特征对预测的贡献**。这就是为什么在特征重要性分析中，**必须先标准化**。

> 💡 **小贴士**：在案例教程的后续模块（如 Lasso 特征选择、SHAP 解释），都会默认使用标准化后的特征，以保证系数可比性。

## 十七、核心要点强调

> 📌 **要点 1：标准化不改变性能**
>
> 标准化只是对特征的"重新参数化"，决策边界不变，预测结果不变，AUC/Recall/Brier 自然不变。如果你看到标准化后性能大幅提升，**很可能是数据泄露或模型本身有问题**（如正则化强度未调整）。

> 📌 **要点 2：标准化加速收敛**
>
> 标准化把损失函数的等高线从"椭圆"变成"圆"，让梯度下降"走直线"而不是"走之字"。本模块的实验显示，收敛速度提升约 17 倍。在大数据集上，提升会更明显。

> 📌 **要点 3：标准化后系数才可比**
>
> 未标准化时，系数大小受量纲影响（year 的系数必然很小，Gender 的系数必然很大），无法横向比较。标准化后，所有特征在相近尺度上，系数大小才真正反映特征重要性。

> 📌 **要点 4：StandardScaler vs RobustScaler**
>
> - 数据无离群值 → 用 StandardScaler
> - 数据有离群值 → 用 RobustScaler
> - 不确定 → 先画箱线图看看（见案例教程 2 的离群值模块）

## 十八、小贴士与常见问题

### 18.1 小贴士

> 💡 **小贴士 1：树模型不需要标准化**
>
> 决策树、随机森林、XGBoost、LightGBM 等树模型是基于**特征排序**的，对单调变换不敏感。所以树模型**不需要标准化**。只有线性模型（逻辑回归、线性回归、SVM）、神经网络、KNN、K-Means 等基于距离或梯度的模型才需要标准化。

> 💡 **小贴士 2：标准化只对训练集 fit**
>
> 永远记住：`fit` 只用在训练集，`transform` 用在测试集和新数据。这是防止数据泄露的铁律。

> 💡 **小贴士 3：正则化时标准化更重要**
>
> 如果逻辑回归加了 L1/L2 正则化（`penalty='l1'` 或 `'l2'`），标准化就**必须**做。因为正则化惩罚系数的绝对大小，未标准化时大量纲特征的系数天然较小，会受到"不公平的轻罚"，导致正则化效果失真。

> 💡 **小贴士 4：Pipeline 是最佳实践**
>
> 在生产代码中，建议用 `Pipeline` 把标准化和模型串联起来：
>
> ```python
> from sklearn.pipeline import Pipeline
> pipe = Pipeline([
>     ('scaler', StandardScaler()),
>     ('lr', LogisticRegression(class_weight='balanced'))
> ])
> pipe.fit(X_train_raw, y_train)
> ```
>
> 这样能自动保证"训练集 fit、测试集 transform"，避免数据泄露。

### 18.2 常见问题

> ❓ **Q1：为什么我的标准化后性能反而下降了？**
>
> **A**：检查以下几点：
>
> 1. 是否在测试集上用了 `fit_transform`（数据泄露）？
> 2. 是否加了正则化但没调 `C` 参数？标准化后系数幅度变化，正则化强度需要重新调整。
> 3. 是否有分类特征被错误地标准化了？分类特征（如 Gender）标准化后可能失去含义。

> ❓ **Q2：StandardScaler 和 RobustScaler 该选哪个？**
>
> **A**：
>
> - 如果数据近似正态分布、无明显离群值 → StandardScaler
> - 如果数据有离群值或分布偏态 → RobustScaler
> - 如果不确定 → 先画箱线图（见案例教程 2），看是否有离群点
> - 在本模块的实验中，两者性能几乎相同，选哪个都行

> ❓ **Q3：为什么 Raw 方法也能收敛？不是说未标准化不收敛吗？**
>
> **A**：未标准化**不是不收敛**，而是**收敛慢**。本模块设了 `max_iter=5000`，给 Raw 方法足够的空间（174 次远小于 5000）。如果用默认的 `max_iter=100`，Raw 方法会触发 `ConvergenceWarning`，结果可能不理想。

> ❓ **Q4：`n_iter_`** **为什么是数组而不是标量？**
>
> **A**：对于多分类问题（OvR 策略），逻辑回归会训练多个二分类器，每个都有自己的迭代次数，所以 `n_iter_` 是数组。对于二分类，数组长度为 1，用 `[0]` 取出。

> ❓ **Q5：标准化后系数变大，是不是模型"更复杂"了？**
>
> **A**：不是。系数大小只是量纲的反映，不代表模型复杂度。模型的复杂度由特征数量、正则化强度、模型结构决定，与系数绝对大小无关。

> ❓ **Q6：为什么** **`tol=1e-5`** **而不是默认的** **`1e-4`？**
>
> **A**：本模块的核心指标是迭代次数。`tol` 越小，算法越"严格"，迭代次数越多，三种方法的差异越明显。如果用默认的 `1e-4`，Raw 可能只迭代 50 次就停了，差异不够显著。

## 十九、本模块小结

### 19.1 我们学到了什么

1. **三种标准化方法的数学原理**
   - Raw：$x' = x$（不变换）
   - StandardScaler：$x' = (x - \mu) / \sigma$（Z-score，受离群值影响）
   - RobustScaler：$x' = (x - \text{median}) / \text{IQR}$（稳健，不受离群值影响）
2. **标准化的两个核心效果**
   - ✅ **加速收敛**：迭代次数从 174 降到 10（17 倍提升）
   - ✅ **让系数可比**：消除量纲差异，系数大小反映特征重要性
   - ❌ **不改变性能**：AUC/Recall/Brier 几乎完全相同
3. **工程实践要点**
   - 测试集只能 `transform`，不能 `fit`（防数据泄露）
   - `tol=1e-5` 用于精确比较迭代次数
   - `n_iter_[0]` 是衡量收敛速度的关键属性
   - 用字典组织多种方法，便于循环对比
4. **理论洞察**
   - 标准化是"重新参数化"，决策边界不变 → 性能不变
   - 标准化改善损失函数几何（椭圆→圆）→ 收敛加速
   - 系数与缩放因子成正比 → 标准化后系数才可比

### 19.2 核心公式速查

| 方法             | 公式                                          | 中心    | 尺度    |
| -------------- | ------------------------------------------- | ----- | ----- |
| Raw            | $x' = x$                                    | 原始    | 原始    |
| StandardScaler | $x' = \frac{x - \mu}{\sigma}$               | 均值→0  | 标准差→1 |
| RobustScaler   | $x' = \frac{x - \text{median}}{\text{IQR}}$ | 中位数→0 | IQR→1 |
