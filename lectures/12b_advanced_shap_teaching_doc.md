# 案例教程 12b：高级 SHAP 可视化增强分析

## 一、前序知识要求

### 1.1 为什么需要"高级"SHAP？

基础 SHAP 分析（蜂群图、条形图、依赖图）回答了"**哪些特征重要？**"，但无法回答更深层的问题：

| 问题 | 基础 SHAP | 高级 SHAP（本教程） |
|------|-----------|-------------------|
| "特征与预测的关系是直线还是曲线？" | 依赖图只画散点 + 线性趋势线 | **二次多项式拟合 + R² 对比** |
| "三个最重要特征联合起来长什么样？" | 逐个看，无法同时看 3 个 | **3D 散点图** |
| "不同预测概率区间的特征贡献如何变化？" | 只看整体 | **趋势图按预测概率排序** |
| "特征之间的交互结构是什么？" | Dependence 图只显示单个交互 | **网络图显示所有交互** |

### 1.2 前序知识

- **教程 12（基础 SHAP）**：理解 SHAP 值、全局/局部解释的概念
- **R² 决定系数**：衡量拟合优度，范围 [0,1]，1=完美拟合
- **皮尔逊相关系数**：衡量两个变量之间的线性相关性

### 1.3 R² 比较的直觉

```
线性拟合 (R²_lin): 假设 y = ax + b
   → 一条直线
   → 如果真实关系是曲线，R² 会很低

二次拟合 (R²_quad): 假设 y = ax² + bx + c
   → 一条抛物线（U 型或倒 U 型）
   → 能捕捉弯曲关系

ΔR² = R²_quad - R²_lin 的意义:
   ΔR² ≈ 0     → 关系本质是线性的
   ΔR² > 0.05  → 存在可检测的非线性
   ΔR² > 0.20  → 强烈非线性，线性方法严重失真
```

## 三、各模块详解

### 模块 2：二次拟合依赖图（核心突破）

#### 2.1 线性 vs 二次——一个直观理解

```
假设 Diagnostic.means 和 SHAP 值的关系:

线性拟合 (R²=0.0457):
  SHAP ↑
       |      ● ●
       |   ● ●   ● ●
       |  ●         ●   ●
       | ●           ● ●
       |●               ●●
       └────────────────────→ Diagnostic.means
  结论: R² 接近 0 → 没有线性关系!

二次拟合 (R²=0.7638):
  SHAP ↑
       |●
       |  ●  ●
       |    ●    ●
       |   ● ●    ●
       |  ●   ●    ●
       | ●     ●    ●●
       └────────────────────→ Diagnostic.means
  结论: R²=0.76 → 存在强烈的 U 型关系!
```

**教学要点**：R²=0.0457 vs R²=0.7638，ΔR²=+0.7181。线性方法几乎完全错过了这种非线性模式。在医学数据中，U 型关系（如诊断方式编码的特殊分布）非常常见，**MUST 使用非线性拟合验证**。

#### 2.2 六个特征的 R² 比较

| 特征 | R² 线性 | R² 二次 | ΔR² | 判断 | 教学含义 |
|------|---------|---------|-----|------|---------|
| **Diagnostic.means** | **0.0457** | **0.7638** | **+0.7181** | **高度非线性** | 线性回归完全错过，树模型才能捕获 |
| year | 0.6276 | 0.8595 | +0.2319 | 非线性 | 近年存活率改善存在加速趋势 |
| Age | 0.1648 | 0.3586 | +0.1938 | 非线性 | 年龄与存活率不是简单线性关系 |
| Code.Profession | 0.4523 | 0.5984 | +0.1461 | 有一定非线性 | 职业编码的分组效应 |
| Extension | 0.5698 | 0.6070 | +0.0372 | ≈ 接近线性 | 肿瘤扩展程度的影响相对规则 |
| **Raca.Color** | **0.7477** | **0.7485** | **+0.0008** | **本质线性** | 种族的影响 ≈ 直线关系 |

#### 2.3 为什么这很重要？

在机器学习建模中有一个常见的误解："树模型不需要关心特征与目标的关系形式，因为它能自动捕捉非线性。"

**这个说法没错，但问题在于——如果你不知道非线性程度，你就无法预判特征工程的优先级。**

```
实践意义:
  Diagnostic.means: ΔR²=0.7181 → 单变量变换（如平方项）可能大幅提升 LR 性能
  Raca.Color:       ΔR²=0.0008 → 无需变换，线性即可
  year:             ΔR²=0.2319 → 对数变换或分段可能有用
```

#### 2.4 教学陷阱

```
注意: R² 高 ≠ 因果关系

year 的 R²_quad=0.8595 意味着"year 与 SHAP 值的关系可以很好地
用抛物线描述"，但这不意味着"提高 year 就能提高存活率"。

这是因为:
  1. year 是时间变量 → 反映了医学进步的趋势
  2. 但 year 可能与诊断技术、治疗方案的改进高度相关
  3. SHAP 值是模型内部的贡献值，不是因果效应
```

### 模块 6：SHAP 趋势图

#### 6.1 含义

```
y 轴: SHAP 值 (特征贡献)
x 轴: 500 个测试样本，按模型预测概率从左(低)到右(高)排列

观察:
  - year 的 SHAP 值曲线: 从左到右稳步上升
    → year 对预测"存活"的贡献随预测概率增加而增加
    → 一致性高

  - Extension 的 SHAP 值曲线: 接近水平
    → Extension 对所有样本的贡献 ≈ 常数
    → 判别力弱

  - 灰色虚线: 归一化的预测概率参考线
    → 如果某个特征的曲线与它重合 → 该特征≈"代理预测概率"
```

#### 6.2 教学发现（本实验）

Top 3 特征的 SHAP 趋势与预测概率参考线高度同步——说明这三个特征（year, Diagnostic.means, Code.Profession）**共同解释了大部分预测方差**。其他特征的作用是在此基础上的"微调"。

### 模块 7：交互网络图

#### 7.1 如何解读

```
节点 = 特征
  - 大小 = 特征重要性 (SHAP mean |value|)
  - 颜色 = 重要性 (红=高, 蓝=低)

边 = 交互强度
  - 粗细 = |SHAP 值相关系数|
  - 透明度 = 交互强度
  - 只显示 |r| > 0.1 的边

网络结构的意义:
  - 高度连接的特征 → 中心节点，是模型的"决策核心"
  - 孤立节点 → 独立贡献，不与其他特征交互
  - 强边 → 两个特征共同作用，贡献模式相似
```

#### 7.2 本实验网络

| 交互对 | \|r\| | 解读 |
|--------|-----|------|
| **Diagnostic.means × year** | 0.156 | 最强交互 |
| **Diagnostic.means × Extension** | 0.155 | 诊断方式与肿瘤扩展相关 |
| **Raca.Color × Extension** | 0.133 | 种族与肿瘤扩展的关联 |
| Raca.Color × Code.Profession | 0.107 | |
| year × Extension | 0.066 | |

**教学讨论**：6 个特征在小样本（500）上交互强度普遍较低（< 0.16）。这说明**特征之间相对独立**——这不是交互少，而是我们当前的特征集（6 个基础特征）经过 Boruta 筛选后，冗余已经被消除了。

|------|--------|-----------------|---------|-----------|-----------|

| Age | 低 | 中 (0.19) | 弱 (0.06) | 低 | — |

### 4.2 特征工程决策树（基于此框架）

```
给定特征 X:

ΔR² (非线性程度) > 0.10?
  │
  ├── 是 → 创建多项式特征 (X², X³) 或分箱
  │     → 尤其推荐: Diagnostic.means, year, Age
  │
  └── 否 → 原始特征即可
        → Raca.Color, Extension

交互强度 > 0.15?
  │
  ├── 是 → 创建交互特征 (X × Y)
  │     → Diagnostic.means × year, Diagnostic.means × Extension
  │
  └── 否 → 无需交互项

分位数跨度 > 0.10?
  │
  ├── 是 → 特征对模型有足够"杠杆作用"
  │
  └── 否 → 特征贡献的波动小，优先级较低
```

## 五、扩展内容

### 5.1 二次拟合结果的统计检验

ΔR² = R²_quad - R²_lin 是否"具有统计显著性"？可以使用 F 检验：

```python
from scipy.stats import f

# F 统计量: F = (ΔR² / df1) / ((1 - R²_quad) / df2)
# df1 = 1 (增加了一个二次项)
# df2 = n - 3 (样本数 - 参数数)
F_stat = (r2_quad - r2_lin) / ((1 - r2_quad) / (n - 3))
p_value = 1 - f.cdf(F_stat, 1, n - 3)
```

### 5.2 高阶多项式 vs 非参数拟合

| 方法 | 灵活性 | 过拟合风险 | 可解释性 | 适用场景 |
|------|--------|-----------|---------|---------|
| 线性 (1阶) | 低 | 低 | 极高 | 关系明确线性 |
| 二次 (2阶) | 中 | 低 | 高 | 普遍适用 |
| 三次 (3阶) | 高 | 中 | 中 | 有 S 型曲线证据 |
| LOESS | 极高 | 高 | 低 | 探索性分析 |
| GAM (样条) | 极高 | 可控 | 中 | 研究级图表 |

### 5.3 交互网络图的局限

```
局限 1: 相关性 ≠ 交互性
  交互网络用 |SHAP 值相关系数| 近似交互强度
  但实际上, 特征 A 和 B 的 SHAP 值相关, 可能只是因为:
    A 和 B 本身相关 (特征共线性), 而非真正的"交互效应"

局限 2: 只捕捉线性交互
  SHAP 值之间的皮尔逊 r 只能捕捉线性相关
  如果交互是非线性的 (如 A 低×B 低→VIVO, A 高×B 低→MORTO),
  皮尔逊 r 可能接近 0

改进方向: 使用 SHAP Interaction Values
  shap.TreeExplainer 可以计算真正的 SHAP 交互值
  但计算量是普通 SHAP 的 N 倍 (N=特征数)
```

## 六、总结与思考题

### 6.1 核心收获

1. **线性拟合 ≠ 全部真相**：Diagnostic.means 的 ΔR²=+0.7181，线性回归几乎完全漏掉其与 SHAP 值的关系
2. **Raca.Color 是唯一"本质线性"特征**（ΔR²=0.0008）→ 它的影响是直线式的，容易建模也容易解释
3. **year 主导一切**：重要性最高（62% 分组贡献）、非线性明显、趋势一致性最强、分位数跨度最大
4. **交互网络相对稀疏**：6 个特征经过 Boruta 筛选后交互强度有限，说明特征集设计合理
5. **7 张图 ≠ 7 个独立结果**，而是从不同视角回答同一个问题——"特征如何驱动预测"
