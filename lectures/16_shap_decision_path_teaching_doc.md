# 案例教程 16：SHAP 决策路径分析 — 白盒化单个预测过程

## 一、前序知识要求

### 1.1 核心问题

> **单个预测是如何产生的？每个特征如何一步步将基准值推向最终预测？**

前面的教程解释了"哪些特征总体重要"和"不同患者群体的解释模式差异"，但没有回答一个核心问题：

**对于这一个患者，从平均预测到最终预测，中间发生了什么？**

本教程的答案——**SHAP 决策路径分析**：

```
基准值 (Base Value)
  ↓ + year 贡献      (第一大步)
  ↓ + Age 贡献       (第二大步)
  ↓ + Diag.means 贡献 (第三大步)
  ↓ ...
  = 最终预测概率
```

### 1.2 从全局到局部再到个体路径

| 教程 | 视角 | 回答的问题 |
|------|------|-----------|
| 12 | 全局 | 所有患者中哪些特征总体最重要？ |
| 13 | 全局+交互 | 哪些特征对相互影响？ |
| 14 | 全局+密度 | 依赖图模式在数据稀疏区还可信吗？ |
| 15 | 子群 | 哪些患者的决策模式相似？ |
| **16 (本教程)** | **个体** | **这个患者的预测是如何一步步生成的？** |

### 1.3 三个核心概念

```
基准值 (Base Value / Expected Value):
  模型的"平均预测"——在所有样本上的平均 log-odds。
  如果没有任何患者信息，模型会预测这个值。

  Base = log(p̄ / (1-p̄))
  其中 p̄ 是训练集中 VIVO 的平均概率

SHAP 值:
  每个特征对单个预测的"贡献"。
  正 → 推高 log-odds → 增加 VIVO 概率
  负 → 推低 log-odds → 增加 MORTO 概率

决策路径:
  从基准值出发，按 |SHAP| 从大到小排序，
  逐一累加每个特征的贡献，
  最终到达该患者的最终预测值。
```

### 1.4 前序知识

- **教程 12（基础 SHAP）**：理解 SHAP 值的含义
- **SHAP Waterfall Plot（教程 12 中的局部解释）**：单一样本的特征贡献排序
- **Log-odds 与概率的转换**：log-odds = log(p/(1-p))，p = 1/(1+e^{-log-odds})

## 三、各模块详解

### 模块 1：决策路径图（列 1）——"如何从起点到终点？"

#### 1.1 图的构成

```
Cumulative
SHAP (log-odds)
  ↑
0.8 │                    ● (特征 4)
    │                 ●
0.6 │           ● (特征 3)
    │        ●
0.4 │  ● (特征 1=year)
    │ /          ● (特征 5)
0.2 │● (Base)
    │       ● (特征 2)        ● (特征 6)
0.0 │──────────────────────────────→
    Base  F1   F2   F3   F4   F5   F6
                Step
```

#### 1.2 如何解读

| 特征 | SHAP 值 | 路径方向 | 含义 |
|------|---------|---------|------|
| year | +0.30 | 上升 | 诊断年份较晚 → 存活概率提升 |
| Age | -0.15 | 下降 | 年龄较大 → 存活概率降低 |
| Diagnostic.means | -0.08 | 下降 | 特定诊断方式 → 存活概率降低 |
| ... | ... | ... | ... |
| **累积** | **+0.50** | **Base → Final** | **最终预测 > 平均** |

#### 1.3 本实验的路径模式

```
样本 1 (低死亡, P=0.003):
  路径: Base(0.50) → 持续下降 → Final(-5.80)
  主要推低特征: year(较老？), Diagnostic.means

样本 4 (中存活, P=0.817):
  路径: Base(0.50) → 主要上升 → Final(1.50)
  主要推高特征: year(近年), Diagnostic.means

样本 5 (高存活, P=0.897):
  路径: Base(0.50) → 大幅上升 → Final(2.16)
  关键特征: year(近年) 和其他推高因素
```

**教学观察**：同样是死亡预测（样本 1 和 2），其决策路径的"形状"可能相似——说明模型的决策逻辑在不同严重程度的患者中保持一致。而存活患者（样本 3-5）的路径形状与死亡患者截然不同。

### 模块 2：SHAP 贡献条形图（列 2）——"每个特征贡献了多少？"

#### 2.1 图的构成

```
year            ██████████████ +0.302
Age             ████████ -0.148
Diagnostic.means ████ -0.076
Extension       ██ -0.032
Raca.Color      ██ -0.028
Code.Profession █ -0.010
                ─────────────────
                绿色=推高VIVO
                红色=推高MORTO
```

#### 2.2 与 Waterfall Plot 的区别

| 维度 | SHAP Waterfall (教程 12) | 本条形图 |
|------|-------------------------|---------|
| 特征排序 | 贡献从大到小 | 决策路径的特征顺序一致 |
| 概率映射 | 显示概率变化 | 显示 log-odds 变化 |
| 作用 | 快速查看影响因素 | 与决策路径配合，理解"顺序" |

**教学要点**：本条形图与决策路径的**特征顺序一致**。第一项对应路径的第一步，第二项对应第二步，以此类推。这是为了让读者将"累积路径"与"单项贡献"一一对应。

  多个特征共同贡献, 每一步增量不大

## 四、核心教学讨论

### 4.1 基准值 (Base Value) 的临床意义

本实验的基准值 = 0.5002（log-odds），对应的概率约为 **p = 62.2%**。

```
Base Value = 0.5002
  → log-odds = log(0.622 / 0.378) ≈ 0.5002
  → 模型的"默认预测"约为 62.2% VIVO 概率

但这个"默认"仅适用于训练集的先验分布。
实际训练使用了 class_weight='balanced'，
改变了先验 → 基准值不再是简单的类别比例。
```

### 4.2 决策路径的局限性

| 局限性 | 原因 | 应对 |
|--------|------|------|
| 路径顺序是人为的 | 按 |SHAP| 排序是一种常见选择，但其他排序方式也会产生不同的"叙事" | 关注累积值本身，而不是顺序 |
| 只展示了 Top K 特征 | 低贡献特征被隐藏 | 检查未展示特征的贡献之和 |
| 不能反映交互效应 | 路径是"可加"的，但模型可能有交互 | 结合教程 13（交互分析） |
| 单个样本解读的局限性 | 无法推广到群体 | 结合教程 15（聚类分析） |

### 4.3 教学价值定位

```
本教程与其他 SHAP 教程的分工:

教程 12: 全局 → "哪些特征重要"
教程 13: 交互 → "特征之间如何影响"
教程 14: 可信度 → "依赖图模式是否可信"
教程 15: 子群 → "哪些患者的决策模式相似"
教程 16: 个体 → "这个患者的决策是如何构成的"

教学定位:
  "从群体的森林中走出来,
   仔细观察一棵树的每一片叶子。"
```

## 五、扩展内容

### 5.1 其他"单样本解释"方法对比

| 方法 | 呈现形式 | 优势 | 劣势 |
|------|---------|------|------|
| SHAP Waterfall | 概率从基值向最终值变化 | 直观、标准 | 没有"步骤"概念 |
| **SHAP 决策路径 (本教程)** | 折线图+条形图+雷达图三位一体 | **多维信息整合** | 视觉负担较重 |
| LIME | 局部线性模型近似 | 快、独立 | 不稳定 |
| 决策树路径 | 展示决策树的分裂路径 | 人可读 | 只适用于单一决策树 |

### 5.2 从"决策路径"到"临床决策支持"

```
临床转化路径:

SHAP 决策路径 → 临床报告 → 医生决策

对于样本 4 (P(VIVO)=0.817):
  "这个患者的 VIVO 概率为 81.7%。
   主要推高因素:
     诊断年份较晚 (SHAP=+0.35) — 医疗技术进步
     诊断方式为 X (SHAP=+0.18) — 特定诊断方式的预后较好
   主要推低因素:
     年龄较大 (SHAP=-0.10)""

医生可以结合临床经验:
  1. 模型给出的解释是否合理?
  2. year 的推高效应是否在近年稳定?
  3. 诊断方式 X 的编码是否正确?
```

### 5.3 路径方向的稳定性

```
问题: 如果改变特征的排序方式,
      路径的"形状"会变吗?

排序方式 1: 按 |SHAP| 降序 (本教程)
  第一大步是最重要的特征 → 路径先"跳"一大步

排序方式 2: 按特征值大小排序
  第一大步是值最大的特征

排序方式 3: 随机排序
  路径随机摇摆

结论: 路径的"叙事"方式取决于排序方式。
     但最终累积值是不变的。
     因此建议:
       - 主要看累积值 (终点)
       - 次要关注哪些特征贡献大 (条形图)
       - 谨慎解读"顺序" (路径的中间步骤形状)
```

### 5.4 批量决策路径

当需要为大量患者生成决策路径时，可以制作"路径热图"：

```
患者 \  特征排序    F1   F2   F3   F4   F5  F6  累积
──────────────────────────────────────────────
患者 1  year   →    +0.20 +0.15 -0.05 +0.02 ... 0.50
患者 2  year   →    +0.18 -0.12 +0.08 -0.01 ... 0.45
患者 3  Diag   →    +0.25 +0.10 -0.08 -0.02 ... 0.30
...

→ 每一行 = 一个患者的决策路径
→ 颜色 = SHAP 值的正负和大小
→ 观察: 哪些患者的路径模式相似?
```

## 六、总结与思考题

### 6.1 核心收获

1. **决策路径将"黑箱预测"转化为可追踪的步骤**——从基准值出发，每一步是一个特征贡献的累加
2. **五组样本展示了不同预测概率下的决策逻辑差异**——死亡预测的路径以"下降"为主，存活预测以"上升"为主
3. **气象图连接了特征值和特征贡献**——将标准化的特征值与 SHAP 方向关联
4. **路径排序会影响"叙事"但不会改变终点**——关注累积值比关注步骤顺序更重要
5. **本教程是"个体化解释"的最佳实践**——填补了全局解释（教程 12-15）和单个 Waterfall 之间的空白

### 6.2 思考题

1. **（基础）** 本教程选择样本的策略是概率分位数（低/中/高）。如果改为随机选择 5 个样本，你认为还能得出有意义的结论吗？为什么？

2. **（进阶）** 决策路径图中的"步骤顺序"是按 |SHAP| 从大到小排序的。如果改为按特征重要性（全局 SHAP mean）排序，路径的"叙事"会如何改变？试在代码中实现并比较两个版本。

3. **（进阶）** 本教程的决策路径展示的是 log-odds 的累加。如果将其转换为概率（p = 1/(1+e^{-log-odds})），路径会变成什么形状？为什么非线性转换会改变路径的可视化效果？

4. **（拓展）** 比较样本 1（低死亡）和样本 5（高存活）的决策路径。它们在哪些特征上的贡献方向完全相反？在哪些特征上方向一致？这种对比揭示了对预测最重要的"区分性"特征是什么？

5. **（拓展）** 本教程的雷达图使用全局归一化（特征值的范围基于全体测试集）。如果改为基于该特征的训练集分位数（用百分位而不是 0-1 范围），雷达图的解释会更好吗？为什么？

6. **（开放）** "黑箱模型的决策路径解释"是否具有法律意义？如果一个模型预测某患者为"高危"从而改变了治疗方案，而决策路径显示了年份（year）是主要推高因素，这种解释是否足够支持医疗决策？还需要哪些额外的验证？
