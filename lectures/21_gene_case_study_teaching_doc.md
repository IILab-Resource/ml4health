## 二、基因组学核心概念 (新知识点)

前 20 个教程的数据要么是"现成表格", 要么是"影像→特征化". 基因组学数据则是
**直接的高维数值矩阵**: 行=样本 (肿瘤), 列=基因, 值=表达量. 理解以下概念才能正确迁移:

### 2.1 基因表达 (Gene Expression)

mRNA 表达量衡量某基因在某样本中的"活跃程度". 本数据的值已做 log 归一化
(典型的 RNA-seq 处理: count → log2(TPM+1)), 故值域 ≈ 0\~20, 0 表示该基因
在该样本中几乎不表达.

### 2.2 p≫n 问题 (高维低样本)

- **p** = 特征数 (20531 基因), **n** = 样本数 (801)
- p/n ≈ 25, 即每个样本要"决定" 25 个基因的权重 → 严重过拟合风险
- 普通逻辑回归此时**无解** (方程数 < 未知数), 必须靠正则化 (L1/L2) 求解
- 这是基因组学/影像组学/文本挖掘的共同痛点, **LASSO 是其标准解法** (见第四节)

### 2.3 Gene Signature (基因签名)

基因组学论文的核心产出. = 从上万基因中筛出的**几十\~几百个判别基因集**,
常命名为 "X-gene signature" (如 "92-gene cancer-type signature"). 它:

- 降维到可解释规模 (临床可做 PCR 验证)
- 每个基因有明确系数 (LASSO 线性) → 直接读出"哪个基因推动哪类癌症"
- 是教程 05 "特征选择" 在基因组学的具体应用形态

### 2.4 共表达与稀疏性 (基因组学特有)

- **共表达 (co-expression)**: 功能相关的基因表达高度相关 (gene modules),
  相关性去冗余是必要前置步骤 (教程 05 第一层在此特别重要).
- **稀疏性**: 2810 个基因在 >50% 样本中表达为 0 (低表达/沉默基因) →
  方差预筛可剔除它们, 减少 LASSO 输入.

## 四、为什么特征选择用 LASSO?

用户问题: *"基因组学特征会很多, 选一个最常用的方法"*.

**答: LASSO (L1 正则逻辑回归) 是基因 signature 论文的事实标准.**

- Golub et al., *Science* 1999 (白血病分型, 引用 14000+) 即用线性判别 + 稀疏.
- Tibshirani 1996 提出 LASSO, 专为 p≫n 设计.
- 优点:
  1. **自动稀疏** — L1 把无关基因系数精确压到 0 = 自动选基因, 无需手动设 K
  2. **可解释** — 线性系数 = 每个基因对每类癌症的"推动力", 直接读出生物学意义
  3. **p≫n 稳定** — L1 是少数在特征数 >> 样本数时仍可求解的方法 (普通 LR 无解)
  4. **与样本量适配** — 通过 CV 选正则强度 C, 自动平衡准确率与稀疏性

### 本案例三步标准流程 (教程 21 第 ④ 步)

1. **ANOVA 单变量预筛** (Top-2000): 20531 → 2000 基因 (剔除低区分度基因)
2. **相关性去冗余** (|r|>0.9): 2000 → 1817 基因 (剔除共表达模块冗余)
3. **LASSO** (L1 多项逻辑回归, 5-fold CV 选 C): 1817 → **92 个 signature 基因**

> 教学要点: 为什么不直接对 20531 基因上 LASSO?
> ① 计算慢且 saga 求解不稳定; ② 单变量预筛是基因组学惯例 (先去掉明显无关基因,
> 降低 LASSO 的搜索空间); ③ 相关性去冗余避免共表达基因都被选 (冗余 signature).

### 多分类 LASSO 的技术细节

- 二分类用 `solver='liblinear'` 即可; **多分类 + L1 必须用** **`solver='saga'`**
  (`liblinear` 不支持 multinomial + L1)
- 签名判定: 一个基因若在**所有 5 类**的系数都为 0 才算被剔除
  (只要在任意一类非零, 即为 signature 基因)
- 输出系数矩阵形状 `(5 类, 基因数)` → 每类有自己的标志基因

## 七、多分类适配要点 (与二分类的关键差异)

教程 01-12 大多基于**二分类** (VIVO/MORTO), 迁移到 5 分类时需注意:

| 维度           | 二分类 (教程)                    | 多分类 (本案例)                             | 改法                                         |
| ------------ | --------------------------- | ------------------------------------- | ------------------------------------------ |
| 统计检验         | T 检验 / Mann-Whitney         | **ANOVA F-test** (>2 组)               | `scipy.stats.f_oneway`                     |
| 效应量          | Cohen's d / rank-biserial r | **η² (eta-squared)**                  | 组间方差/总方差                                   |
| LASSO solver | liblinear                   | **saga** (multinomial+L1)             | `solver='saga', multi_class='multinomial'` |
| 评估指标         | AUC / Recall / F1           | **macro-F1 / weighted-F1 / AUC\_ovr** | `average='macro'`                          |
| 核心可视化        | ROC 单曲线                     | **混淆矩阵 + per-class ROC**              | `confusion_matrix` + `label_binarize`      |
| SHAP 形状      | (n, 特征)                     | **(n, 特征, 类别)** 3D                    | `np.abs(sv).mean(0).sum(1)` 聚合             |
| 不平衡处理        | SMOTE/阈值                    | **class\_weight='balanced'**          | 多分类 SMOTE 易引入噪声                            |

### SHAP 多分类聚合公式 (重点)

多分类树模型 SHAP 返回 3D 数组 `(样本, 基因, 类别)`. 看全局重要性需聚合:

```
global_importance = np.abs(shap_values).mean(axis=0).sum(axis=1)
                  # 先对样本求均值 → (基因, 类别), 再对类别求和 → (基因,)
```

- `.mean(axis=0)`: 每基因每类的平均绝对贡献
- `.sum(axis=1)`: 跨所有类汇总 → 该基因的"总重要性"
- 蜂群图/柱状图传入 `sv_arr.sum(axis=2)` 把 3D 压回 2D `(样本, 基因)`

## 七b、★ 多分类综合评估 (⑨b 补充): 多分类专用指标全家桶

> 用户反馈: "觉得缺少了多分类的评估, 请补充". 本节是 ⑨ 节的强化版, 给出多分类
> 评估的**完整工具箱**, 与教程 09 (建模对比) / 10 (不平衡) / 11 (校准 DCA) 对照.

### 为什么 ⑨ 节还不够? (单看 Accuracy 的盲区)

⑨ 节已给混淆矩阵 + per-class ROC + 校准 + 每类 P/R/F1, 但**仍可能漏判**:

- **Accuracy 是多数类主导的指标** — 本数据若只瞎猜 BRCA (300/801), Acc 也有 37%.
- 多分类 + 轻度不平衡 (BRCA 300 vs COAD 78) 下, 一个指标无法回答:
  "模型是真稳健, 还是靠多数类蒙对的?" → 必须用一组指标**交叉验证**.

### ⑨b 给出的多分类专用指标 (对照教程 09/10/11)

| 指标                         | 公式/含义                              | 解决什么盲区                     | 对照教程 |
| -------------------------- | ---------------------------------- | -------------------------- | ---- |
| **classification\_report** | 每类 P/R/F1 + macro/weighted 平均      | 一表看清各类 + 两种平均差异            | 09   |
| **Balanced Accuracy**      | 各类 Recall 的简单平均                    | 抗不平衡 (COAD 等权)             | 10   |
| **Macro-F1 / Weighted-F1** | 各类 F1 简单平均 / 按样本量加权                | macro 公平少数类, weighted 偏多数类 | 10   |
| **Cohen's κ (Kappa)**      | (观测一致 − 期望随机一致) / (1 − 期望)         | 扣除"瞎猜也能对"的部分, κ>0.8 几乎完美   | 09   |
| **MCC (Matthews)**         | 基于 TP/FP/TN/FN 全部 4 值的相关系数         | 多分类**最稳健**单值, 不被类别分布欺骗     | 09   |
| **Top-2 Accuracy**         | 真实类落在预测概率 Top-2 内即算对               | 5 类下衡量"排序质量"而非硬分类          | —    |
| **Per-class PR + AP**      | 每类 OvR 的 PR 曲线 + Average Precision | 不平衡下比 ROC 更敏感 (COAD 少数类)   | 11   |
| **Per-class AUC (OvR)**    | 每类 OvR 的 ROC-AUC                   | 各类可分性排序能力                  | 09   |

### ★ 为什么 κ 和 MCC 是多分类评估的关键?

Accuracy/F1 在**类别极不平衡**时会"虚高" (多数类对就够). κ 和 MCC 是补救:

- **Cohen's κ**: 若模型和真实完全一致, κ=1; 若只是随机猜, κ=0. 它扣除了"类别先验
  分布"带来的假一致. 例: 80% 样本是 BRCA 时, 全猜 BRCA 的 Acc=0.8, 但 κ=0.
- **MCC**: 唯一同时考虑 TP/FP/TN/FN 四象限的单值指标, 多分类扩展后对所有类别
  的混淆矩阵整体求相关. 范围 −1\~1, 0=随机, 1=完美. 它是**最不会被类别分布欺骗**
  的指标, 故多分类论文常报告 MCC.

> 迁移法则: 二分类看 AUC/Recall 即可; **多分类必看 κ + MCC** 交叉验证稳健性.

```
 LogReg   0.9938    0.9947     0.9918    0.9918   1.0000
    SVM   0.9938    0.9947     0.9918    0.9918   0.9999
     RF   0.9876    0.9848     0.9836    0.9836   1.0000

```

> **教学解读 (即使测试集全对, 指标仍有价值)**:
>
> 1. **5 模型横向对比** 揭示了 Accuracy 掩盖的差异: LogReg/SVM/RF 的 κ=0.99/MCC=0.99
>    略低于 XGBoost/LightGBM 的 1.0 — 即"少数类 COAD 在前 3 个模型上有 1\~2 个错配",
>    而单看 Accuracy (0.99 vs 1.0) 几乎看不出这个差距.
> 2. **κ ≈ MCC ≈ Macro-F1** 三者高度一致 → 模型没有靠类别先验"投机取巧" (否则 κ 会
>    明显低于 F1). 这是多分类模型可信度的关键判据.
> 3. 本数据集测试集近乎完美 (5 类器官特异癌症表达差异大), 故指标都接近 1.0;
>    真正的判别力差异体现在 **CV 阶段** (`21_model_comparison.csv` 的 CV\_Acc\_std)
>    与**少数类 COAD 的 AP/Recall** 上 — 这正是 ⑨b 横向对比 + ⑨ per-class 指标的互补价值.
> 4. 若换到"亚型分类"等更难任务 (各类表达更接近), 这些指标会出现明显分化, κ/MCC
>    将成为模型选型的决定性依据.

### 多分类评估指标选型决策树 (迁移到新多分类任务的速查)

```
是否类别不平衡?
├─ 否 (各类相近) → Accuracy + Macro-F1 足矣
└─ 是 (本案例轻度 3.8:1, 或更极端)
   ├─ 看少数类表现 → per-class P/R/F1 + 每类 AP (PR 曲线, 教程 11)
   ├─ 扣除随机一致 → Cohen's κ
   ├─ 最稳健单值 → MCC
   └─ 模型选型 → κ + MCC 同时最高的模型 (而非 Accuracy 最高)
```

   4 类但各类差异更微妙), signature 会变多还是变少? 准确率会怎样?
2. 本案例用 `class_weight='balanced'`, 若改用 SMOTE 重采样 (教程 10), 在 CV 中
   必须如何放置才不会泄漏? (提示: `imblearn.pipeline`)
3. 92 个 signature 基因若想进一步压缩到 20 个 (临床 PCR panel), 该调哪个参数?
   (提示: 减小 LASSO 的 C 上界, 或在 LASSO 后加 RF importance 取 Top-20)
4. 为什么 ANOVA+FDR 后 95% 基因都"显著" (FDR<0.05)? 这正常吗? 还需 LASSO 吗?
   (提示: 大样本+强效应 → 几乎全显著, 说明单变量筛不够, 需 LASSO 多变量稀疏)
5. 把本案例的 LASSO 换成教程 05 的 Boruta, 在 p=1817 时会怎样?
   (提示: Boruta 对 p≫n 退化, 教程 05 已注明此局限)
6. 若基因表达有缺失 (真实 RNA-seq 常有), 教程 03 的 KNN/MICE 插补在此适用吗?
   高维下哪个更稳? (提示: MICE 在 2 万维下不可行, 基因级 KNN 或中位数更现实)
7. 本案例测试集 XGBoost/LightGBM 全对 (Accuracy=1.0), 为什么仍要算 Cohen's κ 和 MCC?
   (提示: ① 5 模型横向对比中 κ 揭示了 LogReg/SVM 的 1\~2 个 COAD 错配; ② κ 扣除类别
   先验, 是"非偶然"的判据; ③ 换到更难的亚型任务时, κ/MCC 会成为选型决定性指标)
8. 若把 ⑨b 的 Top-2 Accuracy 换成 Top-1 (即普通 Accuracy), 在 5 类下哪个更有区分度?
   若任务变成 50 类 (如细胞类型分型), Top-2/Top-5 的价值如何变化?
   (提示: 类别越多, 硬分类越难, Top-K 衡量"排序质量"越重要)

## 十、与教程 20 (影像组学) 的对照总结

本案例与教程 20 是"新数据集迁移"的两条并行主线, 互为参照:

| 维度           | 影像组学 (教程 20)           | 基因组学 (教程 21)             |
| ------------ | ---------------------- | ------------------------ |
| 数据模态         | CT 影像                  | mRNA 表达矩阵                |
| 特征数          | 93 (需先 pyradiomics 提取) | 20531 (本就是矩阵)            |
| 任务           | 二分类 (增强/平扫)            | **多分类 (5 类癌症)**          |
| p≫n 程度       | 轻 (93 vs 100)          | **极端 (20531 vs 801)**    |
| 特征选择         | LASSO 一步               | **三步 (ANOVA→去冗余→LASSO)** |
| 缺失值          | 无                      | 无 (跳过插补)                 |
| 不平衡          | 完全平衡, 跳过               | **轻度, class\_weight**    |
| SHAP         | 2D                     | **3D 需聚合**               |
| signature 规模 | 13 特征                  | 92 基因                    |
| 测试准确率        | \~0.95                 | \~1.00                   |

**共同结论**: 流程骨架 (EDA→统计→预处理→特征选择→CV→建模→不平衡→评估→SHAP)
跨模态完全通用; 掌握这套骨架, 迁移到任何新数据 (影像/基因/文本/表格) 都只需
调整"因数据特性而不同的那几步". 这正是本教程体系的核心教学目标.
