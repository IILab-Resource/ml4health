# 案例教程 5：特征选择

## 一、前序知识要求

### 1.1 机器学习基础

| 概念                                | 简要说明                | 与本教程的关系                      |
| --------------------------------- | ------------------- | ---------------------------- |
| **共线性（Collinearity）**             | 两个特征高度相关，携带相似信息     | 相关性分析层检测并删除高度共线的变量           |
| **多重共线性（Multicollinearity）**      | 多个特征之间存在线性相关关系      | VIF 层识别并处理                   |
| **过拟合（Overfitting）**              | 模型学到噪声而非信号，泛化能力差    | 减少冗余特征 = 降低过拟合风险             |
| **正则化（Regularization）**           | 在损失函数中添加惩罚项，抑制过大的系数 | LASSO 的 L1 正则化将不重要特征系数压缩至 0  |
| **特征重要性（Feature Importance）**     | 衡量每个特征对模型预测的贡献程度    | Random Forest 和 Boruta 的核心输出 |
| **置换重要性（Permutation Importance）** | 打乱特征值后观察性能下降程度      | 与 Boruta 的"打乱-比较"思想类似        |

### 1.2 数学与统计基础

| 概念                          | 说明                                                   |
| --------------------------- | ---------------------------------------------------- |
| **皮尔逊相关系数 r**               | r ∈ \[-1, 1]，衡量两个变量的线性相关强度；\|r\| > 0.8 视为高度共线        |
| **方差膨胀因子 VIF**              | VIF = 1 / (1 - R²)，解释该特征能被其他特征预测的程度；VIF > 10 视为严重共线性 |
| **F 统计量（ANOVA）**            | 组间方差 / 组内方差，衡量特征对不同目标组的区分能力                          |
| **互信息（Mutual Information）** | 衡量两个变量之间的"共享信息量"，非线性版本的相关性                           |
| **L1 正则化**                  | 在损失函数中添加 Σ\|wᵢ\|，使不重要特征的系数变为 0                       |
| **统计检验**                    | Boruta 使用阴影特征构建零分布，对每个特征做统计显著性判断                     |

### 1.3 前序教程回顾

- 特征工程：[案例 4](04_feature_engineering_teaching_doc.md) 构造了 Age\_Group、Age\_Sq 等特征，但也引入了共线性（Age\_Age\_Sq 相关性 0.98）
- 标准化：StandardScaler 使系数可比，但并不能解决共线性问题
- 统计学分析：所有 22 个特征均显著，但效应量差异巨大——需要特征选择做二次筛选

## 二、本次实践内容与目的

### 2.1 核心目的

> **从 6 个不同角度对特征进行层层筛选，在保留预测能力的前提下最小化特征数量，从而提高模型的泛化能力。**

特征选择不是"要不要做"的问题，而是"做到什么程度"的问题。好的特征选择可以：

- 降低过拟合风险
- 缩短训练时间
- 提高模型可解释性
- 节省数据收集成本

### 2.2 六层特征选择体系

本教程构建了一个从简单到复杂的**六层特征选择流水线**，每层都有其独特的视角和数学依据：

```
                    ┌──────────────────────────────────────┐
                    │        六层特征选择流水线              │
                    └──────────────────────────────────────┘
                                      │
        ┌─────────────┬───────────┬───┴───┬───────────┬─────────────┐
        ▼             ▼           ▼       ▼           ▼             ▼
   ┌─────────┐  ┌─────────┐  ┌──────┐  ┌──────┐  ┌────────┐  ┌─────────┐
   │相关性分析 │  │   VIF   │  │Filter│  │LASSO │  │   RF   │  │  Boruta │
   │         │  │         │  │      │  │      │  │Importance│  │        │
   ├─────────┤  ├─────────┤  ├──────┤  ├──────┤  ├────────┤  ├─────────┤
   │删除高度 │  │识别多重 │  │ANOVA│  │L1正则│  │树模型 │  │随机阴影 │
   │共线对   │  │共线性   │  │+  MI │  │化压缩│  │重要性  │  │变量竞争 │
   └─────────┘  └─────────┘  └──────┘  └──────┘  └────────┘  └─────────┘
        │             │          │         │          │            │
        ▼             ▼          ▼         ▼          ▼            ▼
   ┌──────────────────────────────────────────────────────────────────┐
   │                    最终验证: Logistic Regression                 │
   │         比较不同特征集在 AUC / Recall / Brier 上的表现           │
   └──────────────────────────────────────────────────────────────────┘
```

### 2.3 各层方法对比

| 层 | 方法                | 类型  | 是否考虑特征交互   | 输出形式                         | 计算成本 |
| - | ----------------- | --- | ---------- | ---------------------------- | ---- |
| 1 | 相关性分析             | 无监督 | 否（仅两两关系）   | 保留/删除                        | 极低   |
| 2 | VIF               | 无监督 | 是（多重共线性）   | 迭代删除                         | 低    |
| 3 | Filter (ANOVA/MI) | 单变量 | 否          | Top-K 排序                     | 低    |
| 4 | LASSO             | 嵌入法 | 部分         | 系数 ≠ 0 的特征                   | 中    |
| 5 | RF Importance     | 嵌入法 | 是（树模型天然捕捉） | Top 累积重要性                    | 中    |
| 6 | Boruta            | 包装法 | 是          | Confirmed/Tentative/Rejected | 高    |

### 2.4 关键发现摘要

#### 各层筛选结果

| 层      | 保留     | 删除 | 保留的特征                                                                                                                                             |
| ------ | ------ | -- | ------------------------------------------------------------------------------------------------------------------------------------------------- |
| 原始     | 12     | 0  | Age, year, Gender, Code.Profession, Code.of.Morphology, Diagnostic.means, Extension, Raca.Color, Age\_Group, Age\_Sq, Year\_From\_2000, Is\_Child |
| 相关性    | **9**  | 3  | 移除 Age\_Group, Age, year（与 Age\_Sq/Year\_From\_2000 高度共线）                                                                                         |
| VIF    | **9**  | 0  | 所有特征 VIF ≤ 1.64，无多重共线性问题                                                                                                                          |
| Filter | **8**  | 4  | ANOVA+MI 平均排名 Top 60%                                                                                                                             |
| LASSO  | **11** | 1  | 仅 Age 被压缩至 0                                                                                                                                      |
| RF     | **6**  | 6  | 累积重要性 ≥ 90%                                                                                                                                       |
| Boruta | **11** | 1  | Confirmed: 11个 / Rejected: Is\_Child                                                                                                              |

#### 逻辑回归验证结果

| 特征集               | 特征数    | AUC        | Recall     | Brier      |
| ----------------- | ------ | ---------- | ---------- | ---------- |
| All Features      | 12     | 0.8980     | 0.8874     | 0.1281     |
| Correlation       | 9      | 0.8980     | 0.8871     | 0.1281     |
| Filter            | 8      | 0.8926     | 0.8957     | 0.1310     |
| **RF Importance** | **6**  | **0.8883** | **0.8996** | **0.1342** |
| **Boruta**        | **11** | **0.8979** | **0.8872** | **0.1281** |

> **核心发现（教学重点）**：大多数特征集的 AUC 在 0.888-0.898 之间，差异很小。这意味着：
>
> 1. 用 9 个特征（Correlation）或 11 个特征（Boruta）可以达到与 12 个全特征几乎相同的性能
> 2. 即使只用 6 个特征（RF 选择），AUC 也仅下降约 1%，但 Recall 反而提升
> 3. 冗余特征没有带来额外的预测价值，特征选择成功实现了"降维不降质"

## 三、扩展内容

### 3.1 为什么不同方法给出的"最佳特征集"不同？

| 方法         | 认为最重要的特征                    | 原因                   |
| ---------- | --------------------------- | -------------------- |
| **ANOVA**  | Diagnostic.means            | 线性视角下，该特征对两组均值差异最大   |
| **MI**     | Code.of.Morphology          | 信息论视角下，该特征与目标共享信息最多  |
| **LASSO**  | Diagnostic.means（系数 -0.191） | 正则化路径上，该特征贡献最稳定      |
| **RF**     | Diagnostic.means（重要性 0.32）  | 树模型视角下，该特征最常用于分割节点   |
| **Boruta** | 11 个 Confirmed              | 统计检验视角下，11 个特征显著优于随机 |

**教学要点**：不同的特征选择方法从不同的角度看待"重要性"，没有绝对的"正确"答案。推荐：

- 用多种方法做交叉验证
- 关注所有方法都认可的特征（交集）
- 结合领域知识做最终决策

### 3.2 相关性分析与 VIF 的区别

| 对比维度       | 相关性分析                     | VIF                |
| ---------- | ------------------------- | ------------------ |
| **检测什么**   | 两个特征之间的线性关系               | 一个特征与其他所有特征之间的线性关系 |
| **数学形式**   | r = cov(x, y) / (σx × σy) | VIF = 1 / (1 - R²) |
| **是否是迭代的** | 一次性检测所有对                  | 迭代删除最高 VIF 特征      |
| **本数据集结果** | 移除 3 个高度相关特征              | 无进一步移除（VIF 均 < 2）  |

**关键洞察**：VIF 是多变量的扩展。即使两两相关性不高，一个特征也可能由其他多个特征的线性组合所预测。这就是为什么 VIF 是比简单相关性分析更全面的工具。

### 3.3 LASSO 的路径图解读

LASSO 正则化路径图展示了随着正则化强度（alpha）的增加，不同特征的系数如何被逐步压缩至零：

```
Alpha 很小时:   所有特征的系数都接近"普通逻辑回归"的解
Alpha 增大时:   最不重要的特征系数先被压缩至 0
Alpha 达到最优:  最优 alpha 处，Age 被压缩为 0（其余 11 个保留）
Alpha 继续增大:  更多特征被压缩为 0，模型越来越简单
Lambda → ∞:    所有系数为 0，只剩下截距
```

**教学要点**：LASSO 不仅仅是一个特征选择工具，它还是一个**特征重要性排序器**——被最后压缩到零的特征，就是最重要的特征。

### 3.4 Boruta：随机变量竞争机制详解

Boruta 的核心思想是"让真实特征与随机特征竞争"，具体流程：

```
第 1 步: 创建阴影特征
  ┌─────────────────────────────────────────┐
  │  原始数据    打乱副本                    │
  │  ┌─────┐   ┌─────┐                     │
  │  │ Age │   │Age* │  ← 对 Age 列随机打乱 │
  │  │year │   │year*│  ← 对 year 列随机打乱 │
  │  │ ... │   │ ... │                     │
  │  └─────┘   └─────┘                     │
  └─────────────────────────────────────────┘

第 2 步: 训练随机森林，记录真实和阴影特征的重要性
  ┌─────────────────────────────────────────┐
  │  每个真实特征: 重要性得分 Z             │
  │  每个阴影特征: 重要性得分 Z*            │
  │  阴影特征中的最大值: Z*_max             │
  └─────────────────────────────────────────┘

第 3 步: 统计检验（多次迭代后）
  ┌─────────────────────────────────────────┐
  │  如果 Z > Z*_max (95% 分位): Confirmed │
  │  如果 Z < Z*_max (95% 分位): Rejected  │
  │  如果无法确定: Tentative               │
  └─────────────────────────────────────────┘
```

**为什么 Boruta 在医学论文中越来越流行？**

| 原因                   | 详细说明                                           |
| -------------------- | ---------------------------------------------- |
| **统计严谨性**            | 通过随机阴影变量构建零分布，对每个特征进行正式统计检验                    |
| **内置 Bonferroni 校正** | 多重比较下控制假阳性率                                    |
| **无需预设阈值**           | 不依赖人工指定 K 值、alpha、VIF 阈值                       |
| **交互效应友好**           | 基于 RF，天然捕捉非线性和交互效应                             |
| **结果直观**             | Confirmed / Tentative / Rejected 三分类，临床研究者容易理解 |
| **可重现性**             | 随机种子固定后，结果可完全复现                                |

### 3.5 更值得探索的特征选择方向

#### 方向一：Wrapper 方法

| 方法              | 原理               | 特点               |
| --------------- | ---------------- | ---------------- |
| **RFE（递归特征消除）** | 每次移除最不重要的特征，递归训练 | 考虑特征交互，但计算昂贵     |
| **前向选择**        | 从空集开始逐步添加特征      | 考虑特征交互，但可能陷入局部最优 |
| **后向消除**        | 从全集开始逐步删除特征      | 全面但计算量大          |

#### 方向二：稳定性选择（Stability Selection）

- 在数据的 Bootstrap 样本上多次运行 LASSO
- 统计每个特征被选中的频率
- 频率高的特征更可靠

#### 方向三：模型特定的特征选择

| 模型       | 特征选择方法                              |
| -------- | ----------------------------------- |
| **树模型**  | 特征重要性、SHAP 值、Permutation Importance |
| **线性模型** | LASSO、Elastic Net、逐步回归              |
| **神经网络** | 注意力权重、输入梯度、DeepLIFT                 |

#### 方向四：更多 Filter 方法

| 方法               | 适用场景        | 特点              |
| ---------------- | ----------- | --------------- |
| **卡方检验**         | 分类特征 × 分类目标 | 与案例 2 的统计检验一脉相承 |
| **信息增益**         | 离散特征 × 分类目标 | 决策树的分裂准则        |
| **ReliefF**      | 数值特征 × 分类目标 | 考虑近邻样本的类别差异     |
| **Fisher Score** | 数值特征 × 分类目标 | 类间分散 / 类内聚集     |

### 3.6 讨论：特征选择后的性能为什么没有下降？

在本教程中，Boruta 筛选出的 11 个特征与全特征 12 个的 AUC 几乎完全相同（0.8979 vs 0.8980）。这是巧合吗？

**原因 1：冗余特征没有新增信息**

RF 重要性显示前 6 个特征的累积重要性已达 91.8%，后 6 个特征只贡献了 8.2%。删除这些"尾部特征"不会显著影响模型。

**原因 2：线性模型的局限性**

逻辑回归是线性模型，对于高度相关的特征对（如 Age 和 Age\_Sq），模型会自动调整权重分配——一个特征的权重上升时，另一个会下降。

**原因 3：这不是偶然，而是必然**

信息论告诉我们：特征的相关性结构意味着存在信息冗余。特征选择的核心就是找到"最小充分特征集"——它包含了几乎与全集相同的信息量。

**教学延伸**：如果换一个非线性模型（如 XGBoost），或者数据中存在更强的噪声，特征选择的优势会变得更加明显。

## 四、总结与思考题

### 4.1 核心收获

1. **特征选择 ≠ 牺牲性能**：9 个特征（Correlation）可以达到 12 个全特征的性能水平
2. **不同方法不同视角**：相关性看线性关系，VIF 看多重共线性，Filter 看单变量关联，LASSO 看正则化路径，RF 看树模型贡献，Boruta 看随机变量竞争
3. **VIF > 相关性分析**：VIF 检测的是"一个 vs 所有"，比"一对一"的相关性分析更全面
4. **LASSO 的路径图非常有用**：不仅给出"选中了谁"，还展示"谁先被淘汰"
5. **Boruta 的统计框架最严谨**：不依赖人工阈值，给出正式统计结论
6. **特征选择提高泛化能力**：更少的特征 = 更简单的模型 = 更低的过拟合风险

### 4.2 思考题

> 以下问题可作为课后思考或课堂讨论：

1. **（基础）** 本教程中，LASSO 只压缩了 1 个特征（Age）为 0。如果增大正则化强度 alpha，会有什么变化？试试看。
2. **（进阶）** Boruta 仅将 Is\_Child 判定为 Rejected。Is\_Child 是特征工程阶段构造的二值特征（是否为儿童），但它在 RF 重要性排名最低（0.0003）。为什么这个构造特征没有带来预测价值？
3. **（进阶）** VIF 分析显示本数据集的最高 VIF = 1.64（远小于 10）。哪些类型的数据集容易出现 VIF > 10 的问题？（提示：房价预测中的房间数 × 面积）
4. **（拓展）** RF Importance 选择了 6 个特征，Boruta 选择了 11 个特征。RF 的 6 个特征是 Boruta 的子集。RF 丢弃但 Boruta 保留的 5 个特征（含构造特征 Age\_Group、Age\_Sq）说明了什么？两种方法的保守程度有何差异？
5. **（拓展）** 如果让你在医院的实际项目中做特征选择，你会只用 Boruta 吗？还是会结合其他方法？请设计一个组合策略。
6. **（开放）** "特征选择后性能不变"是好是坏？如果特征选择后模型性能下降了，可能是什么原因？如何决定"性价比最优"的特征数量？
7. **（实践）** 在本实验的数据中加入一个**纯随机噪声特征**（np.random.randn），观察 Boruta 和 LASSO 如何对待它。这个实验说明了什么？
