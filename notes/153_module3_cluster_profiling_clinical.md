# 模块 3：聚类画像与相对重要性临床解读

> 本模块是案例教程 15「SHAP 聚类分析」的**收尾模块**。在模块 2 中，我们绘制了 2×3 综合可视化面板，初步看到了两个簇的差异。本模块将深入分析每个簇的**相对重要性**（Relative Importance），计算 `rel_imp = this_imp / global_imp`，找出"在某些亚群中异常突出"的特征，并把分析结果保存到文本文件。 
>
> 本模块最核心的知识点有三个：**一是相对重要性的创新用法**——`rel_imp > 1` 表示该特征在该簇中比全局更重要，`rel_imp < 1` 表示更不重要；**二是如何把聚类结果转化为临床洞察**——从"Cluster 2 的 Diagnostic.means = 1.83x"推导出"这部分患者的预后主要由诊断方式驱动"；**三是结果文件的格式化输出**——用 f-string 的对齐语法生成整齐的表格。

## 二、详细画像分析

### 2.1 打印模块标题

```python
print("\n" + "=" * 70)
print("[4] 聚类画像详细分析")

```

### 2.2 打印表头

```python
print(f"\n{'Cluster':<12} {'Size':>8} {'%':>8} {'VIVO%':>8} {'MORTO%':>8} {'Top Features (rel. to global)':<40}")
print(f"{'-'*12} {'-'*8} {'-'*8} {'-'*8} {'-'*8} {'-'*40}")
```

#### f-string 对齐语法详解

- **`{'Cluster':<12}`**：左对齐，宽度 12 字符。`<` 表示左对齐。
- **`{'Size':>8}`**：右对齐，宽度 8 字符。`>` 表示右对齐。
- **`{'%':>8}`**：右对齐，宽度 8。
- **`{'VIVO%':>8}`**：右对齐，宽度 8。
- **`{'MORTO%':>8}`**：右对齐，宽度 8。
- **`{'Top Features (rel. to global)':<40}`**：左对齐，宽度 40。

第二行用 `'-'*12` 等生成对应宽度的分隔线，让表格更整齐。

> 💡 **小贴士：f-string 对齐语法**
>
> Python f-string 的对齐语法：`{value:align width}`，其中：
>
> - `align`：`<` 左对齐，`>` 右对齐，`^` 居中。
> - `width`：字段宽度（字符数）。
>
> 例如：
>
> - `f"{'ABC':<10}"` → `'ABC       '`（左对齐，右侧补空格）
> - `f"{'ABC':>10}"` → `'       ABC'`（右对齐，左侧补空格）
> - `f"{'ABC':^10}"` → `'   ABC    '`（居中，两侧补空格）
>
> 对于数字，还可以指定精度：`f"{0.1234:>8.2f}"` → `'    0.12'`（右对齐，宽度 8，2 位小数）。

### 2.3 遍历每个簇生成画像

```python
for c in range(best_n):
    mask = clusters == c
    n = mask.sum()
    pct = n / len(clusters) * 100
    vivo_pct = y_shap[mask].mean() * 100
    morto_pct = 100 - vivo_pct

    this_imp = np.abs(sv[mask]).mean(axis=0)
    global_imp = np.abs(sv).mean(axis=0)
    rel_imp = this_imp / (global_imp + 1e-8)
    top_rel_idx = np.argsort(rel_imp)[::-1][:3]
    top_str = ', '.join([f"{feature_names[i]} ({rel_imp[i]:.2f}x)" for i in top_rel_idx])

    print(f"  Cluster {c+1:<6} {n:>8} {pct:>7.1f}% {vivo_pct:>7.1f}% {morto_pct:>7.1f}% {top_str:<40}")
```

#### 逐行解释

##### 基本统计

- **`for c in range(best_n):`**：遍历每个簇（c=0, 1）。
- **`mask = clusters == c`**：布尔掩码，标记属于簇 c 的样本。
- **`n = mask.sum()`**：簇 c 的样本数。
- **`pct = n / len(clusters) * 100`**：簇 c 占总样本的百分比。
- **`vivo_pct = y_shap[mask].mean() * 100`**：簇 c 的 VIVO 比例（百分比）。
  - `y_shap[mask]`：簇 c 样本的目标值（0/1）。
  - `.mean()`：均值 = VIVO 比例（因为 VIVO=1, MORTO=0）。
  - `* 100`：转为百分比。
- **`morto_pct = 100 - vivo_pct`**：MORTO 比例 = 100 - VIVO 比例。

##### 计算相对重要性

- **`this_imp = np.abs(sv[mask]).mean(axis=0)`**：簇 c 的平均绝对 SHAP 值。
  - `sv[mask]`：簇 c 样本的 SHAP 值，形状 `(n_c, 6)`。
  - `np.abs(...)`：取绝对值。
  - `.mean(axis=0)`：沿样本轴求均值，得到 `(6,)` 数组。
- **`global_imp = np.abs(sv).mean(axis=0)`**：全局平均绝对 SHAP 值，形状 `(6,)`。
- **`rel_imp = this_imp / (global_imp + 1e-8)`**：相对重要性。
  - `+ 1e-8`：防止除以 0（虽然 SHAP 重要性通常不为 0，但这是良好的编程习惯）。
  - `rel_imp` 形状 `(6,)`，每个元素是该特征在簇 c 中的重要性相对于全局的倍数。

> 💡 **小贴士：相对重要性的解读规则**
>
> - `rel_imp > 1.0`：该特征在簇 c 中比全局更重要（**异常突出**）。
> - `rel_imp = 1.0`：该特征在簇 c 中的重要性与全局相同。
> - `rel_imp < 1.0`：该特征在簇 c 中比全局更不重要（**被弱化**）。
>
> 例如：
>
> - `rel_imp = 1.83`：该特征在簇 c 中的重要性是全局的 1.83 倍。
> - `rel_imp = 0.74`：该特征在簇 c 中的重要性只有全局的 74%。

##### 找出 Top 3 相对重要性特征

- **`top_rel_idx = np.argsort(rel_imp)[::-1][:3]`**：
  - `np.argsort(rel_imp)`：返回 rel\_imp 从小到大排序的索引。
  - `[::-1]`：反转，变成从大到小。
  - `[:3]`：取前 3 个，即相对重要性最高的 3 个特征的索引。
- **`top_str = ', '.join([f"{feature_names[i]} ({rel_imp[i]:.2f}x)" for i in top_rel_idx])`**：
  - 列表推导：对每个索引 i，生成 `特征名 (倍数x)` 格式的字符串，如 `Diagnostic.means (1.83x)`。
  - `', '.join(...)`：用逗号空格连接，如 `Diagnostic.means (1.83x), Extension (1.55x), Code.Profession (1.14x)`。

##### 打印行

- **`print(f"  Cluster {c+1:<6} {n:>8} {pct:>7.1f}% {vivo_pct:>7.1f}% {morto_pct:>7.1f}% {top_str:<40}")`**：
  - `Cluster {c+1:<6}`：簇编号，左对齐宽度 6。
  - `{n:>8}`：样本数，右对齐宽度 8。
  - `{pct:>7.1f}%`：百分比，右对齐宽度 7，1 位小数。
  - `{vivo_pct:>7.1f}%`：VIVO 百分比。
  - `{morto_pct:>7.1f}%`：MORTO 百分比。
  - `{top_str:<40}`：Top 特征字符串，左对齐宽度 40。

## 三、相对重要性的完整对比表

根据教学文档，完整的相对重要性对比如下：

| 特征               | 全局 SHAP | Cluster 1 | Cluster 2 | rel\_imp (C1) | rel\_imp (C2) |
| ---------------- | ------- | --------- | --------- | ------------- | ------------- |
| year             | 0.269   | 0.250     | 0.350     | 0.93x         | 1.30x         |
| Diagnostic.means | 0.061   | 0.030     | 0.112     | 0.49x         | **1.83x** ⬆️  |
| Extension        | 0.011   | 0.005     | 0.017     | 0.45x         | **1.55x** ⬆️  |
| Code.Profession  | 0.037   | 0.030     | 0.043     | 0.81x         | 1.14x         |
| Age              | 0.031   | 0.034     | 0.023     | 1.10x         | 0.74x (↓)     |
| Raca.Color       | 0.024   | 0.025     | 0.018     | 1.06x         | 0.78x (↓)     |

### 3.1 关键发现

1. **Diagnostic.means**：在 Cluster 2 中是 1.83x，在 Cluster 1 中只有 0.49x——差异巨大！这是 Cluster 2 的"旗帜特征"。
2. **Extension**：在 Cluster 2 中是 1.55x，在 Cluster 1 中只有 0.45x——同样是 Cluster 2 的标志。
3. **Age**：在 Cluster 1 中是 1.10x，在 Cluster 2 中只有 0.74x——Age 在 Cluster 1 中更重要。
4. **Raca.Color**：在 Cluster 1 中是 1.06x，在 Cluster 2 中只有 0.78x——Raca.Color 在 Cluster 1 中更重要。

### 3.2 两个簇的"对立模式"

```
Cluster 1 的"旗帜": Age (1.10x), Raca.Color (1.06x), year (1.04x)
  → 模型用"年龄、种族、年份"这些常规特征预测
  → 这是"主流模式"

Cluster 2 的"旗帜": Diagnostic.means (1.83x), Extension (1.55x)
  → 模型用"诊断方式、肿瘤扩展"这些特殊特征预测
  → 这是"特殊模式"
```

两个簇的"旗帜特征"几乎不重叠——Cluster 1 看重 Age/Raca.Color/year，Cluster 2 看重 Diagnostic.means/Extension。这就是"决策模式群组"的体现。

> 💡 **小贴士：相对重要性 vs 绝对重要性**
>
> **绝对重要性**（如 Cluster 2 的 year = 0.350）回答"该特征有多重要"。
>
> **相对重要性**（如 Cluster 2 的 year = 1.30x）回答"该特征比平时更重要还是更不重要"。
>
> 相对重要性更能揭示"亚群特异性"——某个特征可能在所有簇中都很重要（绝对值高），但只在某个簇中"异常突出"（相对值高）。
>
> 例如 year 在两个簇中都是最重要的特征（绝对值最高），但它在 Cluster 2 中的相对重要性只有 1.30x，远不如 Diagnostic.means 的 1.83x"异常"。

## 五、相对重要性的创新价值

### 5.1 传统方法的局限

传统分析只看两种重要性：

1. **全局重要性**：`np.abs(sv).mean(0)` — 所有样本的平均绝对 SHAP 值。
2. **簇内重要性**：`np.abs(sv[mask]).mean(0)` — 某个簇内的平均绝对 SHAP 值。

但这两种方法都无法回答："这个特征在该簇中是否**异常突出**？"

### 5.2 相对重要性的创新

相对重要性 `rel_imp = this_imp / global_imp` 直接回答这个问题：

| rel\_imp | 含义               | 解读                |
| -------- | ---------------- | ----------------- |
| > 1.0    | 该特征在该簇中比全局更重要    | **异常突出**，是该簇的"旗帜" |
| = 1.0    | 该特征在该簇中的重要性与全局相同 | 无特殊               |
| < 1.0    | 该特征在该簇中比全局更不重要   | **被弱化**           |

### 5.3 相对重要性的教学价值

```
相对重要性的教学价值:

> 1.0x: 该特征在此聚类中比全局平均更重要
< 1.0x: 该特征在此聚类中比全局平均更不重要

Cluster 2 的"旗帜": Diagnostic.means (1.83x)
  → "如果你属于 Cluster 2, 诊断方式对你的预测影响
     是全局平均的 1.83 倍"
```

这种表述比"Diagnostic.means 在 Cluster 2 中的重要性是 0.112"更直观——它直接告诉读者"这个特征在该亚群中有多特殊"。

## 六、SHAP 聚类的局限性

### 6.1 局限性表

| 局限性           | 原因                 | 缓解                  |
| ------------- | ------------------ | ------------------- |
| K-Means 假设球形簇 | SHAP 空间可能不是球形      | 使用 DBSCAN 或层次聚类交叉验证 |
| 聚类标签无临床含义     | "Cluster 1" 不是诊断代码 | 通过特征画像赋予临床解释        |
| 聚类结果依赖模型      | 不同模型产生不同 SHAP 聚类   | 在多个模型上验证聚类稳定性       |
| 小样本下的聚类不稳定    | SHAP 值随训练数据变化      | 增加样本量或使用自助法         |

### 6.2 应对策略

1. **多算法验证**：用 K-Means、层次聚类、DBSCAN 分别聚类，看结果是否一致。
2. **多模型验证**：用 RF、XGBoost、LightGBM 分别训练模型，看 SHAP 聚类是否稳定。
3. **Bootstrap 验证**：重采样 100 次，看聚类标签的稳定性。
4. **外部验证**：在独立数据集上验证聚类特征模式是否一致。

***

## 七、聚类分析的全流程建议

```
SHAP 聚类 → 临床验证 → 独立验证

第一步: SHAP 聚类 (本教程)
  在训练/测试数据上执行, 发现 2-4 个聚类

第二步: 聚类特征定义
  用 PCA 权重或线性判别分析
  找到哪些特征区分了聚类
  → 为新患者分配聚类标签

第三步: 独立外部验证
  在另一数据集中, 用同样的规则分配聚类
  验证: 聚类间的 AUC/特征模式是否一致

第四步: 临床转化
  如果聚类有明确的临床特征画像
  → 可以开发针对不同聚类的"定制化"解释
```

## 八、其他聚类算法对比

| 算法                | 优点            | 缺点     | 对 SHAP 聚类的适用性 |
| ----------------- | ------------- | ------ | ------------- |
| **K-Means** (本教程) | 快速、直观         | 球形簇假设  | 对 SHAP 空间足够   |
| DBSCAN            | 发现任意形状、自动检测噪点 | 对密度差敏感 | 适合发现异常决策模式    |
| 层次聚类              | 提供完整的层次结构     | 计算成本高  | 适合探索性阶段       |
| GMM (高斯混合)        | 软分类、概率输出      | 需要大量样本 | 适合不确定分类       |

> 💡 **小贴士：什么时候换算法？**
>
> - 如果 Silhouette Score 很低（< 0.3），可以试 DBSCAN，看是否有任意形状的簇。
> - 如果想看聚类的层次结构（如"Cluster 2 是否能再细分"），用层次聚类画树状图。
> - 如果想要"软分类"（每个样本属于每个簇的概率），用 GMM。
>
> 本教程选择 K-Means 是因为 SHAP 空间结构清晰（Silhouette=0.6263），K-Means 足够。

## 常见问题

### Q1: 为什么 Cluster 2 的 VIVO 比例这么高（18.4% vs Cluster 1 的 1.4%）？

**A**: 这是因为 Cluster 2 的患者特征不同：

- year = +1.32（近年诊断）——医疗技术进步，存活率提高。
- Diagnostic.means 重要性 1.83x——诊断方式对预后影响大，可能这些患者接受了更准确的诊断（如早期活检）。
- Extension 重要性 1.55x——肿瘤扩展程度对预后影响大，可能这些患者肿瘤较局限。

综合来看，Cluster 2 是"近年诊断、诊断方式特殊、肿瘤较局限"的患者，所以存活率更高。

### Q2: 相对重要性 1.83x 是什么意思？

**A**: 意思是 Diagnostic.means 这个特征在 Cluster 2 中的平均绝对 SHAP 值，是全局平均绝对 SHAP 值的 1.83 倍。换句话说，对于 Cluster 2 的患者，模型更"依赖"诊断方式来做预测——它对预测的影响比平时大 83%。

### Q3: 为什么 Cluster 1 的所有相对重要性都接近 1.0x？

**A**: 因为 Cluster 1 是"主流模式"，占 79.4% 的样本。全局重要性主要由 Cluster 1 决定（因为它占大多数），所以 Cluster 1 的重要性自然接近全局重要性，相对重要性接近 1.0x。

这也说明相对重要性对"少数簇"更有意义——它能突出少数簇与主流的差异。

### Q4: 聚类标签（Cluster 1, Cluster 2）有临床含义吗？

**A**: 没有。聚类标签是 K-Means 随机分配的，每次运行可能不同。我们需要通过特征画像（相对重要性、特征均值、VIVO 比例）赋予临床含义。

例如，本教程中 Cluster 2 被解读为"诊断方式驱动的特殊模式"，但这个解读是基于特征画像得出的，不是标签本身。

### Q5: 如果换一个模型（如 XGBoost），聚类结果会变吗？

**A**: 可能会变。不同模型有不同的决策逻辑，产生不同的 SHAP 值，聚类结果自然不同。这就是 SHAP 聚类的局限之一——"聚类结果依赖模型"。

应对策略：在多个模型上做 SHAP 聚类，看结果是否一致。如果一致，说明聚类结构是"数据驱动的"；如果不一致，说明聚类结构是"模型驱动的"。（见教学文档思考题 6）

 
 

## 整体教程总结

### 核心收获

1. **SHAP 空间有清晰的聚类结构**（Silhouette=0.6263），最佳聚类数 = 2。
2. **Cluster 1（79.4%）** = "主流模式"——year 和 Age 驱动，几乎都预测死亡（VIVO=1.4%）。
3. **Cluster 2（20.6%）** = "特殊模式"——Diagnostic.means（1.83x）和 Extension（1.55x）异常突出，存活率 18.4% 是 Cluster 1 的 13 倍。
4. **PCA 仅 2 个主成分就解释了 92.32% 的 SHAP 方差**——说明 SHAP 空间本质上是低维的。
5. **对 SHAP 值聚类 vs 对原始特征聚类**——这是"解释模式"聚类 vs "表型"聚类的根本区别。
