# 模块 4：离群值分析

> **核心思想**：离群值（Outlier）是数据集中明显偏离其他观测值的数据点。它们可能是真实的极端现象，也可能是测量或录入错误。识别离群值是 EDA 中至关重要的一步——既不能盲目删除（可能丢失重要信息），也不能视而不见（可能污染模型）。

本模块将以巴西癌症登记数据为例，介绍两种最常用的离群值检测方法：**IQR（四分位距）方法** 和 **Z-score（标准分）方法**，并通过可视化对比它们的差异。

## 2. IQR 方法（四分位距法）

### 2.1 原理详解

IQR（InterQuartile Range，四分位距）方法基于数据的**分位数**，是一种非参数方法，不依赖任何分布假设。

#### 2.1.1 关键概念：Q1、Q3、IQR

- **Q1（第一四分位数，第 25 百分位数）**：将数据从小到大排序后，有 25% 的数据小于等于 Q1。
- **Q3（第三四分位数，第 75 百分位数）**：将数据从小到大排序后，有 75% 的数据小于等于 Q3。
- **IQR（四分位距）**：`IQR = Q3 - Q1`，表示数据中间 50% 的跨度。

> **重要概念**：IQR 描述了数据"主体部分"的离散程度。与极差（max - min）不同，IQR 不受极端值影响，因为 Q1 和 Q3 都是分位数，极端值无论多大或多小，都不会改变它们的位置。

#### 2.1.2 代码实现

```python
q1 = data.quantile(0.25)
q3 = data.quantile(0.75)
iqr = q3 - q1
lower_iqr = q1 - 1.5 * iqr
upper_iqr = q3 + 1.5 * iqr
```

**参数解释**：

- `data.quantile(0.25)`：计算第 25 百分位数（即 Q1）。参数 `0.25` 表示"有 25% 的数据小于此值"。
- `data.quantile(0.75)`：计算第 75 百分位数（即 Q3）。参数 `0.75` 表示"有 75% 的数据小于此值"。
- `iqr = q3 - q1`：四分位距，中间 50% 数据的范围。
- `lower_iqr = q1 - 1.5 * iqr`：离群值下限。
- `upper_iqr = q3 + 1.5 * iqr`：离群值上限。

#### 2.1.3 离群值判定

```python
iqr_outliers = data[(data < lower_iqr) | (data > upper_iqr)]
iqr_pct = len(iqr_outliers) / len(data) * 100
```

**布尔运算详解**：

- `data < lower_iqr`：返回一个布尔 Series，标记每个值是否小于下限。
- `data > upper_iqr`：返回一个布尔 Series，标记每个值是否大于上限。
- `|`：**按位或（bitwise OR）**运算符，对两个布尔 Series 逐元素取"或"。注意：这里**必须用 `|` 而不能用 `or`**，因为 `or` 是 Python 的逻辑运算符，无法对 pandas Series 进行逐元素操作。

> **常见问题**：为什么写成 `(data < lower_iqr) | (data > upper_iqr)` 而不是 `data < lower_iqr | data > upper_iqr`？
>
> 因为 `|` 的优先级高于比较运算符 `<` 和 `>`。如果不加括号，Python 会先计算 `lower_iqr | data`（这会报错），再计算比较。**必须用括号明确优先级**。

#### 2.1.4 为什么是 1.5 倍？

> **重要概念**：1.5 这个系数并非随意选取，而是有统计学依据的。

在**正态分布**假设下：

- Q1 ≈ μ - 0.6745σ
- Q3 ≈ μ + 0.6745σ
- IQR = Q3 - Q1 ≈ 1.349σ
- 下限 = Q1 - 1.5×IQR ≈ μ - 0.6745σ - 1.5×1.349σ ≈ μ - 2.698σ
- 上限 = Q3 + 1.5×IQR ≈ μ + 2.698σ

也就是说，**1.5×IQR 界限大约对应 ±2.7σ**。在正态分布下，超出 ±2.7σ 的概率约为 0.7%，这正好是一个"罕见但不至于过严"的阈值。

- 如果用 1.0×IQR：界限 ≈ ±2.0σ，约 4.6% 数据会被标记，太宽松。
- 如果用 3.0×IQR：界限 ≈ ±4.7σ，几乎不会标记任何数据，太严格。
- **1.5×IQR 是 John Tukey（箱线图发明者）提出的经验值**，在大多数场景下表现良好。

> **小贴士**：对于特别严格的场景，Tukey 还提出了 "far out" 离群值的概念，使用 3×IQR 作为界限，对应正态分布下约 ±4.7σ。

## 3. Z-score 方法（标准分法）

### 3.1 原理详解

Z-score（标准分）方法基于数据的**均值和标准差**，是一种参数方法，隐含假设数据近似服从正态分布。

#### 3.1.1 Z-score 公式

$$Z = \frac{x - \mu}{\sigma}$$

其中：
- $x$ 是单个观测值
- $\mu$ 是数据的均值（mean）
- $\sigma$ 是数据的标准差（std）

**Z-score 的含义**：表示一个观测值偏离均值多少个标准差。

- Z = 0：观测值等于均值
- Z = 1：观测值比均值高 1 个标准差
- Z = -2：观测值比均值低 2 个标准差
- |Z| > 3：观测值偏离均值超过 3 个标准差，通常被认为是离群值

> **重要概念**：Z-score 本质上是一种**标准化变换**（standardization）。变换后的数据均值为 0、标准差为 1，因此不同量纲的特征可以通过 Z-score 进行横向比较。

#### 3.1.2 代码实现

```python
z_scores = np.abs((data - data.mean()) / data.std())
z_outliers = data[z_scores > 3]
z_pct = len(z_outliers) / len(data) * 100

# Mild outliers (Z-score > 2)
mild_z_outliers = data[z_scores > 2]
mild_z_pct = len(mild_z_outliers) / len(data) * 100
```

**代码详解**：

- `data - data.mean()`：每个值减去均值（中心化）。
- `(... ) / data.std()`：再除以标准差（缩放）。这两步合起来就是标准化。
- `np.abs(...)`：取绝对值。因为我们关心的是"偏离均值多远"，而不是方向（正偏离还是负偏离）。`np.abs` 是 NumPy 的函数，会对数组逐元素取绝对值。
- `z_scores > 3`：返回布尔 Series，标记 |Z| > 3 的点。
- `data[z_scores > 3]`：用布尔索引筛选出离群值。

#### 3.1.3 阈值的含义：3σ 原则

> **重要概念（3σ 原则）**：在正态分布下：
> - 约 **68.27%** 的数据落在 ±1σ 内
> - 约 **95.45%** 的数据落在 ±2σ 内
> - 约 **99.73%** 的数据落在 ±3σ 内
>
> 也就是说，|Z| > 3 的概率仅约 0.27%，是非常罕见的事件，因此常被用作离群值阈值。

本教程计算了两个阈值：

| 阈值 | 含义 | 正态分布下理论比例 | 用途 |
|------|------|-------------------|------|
| \|Z\| > 2 | Mild outlier（温和离群值） | 约 4.55% | 提示可能异常 |
| \|Z\| > 3 | Extreme outlier（极端离群值） | 约 0.27% | 强烈提示异常 |

#### 3.1.4 为什么 Z-score 对极端值敏感？

> **重要概念（Z-score 的致命弱点）**：Z-score 公式中用到的 μ 和 σ **本身就会被极端值拉偏**。

举个例子：假设有 100 个数据点，99 个在 [0, 10] 之间，1 个是 1000。

- 没有极端值时：μ ≈ 5，σ ≈ 3
- 有极端值时：μ ≈ 15，σ ≈ 100

可以看到，**单个极端值就让 μ 和 σ 发生了巨大变化**。这导致：

1. μ 被拉向极端值方向
2. σ 被显著放大
3. 其他正常点的 Z-score 反而变小了
4. 极端值本身的 Z-score 也被"稀释"，可能不再超过 3

这就是为什么在**偏态分布**或**有极端值**的数据上，Z-score 方法的效果会大打折扣。而 IQR 方法基于分位数，**不受极端值影响**，因此在这样的场景下更稳健。

> **小贴士**：为了克服 Z-score 的这个弱点，可以使用**修正 Z-score（Modified Z-score）**，用中位数和 MAD（Median Absolute Deviation）替代均值和标准差。公式为：$M_i = 0.6745 \times \frac{x - \text{median}}{\text{MAD}}$。

## 4. 两种方法对比

### 4.1 优缺点总结

| 维度 | IQR 方法 | Z-score 方法 |
|------|---------|--------------|
| **是否参数方法** | 非参数（基于分位数） | 参数（基于均值和标准差） |
| **分布假设** | 无假设，适用于任何分布 | 假设近似正态分布 |
| **对极端值的稳健性** | **稳健**（分位数不受极端值影响） | **不稳健**（μ 和 σ 会被拉偏） |
| **适用分布类型** | 偏态分布、重尾分布、未知分布 | 近似正态分布 |
| **计算复杂度** | 低（只需排序求分位数） | 低（只需均值和标准差） |
| **对样本量的要求** | 小样本时 IQR 估计不稳定 | 小样本时 σ 估计不稳定 |
| **可解释性** | 直观（基于箱线图） | 直观（基于标准差倍数） |
| **阈值含义** | 1.5×IQR ≈ ±2.7σ（正态下） | \|Z\|>3 对应 0.27% 概率 |

### 4.2 选择建议

> **小贴士**：
> - 如果数据**明显偏态**或**有重尾**：优先用 **IQR**。
> - 如果数据**近似正态**且样本量大：**两者都可以**，Z-score 更精细。
> - 如果**不确定**：**两种方法都算一遍**，对比结果。本教程就是这么做的。
> - 如果数据**严重非正态**且想用 Z-score 思路：考虑**修正 Z-score**（基于中位数和 MAD）。

```python

**`boxplot` 参数详解**：

|------|-----|------|

**`bp` 返回值详解**：

- `bp['boxes']`：箱体（Q1 到 Q3 的矩形）
- `bp['whiskers']`：须线（从箱体延伸到 1.5×IQR 界限）
- `bp['caps']`：须线末端的横线
- `bp['medians']`：中位数线
- `bp['fliers']`：离群值点（超出须线范围的点）

> **重要概念（箱线图各组件含义）**：
> - **箱体下边**：Q1（第 25 百分位数）
> - **箱体上边**：Q3（第 75 百分位数）
> - **箱体高度**：IQR = Q3 - Q1
> - **箱体内横线**：中位数（median）
> - **下须线**：从 Q1 向下延伸到 `min(数据, Q1 - 1.5×IQR)` 范围内的最小值
> - **上须线**：从 Q3 向上延伸到 `max(数据, Q3 + 1.5×IQR)` 范围内的最大值
> - **须线外的点**：离群值（fliers），即超出 1.5×IQR 界限的点

fig, ax = plt.subplots(figsize=(10, 6))
plot_cols = [c for c in numeric_features if c in df.columns][:5]  # 最多5个
plot_data = df[plot_cols].dropna()
if len(plot_data) > 30000:
    plot_data = plot_data.sample(30000, random_state=42)
```

- `[:5]`：最多取 5 个特征，避免图过于拥挤。
- `dropna()`：删除任意特征缺失的行。注意：这里要求**所有选定特征都非缺失**，所以样本量会减少。
- `sample(30000, random_state=42)`：同样为了可视化性能进行采样。

#### 6.1.2 宽表转长表（melt）

```python
plot_data_melted = plot_data.melt(var_name='Feature', value_name='Value')
```

> **重要概念（宽表 vs 长表）**：
>
> **宽表（Wide Format）**：每个特征是一列，每行是一个样本。
>
> | Index | Age | year | Code.of.Morphology |
> |-------|-----|------|---------------------|
> | 0     | 65  | 2010 | 80003               |
> | 1     | 72  | 2012 | 81403               |
>
> **长表（Long Format）**：所有特征值堆叠在同一个 `Value` 列，用 `Feature` 列区分来源。
>
> | Feature | Value |
> |---------|-------|
> | Age | 65 |
> | Age | 72 |
> | year | 2010 |
> | year | 2012 |
> | Code.of.Morphology | 80003 |
> | Code.of.Morphology | 81403 |

**`melt` 参数详解**：

- `var_name='Feature'`：原列名变成新列的列名（即"特征名"列）。
- `value_name='Value'`：原值变成新列的列名（即"数值"列）。

> **小贴士**：为什么 seaborn 需要"长表"？
> 因为 seaborn 的 `boxplot` 默认按某个分类变量（`x`）对数值（`y`）分组。长表格式下，`x='Feature'` 自然就把不同特征分开画了。这是 "tidy data"（整洁数据）理念在可视化中的体现。

#### 6.1.3 seaborn 箱线图

```python
sns.boxplot(data=plot_data_melted, x='Feature', y='Value',
            hue='Feature', palette='Set2', ax=ax, legend=False)
```

**`sns.boxplot` 参数详解**：

| 参数 | 值 | 含义 |
|------|-----|------|
| `data` | `plot_data_melted` | 长表格式的 DataFrame |
| `x` | `'Feature'` | x 轴变量（分类变量，决定箱体的分组） |
| `y` | `'Value'` | y 轴变量（数值变量，决定箱体的高度分布） |
| `hue` | `'Feature'` | **分组变量**，决定颜色。这里用 `Feature` 本身，让每个特征有不同颜色 |
| `palette` | `'Set2'` | 调色板名称。`Set2` 是 ColorBrewer 的一个柔和配色 |
| `ax` | `ax` | 指定绘制到哪个 Axes 对象 |
| `legend` | `False` | 是否显示图例。这里设为 `False`，因为 x 轴标签已经标明了特征名，图例多余 |

> **重要概念（seaborn 0.13+ 的 `hue` 用法变化）**：
>
> 在 **seaborn 0.13 之前**，如果想让每个箱体有不同颜色，可以直接用 `palette` 参数，不需要 `hue`。
>
> 在 **seaborn 0.13+**，官方推荐**显式使用 `hue` 参数**来分组着色，否则会触发 `FutureWarning`。同时，使用 `hue` 后会自动生成图例，如果不需要图例，要用 `legend=False` 关闭。
>
> 这是因为 seaborn 在统一其 API：**所有基于分组的着色都通过 `hue` 控制**，使行为更一致、可预测。

> **常见问题**：为什么 `hue='Feature'` 和 `x='Feature'` 用了同一个变量？
>
> 这是一种常见技巧：当 `x` 是分类变量且我们想给每个类别上不同颜色时，把 `hue` 设为同一个变量即可。这样每个箱体会有自己的颜色，但不会产生额外的分组（因为分组维度和 x 维度一致）。

## 8. 讨论：医学异常值是否代表错误数据？

> **重要概念**：离群值 ≠ 错误值。在医学数据中，"异常"往往有临床意义，不能简单地删除。

### 8.1 医学异常值的三种可能

#### 可能性 1：真实的极端病例

医学数据天然存在极端值。例如：

- **极高龄患者**：100 岁以上的癌症患者虽然罕见，但确实存在。
- **罕见病理类型**：某些形态学编码对应的癌症类型本身就很少见。
- **早发型癌症**：20 岁以下的癌症患者虽少，但在儿童肿瘤学中是重要研究对象。

这些"离群值"是**真实的临床信号**，删除它们会丢失重要信息。

#### 可能性 2：特殊人群

某些离群值代表**有研究价值的特殊群体**：

- **少儿癌症患者**（age=0 可能代表新生儿或婴幼儿）：儿童癌症的病因、治疗、预后与成人截然不同，是独立的研究领域。
- **特定职业群体**：某些职业编码可能对应高风险行业（如化工、辐射相关），这些群体值得专门研究。
- **罕见年份的记录**：早期数据可能反映历史诊疗水平，晚期数据可能反映新诊疗技术的引入。

#### 可能性 3：数据录入错误

当然，部分离群值确实是**数据质量问题**：

- **Age 录入错误**：如把 65 岁误录为 655 或 6.5。
- **编码错位**：如把形态学编码填到了职业编码字段。
- **单位错误**：如把月份当成年份。
- **缺失值填充错误**：如用 0 或 999 填充缺失值。

### 8.2 如何判断？

> **小贴士**：判断医学离群值是否为错误，可以遵循以下流程：
>
> 1. **检查取值范围**：Age 是否在 [0, 120] 内？year 是否在合理范围内？
> 2. **结合领域知识**：100 岁癌症患者合理吗？（合理）655 岁合理吗？（不合理）
> 3. **交叉验证**：如果 Age=0，是否有对应的儿童医院记录？如果 Age=100，是否有完整诊疗记录？
> 4. **查看原始数据**：检查相邻字段是否有错位痕迹。
> 5. **与数据提供方确认**：对于无法判断的极端值，咨询数据来源机构。

### 8.3 结论

> **结论**：离群值 ≠ 错误值，需结合临床背景判断。在医学 EDA 中，**保留离群值并记录**通常比直接删除更稳妥。如果决定处理，应该：
> - 记录处理理由（为什么删？删了多少？）
> - 对比处理前后的模型表现
> - 在报告中透明披露

## 9. 讨论：Age 的 IQR 下限是 21 岁合理吗？

### 9.1 问题陈述

从统计结果看，Age 的 IQR 下限是 **21 岁**。这意味着 **21 岁以下的癌症患者被 IQR 方法标记为"离群值"**。这合理吗？

### 9.2 统计学角度

从纯统计学角度，这是**合理的**。因为：

- 癌症确实是**老年高发**疾病，绝大多数患者年龄在 50 岁以上。
- 21 岁以下的患者在统计学上确实属于"少数尾部"，被分位数方法捕获是正常的。
- IQR 方法只关心"是否偏离主体分布"，不关心偏离的原因。

### 9.3 临床角度

但从临床角度，这是**不合理的**。因为：

- **儿童和青少年癌症是一个独立的医学领域**（Pediatric Oncology），有专门的诊疗指南。
- 把儿童癌症患者标记为"离群值"暗示他们是"异常"，这在伦理和科学上都有问题。
- 这些患者不仅不是"错误"，反而是**特别需要关注的研究对象**。

### 9.4 启示

> **重要概念**：统计方法给出的是"统计离群值"，而非"临床离群值"。两者不能混为一谈。

这个例子生动地说明：

1. **统计工具是盲目的**：IQR 不知道"21 岁"在医学上有特殊含义。
2. **领域知识是必要的**：只有结合医学知识，才能正确解读统计结果。
3. **EDA 的目标是理解数据，而非机械地清洗数据**：发现 21 岁以下患者被标记为离群值，应该引发思考"为什么？"，而不是"删掉它们"。

> **小贴士**：在实际项目中，遇到这种"统计离群值"与"领域知识"冲突时，推荐做法是：
> - **保留这些数据**，但在分析中单独标注。
> - 可以做**敏感性分析**：分别用包含/不包含这些"离群值"的数据建模，看结论是否稳定。
> - 在论文或报告中**明确讨论**这一点，体现研究的严谨性。

## 10. 常见问题（FAQ）

### Q1：为什么我的 IQR 离群值比例和 Z-score 离群值比例差很多？

**A**：这通常是因为数据**偏离正态分布**。在正态分布下，IQR 界限 ≈ ±2.7σ，介于 Z>2 和 Z>3 之间。如果数据偏态或重尾，两种方法的差异会放大。建议同时看两种方法的结果，并检查数据的分布形态（直方图、QQ 图）。

### Q2：离群值应该删除吗？

**A**：**不要盲目删除**。删除前要回答三个问题：
1. 这个离群值是错误吗？（如果是，删）
2. 删除后分析结论会变吗？（做敏感性分析）
3. 删除的理由能向同行解释清楚吗？（如果不能，别删）

### Q3：`|` 和 `or` 有什么区别？

**A**：
- `|` 是**按位或**运算符，可以对 pandas Series / NumPy 数组进行**逐元素**运算，返回布尔数组。
- `or` 是 Python 的**逻辑或**运算符，只能对单个布尔值运算，作用于 Series 时会报错（`ValueError: The truth value of a Series is ambiguous`）。
- 类似地，`&` 对应 `and`，`~` 对应 `not`。在 pandas 中过滤数据时，**必须用 `&`、`|`、`~`**，且**要用括号分组**。

### Q4：为什么箱线图的"须线"不等于 Q1-1.5×IQR？

**A**：这是一个常见误解。箱线图的须线**不是**直接画在 Q1-1.5×IQR 处，而是画在**实际数据中不超过该界限的最值**处。也就是说：
- 下须线 = `max(min(data), Q1 - 1.5×IQR)` 范围内的最小**实际数据值**
- 上须线 = `min(max(data), Q3 + 1.5×IQR)` 范围内的最大**实际数据值**

所以须线总是落在某个真实数据点上，而不是界限本身。界限只是用来**判定**哪些点是 fliers。

### Q5：seaborn 报 `FutureWarning: Setting a gradient palette...` 怎么办？

**A**：这是 seaborn 0.13+ 的提示。解决方法是显式使用 `hue` 参数，并设置 `legend=False`（如果不需要图例）。本教程的代码就是这么写的：
```python
sns.boxplot(data=..., x='Feature', y='Value', hue='Feature', palette='Set2', legend=False)
```

### Q6：为什么 `Code.of.Morphology` 的离群值比例高达 18.67%？

**A**：因为它是**分类编码**而非连续变量。形态学编码是 ICD-O-3 体系的标识码（如 80003 代表"上皮细胞肿瘤"），数字大小没有数值意义。把它当连续变量算 IQR 和 Z-score 本身就是误用。正确做法是用 `value_counts()` 看频数分布，或将其作为分类变量处理。

## 11. 模块总结

### 本模块学到的知识点

#### 理论知识

1. **IQR 方法**：
   - Q1（第 25 百分位数）、Q3（第 75 百分位数）、IQR = Q3 - Q1 的含义
   - 离群值界限：Q1 - 1.5×IQR 和 Q3 + 1.5×IQR
   - 1.5 倍系数的统计学依据（正态分布下约对应 ±2.7σ）
   - IQR 方法是**非参数**方法，对极端值**稳健**

2. **Z-score 方法**：
   - 公式 Z = (x - μ) / σ 的含义（标准化变换）
   - 3σ 原则（正态分布下 99.7% 数据在 ±3σ 内）
   - Z > 3（极端离群值）和 Z > 2（温和离群值）的阈值含义
   - Z-score 方法的**致命弱点**：μ 和 σ 会被极端值拉偏

3. **两种方法对比**：
   - IQR 适用于偏态分布、重尾分布、未知分布
   - Z-score 适用于近似正态分布
   - 不确定时**两种都算一遍**，对比结果

#### 代码技能

4. **pandas 操作**：
   - `data.quantile(0.25)` / `data.quantile(0.75)` 计算分位数
   - 布尔索引 `(data < lower) | (data > upper)`，注意 `|` 是按位或，**必须用括号分组**
   - `df.melt(var_name=..., value_name=...)` 宽表转长表

5. **NumPy 操作**：
   - `np.abs(...)` 取绝对值
   - 标准化变换 `(data - data.mean()) / data.std()`

6. **matplotlib 可视化**：
   - `plt.subplots(1, 2, figsize=...)` 创建多子图画布
   - `axes[0].boxplot(data, vert=True, patch_artist=True, widths=0.3)` 绘制箱线图
   - `axes[0].axhline(y=..., color=..., linestyle='--', linewidth=...)` 画水平参考线
   - `axes[1].scatter(x, y, c=..., cmap='viridis', alpha=0.5, s=1)` 绘制带颜色映射的散点图
   - `plt.colorbar(mappable, ax=..., label=...)` 添加颜色条

7. **seaborn 可视化**：
   - `sns.boxplot(data=..., x=..., y=..., hue=..., palette=..., ax=..., legend=False)`
   - seaborn 0.13+ 的 `hue` 用法变化及 `legend=False` 的作用

#### 思维方法

8. **批判性思考**：
   - 离群值 ≠ 错误值，需结合领域知识判断
   - 医学异常值的三种可能：真实极端病例、特殊人群、录入错误
   - 统计离群值与临床离群值的区别（Age 21 岁下限的例子）
   - "数值型"不等于"连续型"，编码类变量不应做离群值分析

9. **最佳实践**：
   - 大数据集可视化前先采样（`sample(50000, random_state=42)`）
   - 不确定分布时多种方法并用
   - 保留离群值并记录处理理由，而非盲目删除
   - 做敏感性分析，对比处理前后的结论稳定性

### 下一模块预告

在下一个模块中，我们将进入**统计检验**的世界，学习如何用 p 值、效应量等方法量化特征与目标变量之间的关联，为特征选择打下基础。

---

> **最后提醒**：离群值分析是 EDA 中"发现问题"的工具，而非"解决问题"的工具。发现离群值后，**理解它们为什么存在**比"如何处理它们"更重要。保持好奇心和批判性思维，是数据科学家的核心素养。
