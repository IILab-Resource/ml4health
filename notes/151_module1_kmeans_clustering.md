# 模块 1：K-Means 聚类与最佳聚类数选择

> 本模块是案例教程 15「SHAP 聚类分析」的**核心模块之一**。在模块 0 中，我们已经训练好随机森林模型（AUC=0.8800），并用 TreeExplainer 在 3000 个测试样本上计算了 SHAP 值矩阵 `sv`，形状为 `(3000, 6)`。本模块将把这个 SHAP 矩阵作为输入，用 **K-Means 算法**对样本进行聚类，目标是发现"模型解释模式相似"的患者群组。 
>
> 本模块最核心的知识点有三个：**一是 K-Means 算法在 SHAP 空间的工作原理**——它把"SHAP 值模式相似"的样本归为一类，等价于"模型决策逻辑相似"；**二是 Silhouette Score（轮廓系数）的含义和计算**——它综合衡量聚类的"紧密度"和"分离度"，是选择最佳聚类数的标准指标；**三是为什么本数据集最佳聚类数是 2**——因为 SHAP 空间本质上只有两种决策模式，强行增加聚类数只会把其中一种模式再细分。

***

学完本模块后，你将能够：

## 二、K-Means 聚类：核心代码

### 2.1 打印模块标题

```python
print("\n" + "=" * 70)
print("[2] K-Means 聚类 (基于 SHAP 值)")
print("=" * 70)
```

- **`"\n" + "=" * 70`**：先输出一个空行，再输出 70 个 `=` 作为视觉分隔。
- **`print("[2] K-Means 聚类 (基于 SHAP 值)")`**：标注本模块的编号和主题。注意括号中明确写了"基于 SHAP 值"，强调聚类的输入是 SHAP 值，不是原始特征。

### 2.2 初始化最佳聚类数搜索

```python
# 确定最佳聚类数: 2~5
best_n = 3  # 默认
best_score = -1
silhouette_scores = {}
```

#### 逐行解释

- **`best_n = 3`**：默认聚类数为 3。这个值只是占位，后续会被搜索结果覆盖。
- **`best_score = -1`**：初始最佳分数设为 -1（Silhouette Score 范围是 \[-1, 1]，-1 是最低可能值，保证任何实际分数都会更新它）。
- **`silhouette_scores = {}`**：空字典，用于存储每个聚类数对应的 Silhouette Score。键是聚类数 n，值是分数。

> 💡 **小贴士：为什么搜索范围是 2\~5？**
>
> - **下界 2**：Silhouette Score 至少需要 2 个簇才能计算（要比较"自身簇"和"最近的其他簇"）。
> - **上界 5**：医学数据中亚群数量通常不多，2\~5 个簇已经足够覆盖常见情况。如果数据真有 6+ 个亚群，通常会先用层次聚类探索。
>
> 这个范围是经验性的，你可以根据具体问题调整。例如，如果你怀疑数据有 10 个亚群，可以把上界改为 10。

### 2.3 遍历聚类数计算 Silhouette Score

```python
for n in range(2, 6):
    km = KMeans(n_clusters=n, random_state=RANDOM_STATE, n_init=10)
    labels = km.fit_predict(sv)
    score = silhouette_score(sv, labels)
    silhouette_scores[n] = score
    print(f"    n_clusters={n}:  Silhouette Score = {score:.4f}")
```

#### 逐行解释

- **`for n in range(2, 6):`**：n 取 2, 3, 4, 5（`range` 是左闭右开区间）。
- **`km = KMeans(n_clusters=n, random_state=RANDOM_STATE, n_init=10)`**：创建 KMeans 实例。
  - **`n_clusters=n`**：聚类数，从 2 到 5。
  - **`random_state=RANDOM_STATE`**：随机种子，保证初始中心点选择可复现。K-Means 对初始中心点敏感，不同种子可能得到不同结果。
  - **`n_init=10`**：运行 10 次独立的 K-Means（每次随机初始化），取 inertia（簇内平方和）最小的那次作为最终结果。这能避免陷入局部最优。
- **`labels = km.fit_predict(sv)`**：一次性完成训练和预测。
  - `fit(sv)`：在 SHAP 矩阵 `sv` 上学习 K 个簇中心。
  - `predict(sv)`：把每个样本分配到最近的簇中心。
  - `fit_predict` 等价于 `fit` + `predict`，但更简洁。
  - 返回值 `labels` 是 `(3000,)` 数组，每个元素是 0, 1, ..., n-1 的簇编号。
- **`score = silhouette_score(sv, labels)`**：计算 Silhouette Score。
  - 第一个参数 `sv`：原始数据（SHAP 值矩阵）。
  - 第二个参数 `labels`：聚类标签。
  - 返回值 `score`：所有样本的平均 Silhouette Score，范围 \[-1, 1]。
- **`silhouette_scores[n] = score`**：把分数存入字典，键是聚类数 n，值是分数。
- **`print(f"    n_clusters={n}:  Silhouette Score = {score:.4f}")`**：打印每个 n 的分数，保留 4 位小数。

#### 实际运行结果

```
[2] K-Means 聚类 (基于 SHAP 值)
==============================================================================
    n_clusters=2:  Silhouette Score = 0.6263
    n_clusters=3:  Silhouette Score = 0.5936
    n_clusters=4:  Silhouette Score = 0.5190
    n_clusters=5:  Silhouette Score = 0.5639
```

### 2.4 选择最佳聚类数

```python
best_n = max(silhouette_scores, key=silhouette_scores.get)
print(f"\n    → 最佳聚类数: {best_n} (Silhouette Score={silhouette_scores[best_n]:.4f})")
```

#### 逐行解释

- **`max(silhouette_scores, key=silhouette_scores.get)`**：
  - `silhouette_scores` 是字典 `{2: 0.6263, 3: 0.5936, 4: 0.5190, 5: 0.5639}`。
  - `key=silhouette_scores.get`：用字典的值（分数）作为比较依据。
  - `max` 返回值最大的键，即 `2`（分数 0.6263 最高）。
- **`best_n = 2`**：最佳聚类数确定为 2。
- **`print(f"\n    → 最佳聚类数: {best_n} (Silhouette Score={silhouette_scores[best_n]:.4f})")`**：打印最佳聚类数和对应分数。

#### 实际运行结果

```
    → 最佳聚类数: 2 (Silhouette Score=0.6263)
```

### 2.5 用最佳聚类数重新聚类

```python
kmeans = KMeans(n_clusters=best_n, random_state=RANDOM_STATE, n_init=10)
clusters = kmeans.fit_predict(sv)
```

#### 逐行解释

- **`kmeans = KMeans(n_clusters=best_n, random_state=RANDOM_STATE, n_init=10)`**：用最佳聚类数 `best_n=2` 创建新的 KMeans 实例。
- **`clusters = kmeans.fit_predict(sv)`**：在 SHAP 矩阵上重新聚类，得到最终的簇标签。

> ⚠️ **常见问题：为什么这里要"重新"聚类？**
>
> 严格来说，前面的循环里 n=2 那次已经聚过类了，这里重新做一次是为了代码清晰——把"搜索最佳 n"和"最终聚类"两个步骤分开。由于 `random_state` 相同，结果完全一致。
>
> 也可以优化为：在循环中记录最佳 `labels`，避免重复计算。但教学代码优先考虑可读性。

## 三、K-Means 算法原理详解

### 3.1 算法步骤

K-Means 是一种**迭代**算法，目标是把 n 个样本分成 K 个簇，使得簇内平方和（inertia）最小。算法步骤：

```
1. 初始化：随机选择 K 个样本作为初始簇中心 c_1, c_2, ..., c_K。

2. 分配步骤（E 步）：把每个样本 x_i 分配到最近的簇中心：
   label(x_i) = argmin_k ||x_i - c_k||²

3. 更新步骤（M 步）：用每个簇中所有样本的均值更新簇中心：
   c_k = mean(x_i for x_i in cluster k)

4. 重复步骤 2-3，直到簇中心不再变化（或变化小于阈值）。
```

### 3.2 在 SHAP 空间的含义

在本教程中，K-Means 的输入是 SHAP 值矩阵 `sv`，形状 `(3000, 6)`：

- 每一行 `sv[i]` 是一个 6 维向量，表示模型对样本 i 的"解释"。
- 每一维对应一个特征的 SHAP 值（如 year 的 SHAP 值、Age 的 SHAP 值）。

K-Means 把"SHAP 值模式相似"的样本归为一类：

- **簇中心** `c_k` 是一个 6 维向量，表示该簇的"典型 SHAP 模式"。
- **距离** `||sv[i] - c_k||²` 衡量样本 i 的 SHAP 模式与簇 k 的典型模式的差异。

> 💡 **小贴士：K-Means 在 SHAP 空间的直觉**
>
> 想象每个样本是一个 6 维空间中的点，6 个坐标分别是 6 个特征的 SHAP 值。
>
> K-Means 找到 K 个"中心点"，把所有样本分配到最近的中心点。
>
> 同一个簇中的样本，它们的 SHAP 模式相似——也就是说，模型用相似的特征组合来解释它们。
>
> 这就是"决策模式群组"的几何含义。

### 3.3 K-Means 的关键参数

| 参数             | 含义     | 本教程取值           | 为什么这么取              |
| -------------- | ------ | --------------- | ------------------- |
| `n_clusters`   | 聚类数    | 2（搜索后）          | Silhouette Score 最高 |
| `random_state` | 随机种子   | 42              | 保证可复现               |
| `n_init`       | 独立运行次数 | 10              | 避免局部最优              |
| `max_iter`     | 最大迭代次数 | 300（默认）         | 一般够用                |
| `init`         | 初始化方法  | 'k-means++'（默认） | 智能选择初始中心，加速收敛       |

## 四、Silhouette Score 详解

### 4.1 计算公式

对于每个样本 i：

```
s(i) = (b(i) - a(i)) / max(a(i), b(i))
```

其中：

- **a(i)**：样本 i 与**同簇**其他样本的平均距离（**紧密度**，越小越好）。
- **b(i)**：样本 i 与**最近的其他簇**中样本的平均距离（**分离度**，越大越好）。

整个数据集的 Silhouette Score 是所有样本 s(i) 的平均值。

### 4.2 取值含义

| s(i)  | 含义           | 解读                    |
| ----- | ------------ | --------------------- |
| 接近 1  | a(i) << b(i) | 样本被很好地聚类，离同簇很近，离其他簇很远 |
| 接近 0  | a(i) ≈ b(i)  | 样本在两个簇的边界上，聚类不确定      |
| 接近 -1 | a(i) >> b(i) | 样本被错误聚类，离其他簇比离同簇还近    |

### 4.3 整体评分的解读

| 整体 Silhouette Score | 解读            |
| ------------------- | ------------- |
| > 0.7               | 聚类结构强         |
| 0.5 \~ 0.7          | 聚类结构合理        |
| 0.25 \~ 0.5         | 聚类结构弱，可能没有自然簇 |
| < 0.25              | 没有聚类结构        |

本教程中 k=2 的 Silhouette Score = **0.6263**，属于"聚类结构合理"偏强的范围，说明 SHAP 空间确实有清晰的聚类结构。

### 4.4 本教程的 Silhouette Score 表

| n\_clusters | Silhouette Score | 判断             |
| ----------- | ---------------- | -------------- |
| **2**       | **0.6263**       | ✅ **最优，结构最清晰** |
| 3           | 0.5936           | 仍可接受，但不如 2     |
| 4           | 0.5190           | 边界模糊           |
| 5           | 0.5639           | 反弹但不稳定         |

> 💡 **小贴士：为什么 k=5 的分数（0.5639）比 k=4（0.5190）高？**
>
> 这看起来违反直觉——"聚类数越多，分数应该越低"。但实际上 Silhouette Score 不是单调的。
>
> 可能的原因：
>
> - k=4 时，某个簇被错误地分裂成两个，导致两个子簇都很"松散"，分数下降。
> - k=5 时，又分裂出另一个簇，使得原来的两个松散子簇各自变得更"紧凑"，分数反弹。
>
> 这说明 Silhouette Score 不能盲目取最大值，还要结合业务理解。本教程中 k=2 明显最优（0.6263 远高于其他），所以选择 k=2 没有争议。

## 五、PCA 降维用于可视化

### 5.1 为什么需要 PCA？

K-Means 在 6 维 SHAP 空间中聚类，但人类无法直接可视化 6 维空间。我们需要把 6 维降到 2 维，才能画散点图。

PCA（主成分分析）是最常用的降维方法：它找到方差最大的方向，把数据投影到这些方向上，保留最多的信息。

### 5.2 PCA 代码

```python
# PCA 降维用于可视化
pca = PCA(n_components=2, random_state=RANDOM_STATE)
sv_pca = pca.fit_transform(sv)
print(f"    PCA 方差解释率: PC1={pca.explained_variance_ratio_[0]:.2%}, PC2={pca.explained_variance_ratio_[1]:.2%}")
print(f"    累积: {pca.explained_variance_ratio_.sum():.2%}")
```

#### 逐行解释

- **`PCA(n_components=2, random_state=RANDOM_STATE)`**：
  - `n_components=2`：降到 2 维（便于画散点图）。
  - `random_state=RANDOM_STATE`：随机种子。PCA 通常是确定性的（用 SVD 分解），但某些求解器（如 'arpack'）有随机性，设种子保证可复现。
- **`sv_pca = pca.fit_transform(sv)`**：
  - `fit(sv)`：在 SHAP 矩阵上学习主成分方向。
  - `transform(sv)`：把 sv 投影到 2 个主成分上，得到 `(3000, 2)` 的降维矩阵。
  - `fit_transform` 一步完成。
- **`pca.explained_variance_ratio_`**：每个主成分解释的方差比例，形状 `(2,)`。
  - `explained_variance_ratio_[0]`：PC1 解释的方差比例。
  - `explained_variance_ratio_[1]`：PC2 解释的方差比例。
  - `.sum()`：两个主成分累积解释的方差比例。
- **`:.2%`**：格式化为百分比，保留 2 位小数（如 `74.00%`）。

#### 实际运行结果

```
    PCA 方差解释率: PC1=74.00%, PC2=18.32%
    累积: 92.32%
```

### 5.3 PCA 结果解读

| 主成分 | 方差解释率      | 累积         |
| --- | ---------- | ---------- |
| PC1 | **74.00%** | 74.00%     |
| PC2 | 18.32%     | **92.32%** |

> 💡 **小贴士：92.32% 是什么意思？**
>
> 这意味着仅用 2 个主成分（PC1 和 PC2），就能解释 SHAP 矩阵 92.32% 的方差。
>
> 换句话说，6 维 SHAP 空间**本质上是低维的**——大部分信息集中在 2 个方向上。
>
> 这说明：
>
> 1. SHAP 空间结构极其清晰，2D 散点图就能很好地展示聚类结构。
> 2. 模型的"决策模式"本质上只有少数几种（与 k=2 最佳聚类数一致）。
> 3. PCA 降维没有丢失太多信息，可视化是可靠的。

### 5.4 PCA 与 K-Means 的关系

注意：本教程中 **K-Means 在原始 6 维 SHAP 空间聚类**，PCA 只用于**可视化**。这是正确的做法——聚类用全维信息，可视化用降维信息。

```
sv (3000, 6) ──K-Means──→ clusters (3000,)     ← 聚类在高维空间
       │
       └──PCA──→ sv_pca (3000, 2)              ← 降维仅用于画图
```

> ⚠️ **常见问题：能不能先 PCA 降维再聚类？**
>
> 可以，但不推荐。PCA 降维会丢失少量信息（本教程中是 7.68%），可能影响聚类结果。
>
> 如果维度很高（如 100+ 维），可以先 PCA 降维再聚类，加速计算。但本教程只有 6 维，直接聚类更好。
>
> 思考题：如果只用前 2 个 PC 重建 SHAP 值并重新聚类，结果会改变吗？（见教学文档思考题 3）

## 七、与教程 6（原始特征聚类）的对比

### 7.1 教程 6 的结果

在教程 6 中，我们对**原始特征**做 K-Means 聚类，得到 ARI ≈ -0.001（与真实标签几乎无关），说明原始特征空间**没有天然聚类结构**。

### 7.2 本教程的结果

本教程对 **SHAP 值**做 K-Means 聚类，得到 Silhouette Score = 0.6263，说明 SHAP 空间**有清晰的聚类结构**。

### 7.3 为什么不同？

| 维度   | 原始特征聚类（教程 6）         | SHAP 值聚类（本教程）            |
| ---- | -------------------- | ------------------------ |
| 输入   | 患者的特征值（Age, year, …） | 模型对每个患者的解释（SHAP 值）       |
| 分组依据 | "患者的特征值相似"           | "模型的决策逻辑相似"              |
| 产出   | 临床表型群组               | 决策模式群组                   |
| 结果   | ARI ≈ -0.001（无结构）    | Silhouette = 0.6263（有结构） |

**核心洞察**：

- 患者本身的特征不一定能形成族群（年龄、性别等是连续的，没有天然分界）。
- 但模型对患者的"解释"形成了清晰的族群——也就是说，模型对不同患者使用了不同的决策逻辑。
- 这是"个性化医学"在解释层面的体现。

> 💡 **小贴士：SHAP 聚类的哲学意义**
>
> 传统观点认为"模型对所有患者一视同仁"。但 SHAP 聚类告诉我们：**模型对不同患者使用不同的决策逻辑**。
>
> 这意味着：
>
> 1. 全局解释（如 SHAP Summary Plot）可能掩盖亚群差异。
> 2. 局部解释（如 SHAP Waterfall）只看了单个患者，无法发现模式。
> 3. SHAP 聚类连接了全局和局部，揭示了"模型有几种决策模式"。

## 小贴士

### 1. K-Means 的"球形簇"假设

K-Means 假设簇是球形的（各方向方差相同）。如果真实的簇是椭圆形或任意形状，K-Means 可能表现不好。

在 SHAP 空间中，由于 PCA 显示 92.32% 方差集中在 2 个方向，簇的形状接近球形，所以 K-Means 适用。

如果你想探索任意形状的簇，可以用 DBSCAN 或层次聚类。

### 2. `n_init=10` 的重要性

K-Means 对初始中心点敏感。不同的初始中心可能导致：

- 收敛到不同的局部最优。
- 甚至得到完全不同的聚类结果。

`n_init=10` 表示运行 10 次独立的 K-Means（每次随机初始化），取 inertia（簇内平方和）最小的那次作为最终结果。这能大幅降低陷入局部最优的风险。

从 sklearn 1.4 开始，`n_init='auto'` 是默认值，会根据 `init` 方法自动选择。但显式设为 10 更明确。

### 3. Silhouette Score 的计算成本

Silhouette Score 需要计算所有样本对的距离，时间复杂度 O(n²)。对于 3000 个样本，这是 900 万次距离计算，仍然可接受。但如果样本量到 100000+，可能需要用采样近似。

### 4. 为什么不直接用 inertia 选择 k？

inertia（簇内平方和）随 k 增大单调下降，所以不能直接取最大值。常用的方法是"肘部法则"（Elbow Method）：画 inertia vs k 的曲线，找"肘部"作为最佳 k。但肘部法则主观性强，不如 Silhouette Score 客观。

## 常见问题

### Q1: 为什么 `random_state=42` 这么重要？

**A**: K-Means 的初始化是随机的，不同种子可能得到不同的聚类结果。设置 `random_state=42` 保证每次运行结果一致，便于复现和调试。如果不设种子，每次运行可能得到不同的簇编号（甚至不同的簇结构），让分析无法重复。

### Q2: K-Means 的簇编号有含义吗？

**A**: 没有。K-Means 的簇编号（0, 1, 2, ...）是任意的，每次运行可能不同。例如，这次运行中"主流簇"可能是 Cluster 1，下次运行可能是 Cluster 0。所以**不要把簇编号当作有序变量**，它只是分类标签。

### Q3: 为什么不试 k=6 或更大的值？

**A**: 因为只有 6 个特征，k=6 意味着每个簇可能只由 1 个特征区分，这过于碎片化，没有业务意义。一般来说，k 不应超过特征数的 2\~3 倍。本教程限制在 k=2\~5 是合理的。

### Q4: Silhouette Score = 0.6263 算高吗？

**A**: 算"合理偏强"。一般解读：

- <br />
  > 0.7：强结构
- 0.5\~0.7：合理结构
- 0.25\~0.5：弱结构
- < 0.25：无结构

0.6263 在 0.5\~0.7 区间，说明 SHAP 空间有合理的聚类结构，但不是"铁板一块"。这与医学数据的复杂性一致——患者群体有一定异质性，但不是截然分开的。

### Q5: PCA 的 `random_state` 有用吗？

**A**: 大多数情况下没用。PCA 默认用 SVD（奇异值分解）求解，是确定性的。但某些求解器（如 `'arpack'`）有随机性，设种子能保证可复现。设了也没坏处，所以本教程设了。

### Q6: 为什么 `fit_predict` 而不是分步调用？

**A**: `fit_predict(sv)` 等价于 `fit(sv)` + `predict(sv)`，但更简洁。在某些情况下（如聚类），`fit_predict` 还能避免重复计算，效率更高。但对于 K-Means，两者性能差异不大，主要是代码简洁性的考虑。

### Q7: 如果 Silhouette Score 都很低（如 < 0.3）怎么办？

**A**: 说明数据没有天然聚类结构。可能的原因：

1. 数据本身就是连续的，没有分群。
2. K-Means 的球形假设不成立，需要换算法（如 DBSCAN）。
3. 特征选择不当，需要重新选择或构造特征。

在本教程中，Silhouette Score = 0.6263，说明 SHAP 空间确实有结构，可以继续分析。
