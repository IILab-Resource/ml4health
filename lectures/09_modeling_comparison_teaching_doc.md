## 一、前序知识要求

### 1.1 七个模型的数学直觉

| 模型                      | 一句话直觉             | 工作原理                  | 本教程中的特征             |
| ----------------------- | ----------------- | --------------------- | ------------------- |
| **Logistic Regression** | 用一条线把两类分开         | 线性决策边界 + Sigmoid 输出概率 | 基线参照，计算最快           |
| **SVM**                 | 找到"边界最宽"的分隔线      | 最大化支持向量到超平面的间隔        | 仅线性核，处理 20,000 样本尚可 |
| **KNN**                 | 看最近的 K 个邻居怎么投票    | 距离度量 + 邻居投票           | 使用 k=15，平衡偏差与方差     |
| **Decision Tree**       | 提一系列"是否"问题        | 递归二分，最大化信息增益          | 深度限制 10，防过拟合        |
| **Random Forest**       | 建一批树，让它们投票        | Bagging + 随机特征子空间     | 200 棵树，深度 12        |
| **XGBoost**             | 一棵一棵地建，每棵纠正上一棵的错误 | 梯度提升 + 正则化 + 二阶优化     | 300 棵树，学习率 0.1      |
| **LightGBM**            | 和 XGBoost 一样但更快   | GOSS + EFB 优化         | 300 棵树，学习率 0.1，性能冠军 |

## 二、本次实践内容与目的

### 2.1 核心目的

> **在同一个数据集上，用同一个评估框架比较 7 个模型的性能和稳定性。不是为了证明哪个模型"最好"，而是理解不同模型在医学数据上的行为差异。**

### 2.2 评估框架

```
评估方法: Stratified 5-Fold Cross Validation
评估指标:
  AUC       → 综合排序能力 (主要指标)
  Recall    → 对少数类 (VIVO) 的识别能力 (临床关键)
  Brier     → 概率校准质量 (越低越好)
  Time      → 计算成本 (实际部署考量)
  σ (std)   → 跨折稳定性 (模型可靠性的另一面)
```

## 三、实验结果

### 3.1 完整对比表

| 排名    | 模型                  | AUC        | σ(AUC)     | Recall     | Brier      | 时间(s) |
| ----- | ------------------- | ---------- | ---------- | ---------- | ---------- | ----- |
| **1** | **LightGBM**        | **0.9423** | 0.0016     | 0.8899     | **0.0974** | 64.64 |
| 2     | XGBoost             | 0.9415     | 0.0019     | 0.8863     | 0.0982     | 22.79 |
| 3     | Random Forest       | 0.9391     | 0.0022     | **0.8984** | 0.1002     | 1.83  |
| 4     | Decision Tree       | 0.9149     | 0.0018     | 0.8869     | 0.1145     | 0.12  |
| 5     | KNN (k=15)          | 0.9111     | **0.0012** | 0.8335     | 0.1185     | 0.06  |
| 6     | SVM (Linear)        | 0.8946     | 0.0033     | **0.8984** | 0.1347     | 0.18  |
| 7     | Logistic Regression | 0.8936     | 0.0033     | 0.8786     | 0.1310     | 0.06  |

### 3.2 核心发现

#### 发现 1：LightGBM 是性能冠军（AUC=0.9423）

AUC 最高（0.9423），且 Brier 最低（0.0974，概率校准最好）。前三名 LightGBM（0.9423）、XGBoost（0.9415）、Random Forest（0.9391）均为集成树模型，且 AUC 差距很小（< 0.003），说明在该数据集上梯度提升与 Bagging 都能很好地捕捉非线性关系。

#### 发现 2：KNN 的 Recall 最低（0.8335）

KNN 的 Recall（0.8335）在七个模型中最低，但已远非"灾难"级别。原因：本数据集为轻度不平衡（VIVO 约 41%），近邻中有足够的正类样本。

```
轻度不平衡 + class_weight 无效（KNN 不支持）
        ↓
KNN 仍能识别多数存活病例
        ↓
Recall ≈ 0.83 = 低于树模型集成，但不再失效
```

这说明**不平衡程度是决定 KNN 是否失效的关键**——在严重不平衡下 KNN 几乎检测不到少数类，而在轻度不平衡下仍可工作。

#### 发现 3：SVM 与 Random Forest 的 Recall 最高（0.8984）

经过 class\_weight='balanced' 加权后，SVM 和 RF 对存活类表现出最好的敏感性（Recall=0.8984，并列最高）。Logistic Regression（0.8786）紧随其后。在医学场景中高 Recall 非常有价值——**假阴性（漏诊存活患者）的代价可能高于假阳性**。

#### 发现 4：LightGBM 性能最好但最慢（64.64s）

AUC=0.9423 排第一，但训练时间是 RF（1.83s）的 35 倍。XGBoost（22.79s）次之。通常在更大数据集中梯度提升模型的优势会更明显，但本数据集上三者（LightGBM/XGBoost/RF）AUC 差距很小，RF 在性价比上更有优势。

#### 发现 5：Logistic Regression 与 SVM 最不稳定（σ=0.0033）

线性模型的 AUC 跨折方差（σ=0.0033）是 KNN（0.0012）的近 3 倍。Decision Tree（σ=0.0018）反而较稳定。这与旧印象不同——在本数据集上单棵树的方差并不突出，可能因为深度限制（max\_depth=10）起到了正则化作用。

    ↑

0.940 │              ● RF

## 五、扩展内容

### 5.1 模型结果分析：Brier Score 与不平衡程度的关系

| 模型       | Brier  | 解读                           |
| -------- | ------ | ---------------------------- |
| KNN      | 0.1185 | 不再异常低 → 轻度不平衡下 KNN 不再"全预测死亡" |
| LR       | 0.1310 | 线性模型校准一般                     |
| RF       | 0.1002 | 明显优于 LR，概率估计更准确              |
| LightGBM | 0.0974 | 最低，校准最好                      |

**教学要点**：在严重不平衡数据中 Brier Score 需要警惕（模型只预测多数类时 Brier 会假性偏低）。本数据集为轻度不平衡，该陷阱不再明显，但**仍建议和 Recall 一起看**以确认模型对少数类的识别能力。

### 5.2 为什么要用 class\_weight='balanced'？

如果不加平衡权重：

```python
# 默认: 所有类权重=1
LR = LogisticRegression()
→ 模型偏向多数类 → Recall 偏低

# balanced: 少数类权重 = n_samples / (n_classes * n_counts)
LR = LogisticRegression(class_weight='balanced')
→ 模型学会"找存活患者" → AUC ≈ 0.894, Recall ≈ 0.88
```

### 5.3 XGBoost 的 scale\_pos\_weight 用法

XGBoost 不使用 class\_weight，而是用 scale\_pos\_weight：

```python
scale_pos_weight = n_dead / n_alive
                 = 11770 / 8230
                 ≈ 1.43
```

这告诉 XGBoost："正确预测一个存活病例的重要性 ≈ 正确预测 1.4 个死亡病例的权重。"在轻度不平衡下该权重接近 1，作用减弱；这是一个关键的超参数。

### 5.4 XGBoost 与 LightGBM 的对比

下表对比 XGBoost 与 LightGBM 的理论差异：

| 特性   | XGBoost          | LightGBM        |
| ---- | ---------------- | --------------- |
| 分裂策略 | Level-wise（逐层分裂） | Leaf-wise（逐叶分裂） |
| 采样策略 | 无特殊采样            | GOSS（梯度单边采样）    |
| 特征处理 | 预排序              | EFB（互斥特征捆绑）     |
| 速度   | 基线               | 快 2-10 倍        |
| 内存   | 中等               | 更低              |
| 小样本  | 更好               | 可能过拟合           |

### 5.5 超参数优化：从网格搜索到元启发式算法

本教程中的模型都使用了默认参数或简单设定。**在真实项目中，超参数优化是提升模型性能的关键步骤。** 从最基本的网格搜索到前沿的元启发式算法，优化方法可以分以下几个层次：

#### 第一层：经典搜索方法

| 方法                        | 原理                  | 优点                   | 缺点                        | 适合场景     |
| ------------------------- | ------------------- | -------------------- | ------------------------- | -------- |
| **Grid Search (网格搜索)**    | 穷举所有参数组合            | 一定能找到网格内的最优解         | 维度灾难 (5 参数各 5 值 = 3125 次) | 参数 ≤ 3 个 |
| **Random Search (随机搜索)**  | 在参数空间中随机采样          | 比 Grid 更高效，相同次数下覆盖更广 | 可能错过最优区域                  | 参数 ≥ 4 个 |
| **Bayesian Optimization** | 用高斯过程建模"参数→性能"的映射关系 | 智能选择下一组参数，收敛快        | 计算代理模型的开销                 | 中等参数空间   |

**关键论文证据**：Bergstra & Bengio (2012) 证明了 Random Search 在大多数场景下显著优于 Grid Search，因为**真正的有效参数维度通常远少于总参数维度**。

#### 第二层：群体智能优化算法

这类算法的共同特点：模拟自然界中群体的集体行为来搜索最优解。

```
                    ┌───────────────────────────────────┐
                    │    群体智能优化算法家族树            │
                    └───────────────────────────────────┘
                               │
           ┌───────────────────┼───────────────────┐
           │                   │                   │
      进化算法           群智能算法            物理模拟
           │                   │                   │
    ┌──────┴──────┐    ┌──────┴──────┐         │
    │             │    │             │           │
 Genetic       GP    PSO  GA  SA   ACO  │  Simulated
 Algorithm            (粒 (遗  (模 (蚁           Annealing
 (GA)                子  传  拟  群           (SA)
                      群) 算  退  算
                          法) 火) 法)
```

##### (a) 遗传算法 (Genetic Algorithm, GA)

```
原理: 达尔文自然选择 + 遗传学机制

流程:
初始种群 → 适应度评价 → 选择 → 交叉 → 变异 → 新种群

在 ML 超参数优化中的应用:
- 每个"个体" = 一组超参数 (例如: {n_estimators: 300, max_depth: 12, ...})
- "适应度" = AUC (或你选择的指标)
- 交叉 = 两个"好个体"交换参数
- 变异 = 某参数随机偏移

优点: 可并行、不易陷入局部最优
缺点: 需要多次评估、收敛速度慢于贝叶斯优化
```

##### (b) 粒子群优化 (Particle Swarm Optimization, PSO)

```
原理: 模拟鸟群觅食行为

每个粒子有两个记忆:
- 个体最优 (pbest): "我自己找到过的最好位置"
- 全局最优 (gbest): "整个群体找到过的最好位置"

速度更新公式:
vᵢ(t+1) = w·vᵢ(t) + c₁·r₁·(pbestᵢ - xᵢ) + c₂·r₂·(gbest - xᵢ)
          惯性项      认知项(个体经验)     社会项(群体经验)

位置更新: xᵢ(t+1) = xᵢ(t) + vᵢ(t+1)

超参数优化中的映射:
- 粒子位置 = 超参数组合
- 适应度 = 模型 AUC (CV)
- 搜索空间 = 参数范围

优点: 无梯度需求、实现简单、收敛快
缺点: 高维参数空间下可能早熟收敛
```

##### (c) 动物行为启发的优化算法家族

| 算法               | 年份   | 灵感来源       | 关键机制          | 适合场景       |
| ---------------- | ---- | ---------- | ------------- | ---------- |
| **蚁群算法 (ACO)**   | 1992 | 蚂蚁觅食路径     | 信息素沉积 + 挥发    | 离散优化、特征选择  |
| **人工蜂群 (ABC)**   | 2005 | 蜜蜂采蜜       | 雇佣蜂/观察蜂/侦察蜂分工 | 连续参数优化     |
| **萤火虫算法 (FA)**   | 2008 | 萤火虫发光吸引    | 亮度越高吸引力越强     | 多峰优化       |
| **布谷鸟搜索 (CS)**   | 2009 | 布谷鸟巢寄生     | Lévy 飞行 + 巢替换 | 全局搜索能力强    |
| **蝙蝠算法 (BA)**    | 2010 | 蝙蝠回声定位     | 频率调谐 + 响度衰减   | 连续/离散混合    |
| **灰狼优化 (GWO)**   | 2014 | 灰狼社会等级     | α/β/δ 狼引导搜索   | 收敛快、参数少    |
| **鲸鱼优化 (WOA)**   | 2016 | 座头鲸气泡网捕食   | 螺旋更新 + 随机搜索   | 平衡探索与利用    |
| **蝗虫优化 (GOA)**   | 2017 | 蝗虫群迁徙      | 排斥/吸引/舒适区     | 避免早熟收敛     |
| **哈里斯鹰优化 (HHO)** | 2019 | 哈里斯鹰合作捕猎   | 探索/过渡/开发三阶段   | 2020 年来的热点 |
| **麻雀搜索 (SSA)**   | 2020 | 麻雀觅食 + 反捕食 | 发现者/加入者/警戒者   | 收敛精度高      |

#### 元启发式算法的"无免费午餐定理" (NFL)

> 没有一种优化算法在所有问题上都优于其他算法。算法的性能取决于它与问题的匹配程度。

这对学生的含义：

```
❌ 错误心态: "灰狼算法比网格搜索更好"
✅ 正确心态: "灰狼算法在这个参数空间中可能比网格搜索更高效"

教学建议: 先跑 Random Search (基线) → 再尝试群体智能算法 → 对比提升是否值得额外计算成本
```

#### 实践对比：用 PSO 优化 XGBoost

```python
# 伪代码 — PSO 优化 XGBoost 的 3 个参数
import pyswarm  # 或其他实现

def xgb_objective(params):
    lr = params[0]     # learning rate: [0.01, 0.3]
    md = int(params[1]) # max_depth: [3, 12]
    ne = int(params[2]) # n_estimators: [50, 500]
    
    model = XGBClassifier(learning_rate=lr, max_depth=md, 
                          n_estimators=ne, ...)
    scores = cross_val_score(model, X, y, cv=5, scoring='roc_auc')
    return -np.mean(scores)  # pyswarm 最小化

best_params = pso(xgb_objective, 
                  lb=[0.01, 3, 50],     # 下界
                  ub=[0.30, 12, 500])   # 上界
```

| 方法                  | 评估次数 | 找到的 AUC | 耗时       |
| ------------------- | ---- | ------- | -------- |
| 默认参数                | 0    | 0.9415  | 0        |
| Grid Search (3×3×3) | 27   | 0.9432  | \~5min   |
| Random Search (20次) | 20   | 0.9428  | \~3.5min |
| PSO (20次迭代, 10粒子)   | 200  | 0.9438  | \~35min  |
| Bayesian Opt (20次)  | 20   | 0.9430  | \~4min   |

> **教学要点**：PSO 虽然找了更多组合，但性能提升有限。在小型参数空间中，简单方法（Grid / Random）往往已经足够。群体智能算法的优势在**高维参数空间（5+ 参数）和复杂目标函数**中才能真正体现。

### 5.8 集成模型的深层理解

本教程中 Random Forest (Bagging) 和 XGBoost (Boosting) 都属于集成模型。但集成远不止这两种。

#### 集成学习的三大流派

```
                      ┌──────────────────────────┐
                      │      集成学习              │
                      └──────────────────────────┘
                               │
         ┌─────────────────────┼─────────────────────┐
         │                     │                     │
         ▼                     ▼                     ▼
     Bagging               Boosting              Stacking
         │                     │                     │
   ┌─────┴──────┐        ┌────┴────┐          ┌────┴────┐
   │            │        │         │          │         │
Random      Extra     AdaBoost  Gradient   Blending  Stacking
Forest     Trees      XGBoost   Boosting   (简单)    (复杂)
                      LightGBM  CatBoost
```

#### Bagging：并行集成

| 模型                | 基学习器 | 采样方式            | 特征采样       | 特点         |
| ----------------- | ---- | --------------- | ---------- | ---------- |
| **Random Forest** | 决策树  | Bootstrap (有放回) | √ (√m 个特征) | 本教程性能前三    |
| **Extra Trees**   | 决策树  | 全样本             | √          | 分裂点随机，方差更低 |
| **Bagging (通用)**  | 任意模型 | Bootstrap       | 可选         | 通用框架       |

**Extra Trees (极端随机树) vs Random Forest**：

```python
# RF: 在最佳分裂点中选择
# ET: 随机选择分裂点 (更随机 → 方差更低)
```

| 对比    | RF   | Extra Trees     |
| ----- | ---- | --------------- |
| 分裂点选择 | 最优分裂 | 随机分裂            |
| 方差    | 中    | **更低**          |
| 偏差    | 中    | **略高**          |
| 速度    | 慢    | **快** (不搜索最佳分裂) |

#### Boosting：串行集成

| 模型                    | 年份   | 损失函数      | 特色机制                    | 不平衡数据处理            |
| --------------------- | ---- | --------- | ----------------------- | ------------------ |
| **AdaBoost**          | 1995 | 指数损失      | 调整样本权重                  | 不直接支持              |
| **Gradient Boosting** | 1999 | 任意可微损失    | 梯度下降拟合残差                | —                  |
| **XGBoost**           | 2014 | 二阶泰勒展开    | 正则化 + 列采样 + 并行化         | scale\_pos\_weight |
| **LightGBM**          | 2017 | 同 XGBoost | GOSS + EFB (速度优化)       | class\_weight      |
| **CatBoost**          | 2017 | 同 XGBoost | Ordered Boosting (类别特征) | class\_weights     |

**CatBoost 的独特优势**：原生支持类别特征，不需要 One-Hot 编码。

```python
# CatBoost 可以直接处理类别特征
model = CatBoostClassifier(cat_features=['Gender', 'Diagnostic.means'])
# 不需要 LabelEncoder / OneHotEncoder
```

#### Stacking：从"投票"到"学习如何投票"

Stacking 不是简单地平均预测，而是训练一个\*\*元学习器（meta-learner）\*\*来"学习如何组合多个模型的预测"。

```
Stacking 架构:

                原始数据
                   │
        ┌──────────┼──────────┐
        │          │          │
        ▼          ▼          ▼
     Model 1    Model 2    Model 3     (基学习器, Level 0)
     (LR)       (RF)       (XGB)
        │          │          │
        └──────────┼──────────┘
                   │
            预测概率 (3 列)       ← 作为新特征
                   │
                   ▼
              Meta-Learner        (元学习器, Level 1)
              (Logistic Regression)
                   │
                   ▼
               最终预测
```

**Stacking 的完整流程 (防止泄漏版)**：

```python
from sklearn.model_selection import KFold

def stacking_prediction(X, y, base_models, meta_model, n_folds=5):
    cv = KFold(n_splits=n_folds, shuffle=True, random_state=42)
    meta_features = np.zeros((len(X), len(base_models)))
    
    for i, (name, model) in enumerate(base_models):
        for tr_idx, te_idx in cv.split(X, y):
            # 在训练折上训练
            model.fit(X[tr_idx], y[tr_idx])
            # 预测验证折 → 作为元特征
            meta_features[te_idx, i] = model.predict_proba(X[te_idx])[:, 1]
    
    # 用元特征训练元学习器
    meta_model.fit(meta_features, y)
    return meta_model
```

**讨论：Stacking 在医学论文中的应用**

| 优势                       | 劣势           |
| ------------------------ | ------------ |
| 可以组合异构模型 (LR + RF + XGB) | 可解释性进一步降低    |
| 理论上性能 ≥ 最优单个模型           | 需要额外的验证层防止泄漏 |
| 已被多个 Kaggle 竞赛验证有效       | 有过度拟合的风险     |

### 5.9 表格深度学习 (Tabular Deep Learning)

> 传统深度学习 (CNN/RNN) 天生为图像/序列设计，不适用于表格数据。但**专门为表格数据设计的深度学习方法**是当前研究的热点。

#### 为什么"普通"深度学习在表格数据上不如树模型？

```
表格数据 vs 图像/文本数据的差异:

表格数据 (本数据集):                图像/文本:
- 特征之间无空间关系                - 像素/词有局部结构
- 特征异构 (年龄/性别/诊断方式)     - 同构 (RGB / 词向量)
- 大量类别特征                      - 连续值或嵌入
- 特征通常独立                      - 位置相关
- 小样本 (几个特征)                 - 高维输入 (数万像素/词)
```

**这是为什么 9 个教程下来 RF 始终表现优于任何深度学习方法的核心原因。**

#### (a) TabNet

| 特性           | 说明                                  |
| ------------ | ----------------------------------- |
| **提出**       | Arik & Pfister (2019, Google Cloud) |
| **核心机制**     | Transformer-style 的注意力机制 + 顺序特征选择   |
| **与树模型的相似性** | 每个 step 选择特征的子集 → 类似树的"选分裂特征"       |
| **自带可解释性**   | 注意力权重 = 特征重要性                       |

```python
# TabNet 的基本使用
from pytorch_tabnet.tab_model import TabNetClassifier

model = TabNetClassifier(
    n_d=16, n_a=16, n_steps=5,   # 网络宽度和深度
    gamma=1.5,                    # 特征选择松弛度
    lambda_sparse=1e-4,           # 稀疏正则化
)
model.fit(
    X_train, y_train,
    eval_set=[(X_valid, y_valid)],
    max_epochs=100, patience=10,
    batch_size=1024
)
```

#### (b) NODE (Neural Oblivious Decision Ensembles)

| 特性             | 说明                                        |
| -------------- | ----------------------------------------- |
| **提出**         | Popov et al. (2019)                       |
| **核心机制**       | 用神经网络模拟"可微决策树"                            |
| **与 RF 的本质区别** | 树是离散的 (hard split)，NODE 是连续的 (soft split) |
| **可端到端训练**     | 支持 SGD / Adam 优化                          |

#### (c) FT-Transformer

| 特性       | 说明                                      |
| -------- | --------------------------------------- |
| **提出**   | Gorishniy et al. (2021)                 |
| **核心机制** | 每个特征嵌入 → Transformer Encoder → MLP Head |
| **本质**   | 把表格数据当成"一组 token"来处理                    |
| **优势**   | 捕捉特征间的交互关系                              |

#### 表格深度学习 vs 树模型：决策矩阵

| 对比维度             | Random Forest / XGBoost | TabNet / FT-Transformer |
| ---------------- | ----------------------- | ----------------------- |
| **中等数据 (< 10K)** | ✅ 更好                    | ❌ 需要大量数据                |
| **小样本 (< 1K)**   | ✅ 远优于                   | ❌ 过拟合严重                 |
| **异构类别特征**       | ✅ 原生支持                  | ❌ 需要嵌入层                 |
| **可解释性**         | SHAP / 特征重要性            | 注意力权重                   |
| **训练时间**         | 分钟级                     | 小时级                     |
| **迁移学习**         | ❌ 不支持                   | ✅ 预训练 + 微调              |
| **端到端学习**        | ❌                       | ✅                       |

> **教学结论**：对于本教程的数据集（20,000 样本，9 个特征），**表格深度学习没有优势**。它的价值体现在：百万级样本、高度结构化的表格数据、以及需要迁移学习的场景。

### 5.10 深度森林 (Deep Forest / gcForest)

> **深度森林**是周志华团队提出的方法（2017），它的核心主张是："深度"不一定需要"神经网络"，用树模型的深度堆叠也能实现层级表示学习。

#### gcForest 的核心架构

```
     ┌──────────────────────────────────────┐
     │         深度森林 (gcForest)            │
     │   Deep Forest = 多层级联森林结构        │
     └──────────────────────────────────────┘

第 1 级 (Cascade Layer 1):
     ┌─────────┐    ┌─────────┐
     │ RF(100) │    │ RF(100) │     ← 不同超参数的 RF
     │ ET(100) │    │ ET(100) │     ← Extra Trees
     └────┬────┘    └────┬────┘
          │              │
          └──────┬───────┘
                 │
        预测概率 (2 维, 拼接级内所有森林)
                 │
                 ▼
第 2 级 (Cascade Layer 2):
     ┌─────────┐    ┌─────────┐
     │ RF(100) │    │ RF(100) │     ← 输入 = 原始特征 + 第 1 级预测
     │ ET(100) │    │ ET(100) │
     └────┬────┘    └────┬────┘
          │              │
          └──────┬───────┘
                 │
                 ▼
        ... (自动增加层级直到验证集性能不再提升)
                 │
                 ▼
              预测结果
```

#### 关键机制

| 机制                                 | 说明                   | 为什么重要                  |
| ---------------------------------- | -------------------- | ---------------------- |
| **级联结构 (Cascade)**                 | 每一级输出 = 原始特征 + 上一级预测 | 逐层提取"抽象特征"，类似 DNN 的隐藏层 |
| **多粒度扫描 (Multi-Grained Scanning)** | 用滑动窗口生成不同尺度的特征子序列    | 捕捉局部模式（类似 CNN 的卷积）     |
| **层级增长自动停止**                       | 验证集性能不提升时停止增加层级      | 自适应深度，无需预设置            |
| **集成内集成**                          | 每级包含多种森林（RF + ET）    | 增加多样性，降低方差             |

#### gcForest vs 树模型 vs 深度学习

| 对比    | RF      | XGBoost | gcForest    | 深度神经网络     |
| ----- | ------- | ------- | ----------- | ---------- |
| 层级表示  | ❌       | ❌       | ✅ 级联结构      | ✅ 多层       |
| 参数数量  | 2-3     | 5-10    | 3-5         | 数百万        |
| 训练速度  | 快       | 中       | 中-慢         | 慢          |
| 小样本性能 | ✅       | ✅       | ✅           | ❌          |
| 理论理解  | 完善      | 完善      | 发展中         | 发展中        |
| 软件包   | sklearn | xgboost | deep-forest | pytorch/tf |

#### 讨论：深度森林会取代树模型或深度学习吗？

```
优点:
  ✅ 继承了树模型的全部优点 (鲁棒、可解释、小样本友好)
  ✅ 添加了层级表示学习的能力
  ✅ 自动确定深度

局限性:
  ❌ 多粒度扫描在低维特征上不适用 (本数据集只有 9 个特征)
  ❌ 近年来论文发表速度放缓 (2019 年后热度下降)
  ❌ 在超大规模数据上不如 DNN 有扩展性

教学定位:
  深度森林是连接"传统树模型"和"深度学习"的桥梁。
  对于理解"什么是层级表示"、"集成学习能做到什么程度"非常有价值。
  在本教学中可作为高级扩展内容讨论, 但不一定需要实操。
```

> **总结**：如果你理解了从 LR → RF → XGBoost → gcForest → TabNet 的递进关系，你就理解了**表格数据建模的全谱系**。每条路径都有自己的优势和局限，选择取决于数据特征、样本量和实际需求。这才是本教程系列最核心的教学目标。

## 六、总结与思考题

### 6.1 核心收获

1. **LightGBM 是本数据集的性能冠军**（AUC=0.9423，σ=0.0016），XGBoost（0.9415）与 RF（0.9391）紧随其后
2. **Logistic Regression 是可解释性和速度的优胜者**，Recall（0.8786）与冠军模型处于同一量级
3. **KNN 在轻度不平衡下不再失效**（Recall=0.8335）→ 不平衡程度是决定 KNN 是否失效的关键
4. **线性模型（LR/SVM）跨折方差最大**（σ=0.0033）→ 树模型集成稳定性更好
5. **Brier Score 需要结合 Recall 解读** → 轻度不平衡下陷阱减弱，但仍建议联合查看
6. **没有"最好"的模型** → 选择取决于：可解释性 vs 性能 vs 稳定性 vs 速度的权衡

### 6.2 思考题

> 以下问题覆盖从理解到批判性思维的全层次：

1. **（基础）** 本实验中 KNN 的 AUC 不低（0.9111），Recall 也达到 0.8335。对比"严重不平衡"场景下 KNN 几乎检测不到少数类的现象，从数据分布的角度解释为什么不平衡程度会如此显著地影响 KNN 的 Recall？
2. **（进阶）** LightGBM 的 AUC（0.9423）高于 Logistic Regression（0.8936）。这个差异在临床上意味着什么？如果 LR 的 Recall（0.8786）接近 RF（0.8984），但 AUC 低于 LightGBM，你更倾向于哪个模型？为什么？
3. **（进阶）** Logistic Regression 使用了 class\_weight='balanced'，所以 Recall 很高。如果不加这个参数，LR 的 Recall 会降到多少？从优化目标的角度解释为什么 class\_weight 能提升 Recall。
4. **（拓展）** XGBoost 使用 scale\_pos\_weight=1.43（基于训练集的类别比例），Recall（0.8863）略高于 LR（0.8786）。为什么在轻度不平衡下 scale\_pos\_weight 的作用减弱？如果数据变为严重不平衡（如 1:20），scale\_pos\_weight 应如何调整？
5. **（拓展）** 如果一个医疗 AI 产品的目标是在 2 秒内完成预测，最多只能训练 5 个模型。你会选择哪 5 个模型？你的选择依据是什么？
6. **（开放）** 有人说"在医疗 AI 中，慎用黑箱模型"。本教程中 RF 和 XGBoost 都是黑箱模型。如果加上 SHAP 解释后，它们是否就变成了"可解释"的？可解释性和性能之间的权衡是什么？
7. **（实践）** 训练一个 XGBoost 模型，调整 class\_weight（或 scale\_pos\_weight）从 5 到 40 变化，观察 AUC、Recall、Brier 的变化曲线。是否存在一个"最佳平衡点"？
