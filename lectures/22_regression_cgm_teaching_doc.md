## 二、回归核心概念 (新知识点)

前 21 个教程全是**分类** (二分类/多分类). 回归任务的核心差异集中在这 4 点:

### 2.1 连续目标 vs 类别标签

- **分类**: 目标是离散类别 (0/1, 或 5 种癌症), 评估看混淆矩阵/AUC/Recall
- **回归**: 目标是连续值 (iauc\_2h ∈ \[0, 14341]), 评估看残差 (MAE/RMSE/R²)
- **统计也变**: 分类用组间检验 (T 检验/MW-U/ANOVA); 回归用**相关分析**
  (Pearson/Spearman) — 因为目标是连续的, 没有"组"可分

### 2.2 分组 + 时间数据结构 (i.i.d. 假设破坏)

本数据每行是一**餐次**, 但同一受试者有 17-63 餐. 这导致:

- **同一人的餐次高度相关** (代谢基线/饮食习惯/CGM 传感器特性) → 朴素 KFold
  会把同一个人的餐次分到训练集和测试集 → **泄漏** (模型学到的是"认人"而非"预测血糖")
- **餐次有时间顺序** → 可做"未来餐次预测" (用历史餐次训练, 未来餐次测试)
- 这两点决定了本案例必须用 **4 种划分策略** (见第四节, 核心教学点)

### 2.3 划分策略决定回答什么问题

| 策略                    | 回答的问题            | 泄漏风险                    | 适用场景       |
| --------------------- | ---------------- | ----------------------- | ---------- |
| 朴素 KFold              | (反面教材)           | 🔴 高 (同人餐次跨 train/test) | ❌ 不应用于分组数据 |
| GroupKFold            | 能泛化到**新用户**吗?    | ✅ 无                     | 部署上线 (新患者) |
| 受试者内时间 80/20          | 能预测某用户**未来餐次**吗? | ✅ 无                     | 老用户持续监测    |
| Few-shot k=0,1,3,5,10 | 新用户需登录**多少餐**?   | ✅ 无                     | 冷启动/个性化    |

### 2.4 回归评估 = 残差思维

分类有阈值 → 校准曲线/DCA/混淆矩阵. 回归**无阈值**, 评估围绕**残差** (pred − obs):

- **点估计**: MAE (平均绝对误差), RMSE (均方根误差, 惩罚大误差), R² (方差解释率), Pearson r
- **不确定性**: subject-level bootstrap CI (重抽样受试者, 非餐次)
- **诊断**: 残差 vs 预测 (同方差), Q-Q 图 (正态性), Scale-Location (异方差)
- **校准**: 观测 = intercept + slope × 预测 (理想 slope=1, intercept=0; slope<1 = 预测被压缩)

## 三、★ 迁移对照表 (本案例核心教学价值)

下表把 CGM 回归案例的每一步与前面教程 01-12 的分类知识点显式对照, 让学生看清:
**哪些步骤完全通用, 哪些步骤因回归/分组结构而需要调整**.

| 步骤          | 分类 (教程 01-12)         | **回归 (本案例 22)**                                    | 异同                      |
| ----------- | --------------------- | -------------------------------------------------- | ----------------------- |
| **数据加载**    | `read_csv`            | `read_csv` (现成建模表)                                 | 🟢 完全相同                 |
| **①EDA**    | 类别分布/缺失/离群            | **连续目标分布 (右偏) + log1p + 每受试者餐次**                   | 🟡 相同思路, 关注目标分布         |
| **②统计**     | T检验/MW-U/ANOVA (组间)   | **Pearson/Spearman 相关 + BH-FDR**                   | 🔴 连续目标 → 相关分析 (非组间检验)  |
| **③预处理**    | 插补 + 标准化              | 中位数插补 + OneHot + StandardScaler                    | 🟢 完全相同, 多了分类编码         |
| **④特征选择**   | 6 层 / LASSO signature | **M0→M4 消融为主 + LASSO signature 辅助**                | 🟡 消融是回归重点, LASSO 辅助    |
| **⑤防泄漏**    | Pipeline + 训练集 fit    | + **分组/时间结构 → 4 种划分 + 泄漏审计**                       | 🔴 分组数据必须 GroupKFold    |
| **⑥交叉验证**   | K-Fold / 分层 / 重复      | **朴素 KFold (泄漏!) vs GroupKFold vs 时间 vs few-shot** | 🔴 分组泄漏是核心陷阱            |
| **⑦建模**     | 11 个 Classifier       | **同 11 算法族的 Regressor 版**                          | 🟢 同算法族, 换 predict/loss |
| **⑧不平衡**    | SMOTE / 阈值 / 加权       | **不适用 → winsorize + log1p + 残差思维**                 | 🔴 回归无类别不平衡             |
| **⑨校准/DCA** | 校准曲线 / Brier / DCA    | **不适用 → 回归校准 + 残差诊断 (QQ/同方差)**                     | 🔴 回归无阈值 → 残差诊断         |
| **⑩SHAP**   | 蜂群 (log-odds)         | **蜂群 (目标单位 mg/dL·min)**                            | 🟢 同 API, 值含义变          |

**一句话迁移法则**: 流程骨架 (EDA→统计→预处理→特征选择→CV→建模→不平衡→评估→SHAP)
完全通用; 回归的特殊性集中四处 — **①连续目标 → 相关分析; ②分组数据 → 4 种划分
(防泄漏核心); ③无类别不平衡 → 目标变换 + 残差; ④无阈值 → 残差诊断替代校准/DCA.**

## 四、★ 为什么强调划分策略? (本案例核心)

> 这是本案例与教程 01-21 **最大的不同**. 前面所有教程的数据是 i.i.d. (每个样本独立),
> 用朴素 KFold / train\_test\_split 即可. 本案例数据**非 i.i.d.** (同一受试者多餐次),
> 划分策略的选择直接决定了评估是否可信.

### 4.1 朴素 KFold 泄漏的直观演示 (★核心教学图)

脚本第 ⑥ 步用同一模型 (HGB) 对比两种划分:

| 策略                | MAE    | RMSE   | R²         | Pearson r |
| ----------------- | ------ | ------ | ---------- | --------- |
| ① 朴素 KFold (泄漏!)  | 1434.2 | 1921.0 | **0.507**  | 0.713     |
| ② GroupKFold (真实) | 1854.8 | 2444.2 | **0.203**  | 0.483     |
| → R² 虚高           | —      | —      | **+0.305** | —         |

**教学解读**: 朴素 KFold 的 R²=0.507 看起来不错, 但这是**假象** — 同一受试者的餐次
同时出现在训练集和测试集, 模型只需"认出是同一个人"就能预测 (因为同一个人的血糖
反应基线相似). GroupKFold 强制训练/测试受试者不交叉, R² 掉到 0.203 → **这才是模型
对未知用户的真实泛化能力**.

> 迁移法则: **只要数据有分组结构 (同人多样本/同医院多患者/同设备多测量), 必须用
> GroupKFold**, 否则 R²/Accuracy 会虚高 0.1-0.3, 导致错误结论.

### 4.2 4 种划分策略详解

| # | 策略                    | 实现                                   | 回答的问题      | 泄漏?  | 本案例结果                 |
| - | --------------------- | ------------------------------------ | ---------- | ---- | --------------------- |
| ① | 朴素 KFold              | `KFold(5, shuffle=True)`             | (反面教材)     | 🔴 是 | R²=0.507 (虚高)         |
| ② | GroupKFold            | `GroupKFold(5, groups=subject_id)`   | 能泛化到新用户吗?  | ✅ 否  | R²=0.203 (真实)         |
| ③ | 受试者内时间 80/20          | 每人按 meal\_time 排序, 前 80% 训练/后 20% 测试 | 能预测未来餐次吗?  | ✅ 否  | R²=0.430 (老用户)        |
| ④ | Few-shot k=0,1,3,5,10 | 训练=其他全部受试者+本人前 k 餐, 测试=本人剩余          | 新用户需登录多少餐? | ✅ 否  | k=0→10, MAE 1872→1549 |

**关键观察**:

- ② GroupKFold R²=0.203 (新用户, 最难) < ③ 时间 R²=0.430 (老用户, 较易) →
  说明**个人基线信息很重要**, 模型对已见过的用户表现更好
- ④ few-shot 曲线: k=0 (纯群体模型) MAE=1872 → k=10 (10 餐个性化) MAE=1549,
  下降 17% → **新用户登录 3-5 餐后预测开始有用, 10 餐后接近老用户水平**

**教学解读**:

1. **线性模型崩溃** (LinearRegression R²=−121, Ridge −59, Lasso −130) → M3 有 29 个

   完全新用户容易 → 个人基线信息贡献了一半的预测力.

| 特征集          | GroupKFold R² | GroupKFold MAE |   时间 R²   |
| ------------ | :-----------: | :------------: | :-------: |
| M0 (仅餐次营养)   |     0.066     |     2038.1     |   0.066   |
| M1 (+餐前CGM)  |     0.137     |     1920.0     |   0.057   |
| M2 (+可穿戴/时间) |     0.091     |     1989.8     |   0.127   |

| M4 (+肠道)     |     0.134     |     1947.0     |   0.431   |

**教学解读**:

- M0→M1: 加餐前 CGM 状态, R² 从 0.066 翻到 0.137 → **餐前血糖是强信号**
- M2 反而降 (0.091): 可穿戴特征噪声大/缺失多 (mets 缺失 22.5%), 在 GroupKFold 下引入过拟合
- M3 最强 (0.203): 临床基线 (A1c/Fasting GLU/Age) 是跨受试者泛化的关键 → **部署主模型用 M3**
- M4 降 (0.134): 肠道评分在 GroupKFold 下过拟合 (受试者级特征, 训练集没见过该受试者时无意义)

### 6.9 回归校准 + 残差诊断 (⑨)

| 指标           | 值      | 解读                   |
| ------------ | ------ | -------------------- |
| 校准 slope     | 0.735  | <1 → 预测被压缩 (回归到均值)   |
| 校准 intercept | +743.0 | >0 → 低值高估, 高值低估      |
| 预测SD/观测SD    | 0.657  | <1 → 预测方差小于真实方差 (压缩) |
| 残差均值         | +17.4  | ≈0 → 无系统偏差           |
| 残差 std       | 2444.1 | —                    |
| 残差偏度         | −0.72  | 左偏 (高值预测偏高时残差负偏)     |

**残差诊断 4-panel 图** (`22_residual_diagnostics.png`):

- (i) 残差 vs 预测: 漏斗形 → **异方差** (高值预测的误差更大)
- (ii) Q-Q 图: 两端偏离直线 → **重尾非正态** (置信区间不可靠)
- (iii) 残差直方图: 左偏, 偏度=−0.72
- (iv) Scale-Location: 趋势上升 → 确认异方差

> 教学解读: slope=0.735 说明模型"保守" — 对极端高血糖反应预测不足, 把所有预测
> 拉向均值. 这是小样本 + GroupKFold 的典型现象 (训练集没见过极端受试者).
> 残差非正态 → 报告置信区间时不能用正态假设, 应用 bootstrap.

### 6.10 SHAP 可解释性 (⑩, HGB)

SHAP 值单位 = **mg/dL·min** (目标单位, 非 log-odds), 即"该特征把这个餐次的预测
推高/拉低多少 mg/dL·min".

|  排名 | 特征                        | \|SHAP\| (mg/dL·min) | 方向解读                       |
| :-: | ------------------------- | :------------------: | -------------------------- |
|  1  | `eff_carbs`               |         714.7        | 碳水越高 → iauc 越高 (正相关, 临床已知) |
|  2  | `Fasting GLU - PDL (Lab)` |         488.5        | 空腹血糖越高 → iauc 越高           |
|  3  | `prev_meal_gap_min`       |         366.1        | 距上餐越久 → iauc 越高 (空腹效应)     |
|  4  | `A1c PDL (Lab)`           |         356.7        | 糖化越高 → iauc 越高             |
|  5  | `baseline_glucose`        |         330.1        | 餐前血糖越高 → **增量越小** (回归到均值)  |
|  6  | `pre_glucose_slope_60`    |         253.5        | 餐前血糖上升趋势 → iauc 越高         |
|  7  | `Cholesterol`             |         244.5        | —                          |
|  8  | `eff_protein`             |         218.2        | 蛋白质 → 降低 iauc (延缓吸收)       |
|  9  | `pre_glucose_sd_30`       |         158.8        | 餐前血糖波动大 → iauc 越低          |

## 七、回归适配要点 (与分类的关键差异)

| 维度           | 分类 (教程 01-12)               | 回归 (本案例 22)                            | 改法                               |
| ------------ | --------------------------- | -------------------------------------- | -------------------------------- |
| **统计检验**     | T 检验 / MW-U / ANOVA (组间)    | **Pearson / Spearman 相关 + FDR**        | `scipy.stats.pearsonr/spearmanr` |
| **评估指标**     | AUC / Recall / F1 / Brier   | **MAE / RMSE / R² / Pearson r / MAPE** | `mean_absolute_error, r2_score`  |
| **不平衡处理**    | SMOTE / 阈值 / class\_weight  | **不适用 → winsorize + log1p + 残差**       | 回归无类别, 处理目标分布                    |
| **校准**       | 校准曲线 / Brier / DCA          | **回归校准 (slope/intercept) + 残差诊断**      | `np.polyfit(y_pred, y_true, 1)`  |
| **SHAP 值含义** | log-odds (分类)               | **目标单位 (mg/dL·min)**                   | TreeExplainer 同 API, 输出单位变       |
| **模型类**      | Classifier (predict\_proba) | **Regressor (predict)**                | 同算法族换 `*Regressor`               |

### 回归评估指标速查

| 指标            | 公式                     | 优点              | 缺点                           |
| ------------- | ---------------------- | --------------- | ---------------------------- |
| **MAE**       | 平均\|pred−obs\|         | 抗异常值, 与目标同单位    | 不惩罚大误差                       |
| **RMSE**      | √(平均(pred−obs)²)       | 惩罚大误差           | 对异常值敏感                       |
| **R²**        | 1 − SS\_res/SS\_tot    | 0-1 可解释 (方差解释率) | 可为负 (模型比均值还差)                |
| **Pearson r** | 线性相关系数                 | 衡量排序能力          | 只捕捉线性关系                      |
| **MAPE**      | 平均\|pred−obs\|/\|obs\| | 无量纲, 可跨任务比      | obs≈0 时不稳定 (本案例用 safe\_mape) |

## 八、思考题 (迁移练习)

1. **若目标有负值 (如** **`peak_delta_2h`), 能否用 log1p 变换?** 为什么?
   (提示: log 要求非负. peak\_delta\_2h 有餐后血糖下降 (负值) → 只能 winsorize, 不能 log)
2. **few-shot k=0 与 GroupKFold 有什么区别?** 两者都是"训练集不含测试受试者".
   (提示: k=0 的测试集是本人**全部**餐次, GroupKFold 的测试集是**部分**受试者的全部餐次;
   k=0 衡量"纯群体模型对个人的预测", GroupKFold 衡量"对未见受试者的泛化")
3. **残差 vs 预测图呈漏斗形说明什么?** 该如何处理?
   (提示: 异方差 — 高值预测的误差更大. 处理: log1p 目标 / 加权回归 / 报告时分区给 CI)
4. **朴素 KFold R²=0.507 比 GroupKFold R²=0.203 高, 能说朴素 KFold 模型更好吗?**
   (提示: 不能! 0.507 是泄漏假象, 模型"认人"而非"预测血糖". 0.203 才是真实泛化)
5. **消融中 M4 (加肠道) 比 M3 (临床) R² 反而降, 说明肠道特征没用吗?**
   (提示: 不一定. GroupKFold 下肠道评分是受试者级特征, 训练集没见过该受试者时无法
   学习其肠道基线 → 过拟合. 在时间划分 (受试者内) 下 M4 R²=0.431 ≥ M3 0.430,
   说明肠道特征对**老用户**有价值)
6. **本案例 log1p 变换未改善 R², 但偏度从 1.26 降到 −1.85, 矛盾吗?**
   (提示: 不矛盾. log1p 修正了右偏但引入左偏, 且 HGB 树模型对目标单调变换不敏感
   (分裂点不变). log1p 主要帮助线性模型的残差正态性, 对树模型收益不大)

## 十、与教程 20/21 的对照总结

本案例与教程 20 (影像组学二分类) / 21 (基因组学多分类) 构成"迁移到新任务类型"的三方对比:

| 维度       | 影像组学 (教程 20)       | 基因组学 (教程 21)         | **CGM 回归 (本案例 22)**                  |
| -------- | ------------------ | -------------------- | ------------------------------------ |
| 数据模态     | CT 影像              | mRNA 表达矩阵            | CGM 餐级建模表                            |
| 任务类型     | **二分类**            | **多分类**              | **回归**                               |
| 目标       | Contrast (0/1)     | 癌症类型 (5 类)           | iauc\_2h (连续, mg/dL·min)             |
| 特征数      | 93                 | 20531                | 31 (M3)                              |
| 样本量      | 100                | 801                  | 1665 餐 (45 受试者)                      |
| **数据结构** | i.i.d.             | i.i.d.               | **分组 + 时间 (非 i.i.d.)**               |
| **划分策略** | 分层 KFold           | 分层 KFold             | **★4 种 (朴素/GroupKFold/时间/few-shot)** |
| 统计检验     | Mann-Whitney + FDR | ANOVA + η² + FDR     | **Pearson/Spearman + FDR**           |
| 特征选择     | LASSO (一步)         | 三步 (ANOVA→去冗余→LASSO) | **消融 M0-M4 + LASSO**                 |
| 不平衡      | 平衡, 跳过             | class\_weight        | **不适用 (winsorize + log1p)**          |
| 校准/评估    | 校准曲线 + DCA         | 混淆矩阵 + per-class ROC | **回归校准 + 残差诊断 (QQ/同方差)**             |
| SHAP 值   | log-odds           | log-odds (3D 需聚合)    | **目标单位 mg/dL·min**                   |
| 核心教学点    | LASSO signature    | p≫n 三步特征选择           | **★划分策略 + 回归评估**                     |
| 最佳模型测试性能 | AUC≈0.95           | Acc≈1.00             | GroupKFold R²=0.203 / 时间 R²=0.430    |

**共同结论**: 流程骨架 (EDA→统计→预处理→特征选择→CV→建模→不平衡→评估→SHAP)
跨任务类型完全通用; 掌握这套骨架, 迁移到任何新任务 (分类/回归/多分类) 都只需
调整"因任务特性而不同的那几步".

**回归迁移的 4 个关键差异** (本案例核心):

1. **连续目标 → 统计用相关分析 (非组间检验); 评估用 MAE/RMSE/R² (非 AUC)**
2. **分组数据 → 4 种划分策略 (朴素 KFold 泄漏! GroupKFold/时间/few-shot)**
3. **无类别不平衡 → 目标 winsorize + log1p + 残差思维 (非 SMOTE)**
4. **无阈值 → 回归校准 + 残差诊断 (非校准曲线/DCA)**

> 教学目标达成判据: 学生读完后能回答 — "换一个回归任务 (如预测住院天数/药代动力学
> AUC/生存时间), 哪些步骤照搬, 哪些步骤要改?" 答: 流程骨架照搬, 上述 4 处要改.
