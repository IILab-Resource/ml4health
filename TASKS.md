## Node 1：完整 EDA + 统计分析报告（模块 01-02 综合）

**前置模块**：01 EDA, 02 统计分析

**任务目标**：对一个新选择的特征子集（或其他公开数据集），独立完成一套完整的 EDA → 统计检验分析，输出一份结构化报告。

### 选项 A：本数据集深挖

从数据集中选择一个你尚未深入分析的方向：

- **类别特征专项**：对 Topography.Code、Code.of.Morphology 等类别特征做详细的频数分布、类别合并建议、与目标变量的交叉分析
- **时间维度分析**：利用 year、Date.of.Diagnostic 等信息做存活率时间趋势分析
- **人口学子群分析**：按 Age_Group × Gender × Raca.Color 分群，做分组 EDA + 组间统计检验

### 选项 B：新数据集探索

找一个公开的医学相关数据集（如 Kaggle、UCI、MIMIC 子集），完成：

1. **数据概况**
   - 样本量、特征数、缺失率概览
   - 目标变量分布（是否平衡）
2. **EDA 核心产出**
   - 缺失值矩阵 + 缺失模式分析
   - 数值特征的分布 + 离群值分析
   - 类别特征的频数分布
   - 至少 3 张有洞察的可视化图表（非默认模板）
3. **统计检验**
   - 对所有特征做与目标变量的统计检验
   - 标注效应量大小，区分"统计显著"与"实际显著"
   - 指出哪些特征值得进入后续建模

### 交付物

- 一份报告（含图表）
- 对应的 Python 脚本或 Notebook

### 整合要求

| 用到的模块 | 具体内容 |
|-----------|---------|
| 01 EDA | 缺失值分析、分布分析、离群值检测、可视化 |
| 02 统计分析 | 正态性判断、T/U 检验、卡方检验、效应量、多重比较校正 |

## Node 2：完整特征工程流水线（模块 03-05 综合）

**前置模块**：03 预处理, 04 特征工程, 05 特征选择

**任务目标**：基于 Node 1 的分析结果，自行设计并实现一条完整的数据预处理 → 特征工程 → 特征选择的流水线，并评估每个环节对模型性能的影响。

### 任务要求

1. **数据预处理**
   - 自行决定哪些列删除、哪些列插补、用什么方法插补
   - 至少对比 2 种插补方法的差异（如均值 vs KNN，或 KNN vs MICE）
   - 给出选择理由

2. **特征构造**
   - 至少构造 2 个有领域意义的新特征（如年龄分组、交互特征、多项式特征）
   - 解释为什么你觉得这些特征对预测目标变量有帮助

3. **特征选择**
   - 按教程的六层方法运行完整的特征筛选
   - 记录每层筛掉了多少特征
   - 最终特征集 ≤ 20 个（给出理由）

4. **对比实验**
   - 训练两个逻辑回归模型：一个用"全特征"，一个用"选择后特征"
   - 在测试集上对比 AUC、Recall、训练时间

### 交付物

- 完整的 Python 流水线脚本（可复现、可修改）
- 一个实验对比表格（全特征 vs 特征选择后）

### 整合要求

| 用到的模块 | 具体内容 |
|-----------|---------|
| 03 预处理 | 缺失值插补策略选择、标准化 |
| 04 特征工程 | 特征缩放比较、特征构造 |
| 05 特征选择 | 六层筛选流水线、VIF、Boruta |

## Node 3：防泄漏审计与流水线重构（模块 06-07 综合）

**前置模块**：06 降维, 07 数据泄漏

**任务目标**：审查 Node 2 的流水线是否存在数据泄漏，重构为无泄漏版本，并用降维方法对比效果。

### 步骤

1. **泄漏审计**
   - 逐行检查 Node 2 的代码，标注每一处可能泄漏的操作
   - 分类标注：泄漏类型（预处理泄漏 / 特征选择泄漏 / 插补泄漏）
   - 写出为什么这是泄漏、有什么后果

2. **重构流水线**
   - 用 `sklearn.pipeline.Pipeline` 重构整条流水线
   - 确保所有操作都在 CV 折内独立执行
   - 添加 ColumnTransformer 区分数值列和类别列的处理方式

3. **降维对比**
   - 在重构后的流水线上加入 PCA 降维（模块 06）
   - 对比"原始特征"、"特征选择后"、"PCA 降维"三种方案的 5-Fold CV 结果

### 交付物

- 泄漏审计清单（表格形式，标注位置 + 泄漏类型 + 修复方案）
- 重构后的无泄漏 Pipeline 代码
- 三种方案的 CV 对比表

### 整合要求

| 用到的模块 | 具体内容 |
|-----------|---------|
| 06 降维 | PCA 原理与实现、方差解释率 |

## Node 4：端到端建模挑战赛（模块 08-11 综合）

**前置模块**：08 交叉验证, 09 建模对比, 10 不平衡数据, 11 校准 DCA

**任务目标**：在 Node 3 的无泄漏流水线基础上，完成多模型对比、不平衡处理、校准评估的完整实验。

### 推荐方案：新数据集挑战

找一个不同的公开数据集（如 [MIMIC-IV](https://physionet.org/content/mimiciv/) 子集、[NIH Chest X-ray](https://www.kaggle.com/datasets/nih-chest-xrays/data)、[Diabetes dataset](https://www.kaggle.com/datasets/uciml/pima-indians-diabetes-database) 等），从头到尾跑完完整流程。

### 必须完成的内容

1. **多模型对比**（模块 09）
   - 至少对比 4 个不同类型的模型（含 LR + 1 个树模型 + 1 个 boosting 模型）
   - 使用 Stratified 5-Fold CV，报告 AUC、Recall、Brier Score、训练时间

2. **不平衡处理**（模块 10）
   - 如果新数据集是平衡的，人为制造不平衡（下采样多数类至 IR ≈ 10:1）
   - 对比"不处理" vs "SMOTE" vs "class_weight" 三种策略

3. **校准评估**（模块 11）
   - 对最优模型绘制校准曲线
   - 选择临床决策阈值（TP/FP 代价分析）
   - 运行 DCA（决策曲线分析），找出"净获益"最大的阈值范围

4. **最终模型选择**
   - 综合 AUC、Recall、校准度、DCA、训练成本 5 个维度，给出推荐模型及理由

### 交付物

- 完整的实验 Notebook（可复现）
- 模型对比雷达图（5 个维度归一化后绘制）
- 最终推荐表（含理由）

### 整合要求

| 用到的模块 | 具体内容 |
|-----------|---------|
| 08 交叉验证 | Stratified K-Fold、CV 评估框架 |
| 09 建模对比 | 7 模型对比、评估指标矩阵 |
| 10 不平衡数据 | SMOTE、代价分析、PR Curve |
| 11 校准 DCA | 校准曲线、决策曲线分析 |

## Node 5：综合 SHAP 可解释性报告（模块 12-17 综合）

**前置模块**：12 SHAP 基础, 12b 高级 SHAP, 13 SHAP 交互, 14 SHAP 依赖, 15 SHAP 聚类, 16 SHAP 决策路径, 17 SHAP 稳定性

**任务目标**：对 Node 4 选出的最优模型，做一套完整的 SHAP 可解释性分析，从全局到局部到稳定性评估。

### 必须完成

1. **全局解释**
   - SHAP Summary Plot（含特征重要性排序）
   - SHAP Bar Plot（平均绝对 SHAP 值）
   - 指出 Top-5 重要特征及其影响方向

2. **交互效应分析**（模块 13）
   - 至少识别 2 对显著交互的特征
   - SHAP Interaction Heatmap
   - 用依赖图解释交互的具体模式

3. **依赖关系与分布**（模块 14）
   - 对 Top-3 特征绘制 SHAP Dependence Plot
   - 叠加特征分布密度
   - 判断：线性还是非线性？是否存在阈值效应？

4. **个体解释**
   - 选 2 个典型样本（1 个正确分类 + 1 个错误分类）
   - SHAP Waterfall Plot
   - SHAP Decision Plot
   - 解释：模型为什么这么判断？

5. **稳定性评估**（模块 17）
   - Bootstrap SHAP 值（至少 50 次重采样）
   - 四象限可靠性图
   - 指出哪些特征的 SHAP 解释可靠、哪些不可靠

### 可选高难度

- **SHAP 聚类分析**（模块 15）：对 SHAP 值做聚类，识别亚组模式
- **SHAP 决策路径**（模块 16）：绘制多样本的决策路径图

### 交付物

- 一份 SHAP 解释报告（含图表），结构为：全局 → 交互 → 依赖 → 个体 → 稳定性
- 报告末尾给出"对模型可信度的总体判断"

### 整合要求

| 用到的模块 | 具体内容 |
|-----------|---------|
| 12 / 12b | SHAP 基础、Summary/Waterfall/Force |
| 13 | SHAP Interaction 交互效应 |
| 14 | SHAP Dependence + 分布叠加 |
| 15（可选） | SHAP 聚类分析 |
| 16（可选） | SHAP 决策路径 |

## Node 6：因果推断 + 预测全栈项目（模块 18 综合）

**前置模块**：18 HTE DML

**任务目标**：选择一个有明确"处理/干预变量"的数据集，同时完成**预测建模**和**因果推断**，对比两类方法得出的结论。

### 数据要求

- 必须有一个明确的**二值处理变量 T**（如是否接受某种治疗、是否发生转移、是否吸烟等）
- 包含足够多的协变量（混淆变量）
- 推荐数据集：本数据集的"Extension"（是否转移）作为 T

### 必须完成

1. **预测视角**
   - 用 Node 4 的方法训练一个模型预测 Y（结局）
   - 输出 T 的特征重要性（SHAP 值）
   - 回答：从预测角度看，T 对 Y 有多重要？

2. **因果视角**
   - 用 DML 估计 `E[Y(1) - Y(0)]`（平均处理效应 ATE）
   - 用 CATE 分析找出"谁受益最大、谁受益最小"
   - 回答：从因果角度看，T 对 Y 的影响有多大？在哪些人群中最强？

3. **对比分析**
   - 预测重要性排序 vs 因果效应排序：Top-5 特征是否一致？
   - 对 T 这个变量：预测视角的"重要性" vs 因果视角的"效应量"——哪个更大？为什么？
   - 讨论：什么情况下预测和因果的结论会一致/不一致？

### 交付物

- DML 分析 Notebook（含 CATE 异质性分析）
- 预测 vs 因果的对比报告
- 一个"预测-因果对照图"（横轴=SHAP importance of T，纵轴=CATE of T，每个点是一个亚组）

### 整合要求

| 用到的模块 | 具体内容 |
|-----------|---------|
| 09 建模对比 | 预测模型的训练与评估 |
| 12-17 SHAP | 预测模型的可解释性 |
| 18 DML | 双重机器学习、ATE/CATE 估计 |

## Node 7：ML Pipeline 可视化系统（全模块综合）

**前置模块**：全部 18 个模块

**任务目标**：将 Node 1-6 的全部成果整合为一个**交互式可视化系统**，让用户能够通过界面探索数据、配置流水线、对比模型、解释结果。

> 这是最终章——把 18 个模块的知识转化为一个可交付、可展示的应用系统。

### 技术选型（三选一）

| 框架 | 适用场景 | 上手难度 |
|------|---------|---------|
| **Streamlit** | 快速构建数据应用，Python 原生 | ★☆☆ |
| **Gradio** | ML 模型演示，适合快速出原型 | ★☆☆ |
| **Dash (Plotly)** | 复杂仪表盘，高度定制化 | ★★☆ |

推荐使用 **Streamlit**——学习成本最低、社区生态最成熟、适合数据科学场景。

### 系统功能要求

#### 页面一：数据总览（Data Overview）

- 数据集基本信息展示（样本量、特征数、缺失率）
- 交互式缺失值矩阵（可按列排序/过滤）
- 目标变量分布（饼图 + 柱状图，显示 IR）
- 数值特征分布选择器（下拉选择特征 → 直方图 + 箱线图叠加）
- **整合自**：模块 01 EDA

#### 页面二：预处理配置（Preprocessing Configurator）

- 特征选择面板（勾选哪些列保留，哪些列删除）
- 插补方法配置（按列选择：均值 / 中位数 / KNN / MICE）
- 标准化方法选择（StandardScaler / RobustScaler / MinMaxScaler）
- 运行按钮 → 显示预处理前后对比（分布变化、缺失率变化）
- **整合自**：模块 03-04 预处理与特征工程

#### 页面三：特征筛选仪表盘（Feature Selection Dashboard）

- 六层筛选结果逐层展开（相关性 → VIF → F 检验 → MI → LASSO → Boruta）
- 可交互的 Sankey 图（每一层多少特征通过/被筛掉）
- 最终特征集导出按钮
- **整合自**：模块 05 特征选择

#### 页面四：模型竞技场（Model Arena）

- 模型选择多选框（从 7 个模型中选择要对比的）
- 超参数滑块（如树深度、学习率、K 值）
- 运行对比 → 输出对比雷达图 + 指标表格 + 训练时间
- 最优模型自动高亮
- **整合自**：模块 08-09 交叉验证与建模对比

#### 页面五：不平衡实验（Imbalance Lab）

- 重采样方法选择（Under / Over / SMOTE / ADASYN）
- IR 调节滑块（控制重采样后的比例）
- 对比"不处理" vs selected method 的 ROC / PR 曲线叠加
- **整合自**：模块 10 不平衡数据

#### 页面六：决策曲线分析（DCA Tool）

- 模型选择（从页面四的结果中选择 1-2 个模型）
- 交互式 DCA 曲线（可拖动阈值线查看净获益值）
- "Treat All" / "Treat None" 参考线叠加
- **整合自**：模块 11 校准与 DCA

#### 页面七：SHAP 解释器（SHAP Explorer）

- 全局 SHAP Summary Plot（可排序/过滤特征）
- 交互式 SHAP Dependence Plot（选择 X 轴特征 + 颜色编码特征）
- 单样本解释器（输入样本 ID → Waterfall + Decision Plot）
- SHAP 稳定性指示器（显示哪些特征的 SHAP 值可靠）
- **整合自**：模块 12-17 SHAP 全系列

#### 页面八（可选）：因果推断面板（Causal Panel）

- ATE 展示 + CATE 分布
- 亚组分析表（按特征分组的 CATE）
- 预测 vs 因果对照图
- **整合自**：模块 18 DML

### 交付要求

| 等级 | 最低页面数 | 技术要求 |
|------|-----------|---------|
| **基础** | ≥3 个页面 | 数据总览 + 预处理配置 + 模型竞技场 |
| **进阶** | ≥5 个页面 | 数据总览 + 预处理 + 模型竞技场 + SHAP 解释器 + DCA |
| **挑战** | 7-8 个页面 | 全部功能 + 可部署（Docker / Streamlit Cloud） |




### 与其他节点的关系

> Node 7 不是重新做一遍 Node 1-6，而是**把 Node 1-6 的输出包装成交互式界面**。Node 1-6 产出的分析结果、模型、SHAP 值可以作为 Node 7 的"预计算数据源"直接加载。
