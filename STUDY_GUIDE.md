# 学习实践指南

## 三个文件夹的关系

```
lectures/      教学讲义    → 告诉你"学什么、为什么学"以及"还能做什么"
notes/         代码精读     → 告诉你"代码怎么写"（逐模块对应 src/ 行号详解）
jupyter/       交互运行     → 让你"动手跑代码"（Notebook 交互式环境）
```

**三者关系**：`lectures` 建立认知框架 → `notes` 提供代码理解指引 → `jupyter` / `src` 动手实践。

---

## 学习流程（每个模块的四步法）

讲义中的内容分为两部分：
- **主体内容**（前 3 节）：前置知识、实践目的、核心方法——**跑代码前先读**
- **扩展内容**（后几节）：进阶方向、盲区讨论、思考题——**跑完代码后再读**

### Step 1：读讲义主体 — 建立认知框架

打开 `lectures/XX_*_teaching_doc.md`，阅读**前半部分**：
- **一、前序知识要求** — 检查自己是否具备所需基础
- **二、本次实践内容与目的** — 明确要做什么、为什么做
- **三、核心教学内容（前半）** — 理解主流程和方法论

> 这一遍只求"知道要做什么"，不纠结代码细节。

### Step 2：看精读笔记 — 理解代码意图

配有 `notes/` 逐行精读笔记：
- 每段代码解释"为什么这么写"而不是"写了什么"
- 特别标注了**常见的坑**和**陷阱**

### Step 3：动手运行 — `jupyter/` 

用 `jupyter/` 交互式运行，边看边改


### Step 4：再读讲义扩展 — 深入与拓展

回到 `lectures/`，阅读**后半部分**：
- **进阶方向 / 扩展内容** — 在基础之上还能做什么？
- **盲区分析** — 当前方法有哪些局限？
- **思考题** — 检验是否真正理解

> 这一步最容易跳过，但也最有价值——只跑代码只能学会"操作"，读扩展才能学会"思考"。

---

## 四步法示意图

```
  Step 1                      Step 2 & 3                  Step 4
┌─────────────────┐     ┌─────────────────────┐     ┌─────────────────┐
│  讲义（前半）       │ ──→ │  notes + 跑代码       │ ──→ │  讲义（后半）       │
│  前置知识         │     │  理解代码为什么这么写  │     │  进阶方向         │
│  实践目标         │     │  动手运行观察结果      │     │  盲区讨论         │
│  核心方法         │     │                     │     │  思考题           │
└─────────────────┘     └─────────────────────┘     └─────────────────┘
        ↑                          ↑                          ↑
    "学什么、为什么"           "怎么写、怎么跑"            "还能做什么、有什么坑"
```
