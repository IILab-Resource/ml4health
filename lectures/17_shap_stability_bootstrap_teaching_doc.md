# 案例教程 17：SHAP 稳定性 Bootstrap 分析

## 一、前序知识要求

### 1.1 核心问题

> **单次模型训练得出的特征重要性，换一批数据还会一样吗？**

所有之前的教程（教程 12-16）中的 SHAP 分析都基于**一次**训练好的模型。但你有没有想过：

- 如果训练数据稍微变化，重要特征的排序会变吗？
- year 永远第一，还是某些时候会被其他特征超过？
- Code.Profession 的"重要性"是真实信号还是噪声？

**本教程使用 Bootstrap 方法回答这些问题。**

### 1.2 为什么需要稳定性分析？

```
医学论文中常见的"问题"

问题 1: "单次报告"
  作者: "我们训练了模型，发现 year 最重要（SHAP=0.27）"
  审稿人: "你只训练了一次模型。这个结论在小样本扰动下还成立吗？"

问题 2: "过度解读"
  作者: "Code.Profession 是第二重要特征（SHAP=0.08）"
  审稿人: "这是否只是抽样噪声？"

Bootstrap 稳定性分析回答这些问题:
  给每个特征的 "重要程度" 打上 "置信区间"
  给整个特征排序 打上 "稳定性评分"
```

### 1.3 什么是 Bootstrap？

```
原始数据 (n=5000):
  患者 1, 患者 2, 患者 3, ..., 患者 5000

Bootstrap 样本 1 (有放回抽样):
  患者 1, 患者 1, 患者 5, 患者 5000, 患者 3, ...
  → 某些患者出现多次, 某些没出现
  → 模拟 "略有差异的训练数据"

Bootstrap 样本 2:
  患者 2, 患者 4, 患者 5, 患者 4999, ...
  → 与样本 1 有不同的组成

... 重复 30 次 ...

结果: 30 个模型, 30 组 SHAP 重要性
  → 计算均值和置信区间
  → 评估哪些特征的排名是稳定的
```

### 1.4 前序知识

- **教程 12（基础 SHAP）**：了解 SHAP 重要性
- **Bootstrapping**：有放回重采样
- **变异系数 (CV)**：CV = std / mean，无量纲的波动度量
- **Spearman 等级相关**：衡量排序相似性

## 二、本次实践内容与目的

### 2.1 核心目的

> **通过 30 次 Bootstrap 迭代，量化每个特征重要性的置信区间、变异系数和排名稳定性，回答"特征重要性的结论是否可靠"。**

### 2.2 五个量化指标

| 指标 | 公式 | 含义 | 判断标准 |
|------|------|------|---------|
| **Mean SHAP** | E[importance] | 30 次 Bootstrap 的平均重要性 | 越高则特征越重要 |
| **Std SHAP** | σ[importance] | 重要性的波动程度 | 越低越稳定 |
| **CV (变异系数)** | σ / μ | 相对波动量 | <0.1: 高度稳定, 0.1-0.3: 中等, >0.3: 不稳定 |
| **CI Width** | P97.5 - P2.5 | 95% 置信区间的宽度 | 越窄越可信 |
| **Rank ρ** | Spearman(rank_i, rank_mean) | 单次排名与平均排名的一致性 | >0.9: 稳定, 0.7-0.9: 中等, <0.7: 不稳定 |

### 2.3 六张子图的读取指南

```
┌─────────────────────────┬─────────────────────────┬─────────────────────────┐
│ 子图 1: 重要性+CI       │ 子图 2: CV 稳定性排名    │ 子图 3: Bootstrap 分布   │
│                         │                         │                         │
│  条形图 + 误差棒:       │  水平条形图:             │  箱线图 + 散点:          │
│  越高=越重要            │  越靠左=越稳定           │  越窄=越稳定             │
│  误差棒越短=越稳定      │  绿色=稳定 红色=不稳定   │  无离群点=好             │
├─────────────────────────┼─────────────────────────┼─────────────────────────┤
│ 子图 4: 变化轨迹         │ 子图 5: 排名稳定性       │ 子图 6: CI 宽度排名       │
│                         │                         │                         │
│  折线图:                 │  直方图:                 │  条形图:                 │
│  每条线=一个特征         │  越集中越靠右=            │  越高=该特征的不确定性大  │
│  线波动越小=越稳定       │  排名越稳定              │  即"不接受的结论不可靠"   │
└─────────────────────────┴─────────────────────────┴─────────────────────────┘
```

## 四、教学讨论

### 4.1 为什么 Diagnostic.means 是最稳定的？

```
Diagnostic.means 的 CV=0.0439, 是倒数第二名的约一半水平。

原因:
  1. Diagnostic.means 是分类变量, 编码后分布集中
  2. Diagnostic.means 的重要性极高 (0.2106, 是第二名的 1.3 倍)
  3. Diagnostic.means 与目标的相关性稳定 → 无论 Bootstrap 怎么抽样,
     Diagnostic.means 与 VIVO 的关联始终存在

教学启示:
  "越重要的特征, 通常也越稳定"
  → 因为真实信号在数据扰动下"幸存"
  → 而噪声特征在不同样本中"忽高忽低"
```

### 4.2 Code.Profession 为什么不稳定？

```
Code.Profession 的 CV=0.2000, 是 Diagnostic.means 的 4.6 倍。

原因:
  1. 该特征值大量为 0, 分布稀疏
  2. Bootstrap 抽样中, 非零值的出现频率在不同样本中差异大
  3. 少部分非零值患者被抽中/漏掉, 大幅改变该特征的重要性

教学启示:
  "不稳定不意味着"不重要"——它可能意味着"不稳定重要"
  → 即该特征对某些患者很重要, 但不是对所有患者
  → 结合教程 15 (SHAP 聚类) 理解: Code.Profession
    可能在 Cluster 2 中很重要 (rel_imp=1.20x)
```

### 4.3 "高 CV 但低重要性"的解读陷阱

```
Code.Profession: Mean=0.017, CV=0.2000

两种可能的解读:

解读 A (错误):
  "Code.Profession 不稳定 → 不可信 → 放弃它"

解读 B (正确):
  "Code.Profession 的重要性很低 (0.017) 且不稳定 (0.200)。
   低重要性 + 高 CV 意味着它的 '重要性' 主要由抽样噪声决定。
   应谨慎将其作为 '重要特征' 报告。"

判断规则:
  - 高重要性 + 低 CV → 该特征确实重要, 结论可信
  - 低重要性 + 高 CV → 该特征可能不重要, 结论不可信
  - 高重要性 + 高 CV → 该特征重要但波动大, 需结合领域知识
  - 低重要性 + 低 CV → 该特征确实不重要, 结论可信
```

### 4.4 排名稳定性详解

```
平均 ρ=0.4114 → 全局排名**不稳定**

这意味着:
  特征排名在不同 Bootstrap 迭代中差异很大
  → 如果只训练一次模型, 得到的特征排名
    可能是一个"偶然"的结果

特别值得注意的是 ρ 的最小值 = -0.4286:
  这意味着在某次 Bootstrap 迭代中,
  特征排名的顺序与平均排名**完全相反**!

解决方案:
  报告平均排名 + 95% 置信区间
  (这正是本教程的核心输出)
  → 在论文中, 应报告"排名范围"而非"唯一点排名"
```

## 五、与之前教程的整合

### 5.1 交叉验证 vs Bootstrap 稳定性

| 方法 | 教程 | 解决的问题 | 区别 |
|------|------|-----------|------|
| K-Fold CV | 教程 8 | 模型泛化性能的稳定性 | 评估**模型性能** |
| Bootstrap SHAP | **本教程** | 特征重要性的稳定性 | 评估**解释的稳定性** |

```
CV 回答: "模型在不同数据划分下 AUC 波动多少?"
Bootstrap 回答: "特征重要性在不同数据扰动下波动多少?"

两者互补:
  CV 好的模型 → 稳定性不一定好
  Bootstrap 稳定的特征 → CV 不一定好
  好的论文需要同时报告两者
```

### 5.2 与教程 12 (SHAP 重要性) 的对比

```
教程 12:
  单次训练 → 一组 SHAP 值
  → "Diagnostic.means 的 SHAP = 0.24"

本教程:
  30 次训练 → 30 组 SHAP 值
  → "Diagnostic.means 的 SHAP = 0.21 ± 0.01, CV = 0.04"
  → "95% CI = [0.19, 0.23]"

改进: 从"点估计"到"区间估计"
```

### 5.3 在论文中的用法

```
在论文方法部分:

"为了评估特征重要性的稳定性, 我们进行了 30 次
Bootstrap 重采样。每次迭代中, 从原始数据集有放回
抽取 N 个样本, 重新训练模型并计算 SHAP 重要性。
最终报告了每个特征的均值、95% 置信区间和变异系数 (CV)。"

在论文结果部分:

"如表 X 所示, Diagnostic.means 是最稳定的特征 (CV=0.044),
而 Code.Profession 的波动最大 (CV=0.200)。
整体特征排名的 Spearman 相关系数为 0.41,
表明排名在不同 Bootstrap 迭代中存在较大波动,
在论文中报告特征排名时应谨慎。"

在论文讨论部分:

"SHAP 稳定性分析确认了 Diagnostic.means 作为首要预测因子的
可靠性 (95% CI [0.19, 0.23])。同时,
Code.Profession 的高变异系数 (0.200) 提示
其重要性受采样波动影响较大, 解读时应谨慎。"
```

## 六、扩展内容

### 6.1 应该用多少 Bootstrap 迭代？

| 迭代次数 | 适用场景 | 优缺点 |
|---------|---------|--------|
| 10-20 | 快速探索 | 大致了解稳定性, CI 不可靠 |
| **30-50** | **本教程** | **合理的精度-时间平衡** |
| 100-200 | 正式论文 | CI 准确, 但计算成本高 |
| 1000+ | 高精度研究 | CI 极其准确, 仅用于小数据集 |

### 6.2 Sub-sampling Bootstrap（另一种选择）

本教程使用常规 Bootstrap（有放回抽样 n 个样本）。另一种方法是 Sub-sampling：

```python
# Subsample: 从 n 中随机抽取 m < n 个样本 (不放回)
indices = np.random.choice(len(X), size=int(len(X) * 0.8), replace=False)
```

| 方法 | 样本量 | 理论依据 | 适用场景 |
|------|--------|---------|---------|
| Bootstrap | n 个 (有放回) | 对真实分布的渐近近似 | 标准推荐 |
| Sub-sampling | m < n (无放回) | m 越大, 估计越稳定 | 对 Bootstrap 有偏的修正 |

### 6.3 本方法的局限性

| 局限性 | 说明 | 缓解 |
|--------|------|------|
| 仅评估"数据扰动" | Bootstrap 只引入采样变异, 不涵盖所有不确定性来源 | 结合模型不确定性 (随机种子、超参数) |
| 计算成本高 | 30 次 × 训练 + SHAP = 30 倍成本 | 减少 n_estimators, n_shap |
| CV 对均值敏感 | 均值低 → CV 自动升高 | 同时看 Mean 和 CV |
| 排名稳定性的标度问题 | 6 个特征的排名空间小 → Spearman 可能不敏感 | 报告完整排名分布 |

### 6.4 半衰期分析（扩展思路）

```
一个有趣的问题:
  特征重要性在多少 Bootstrap 迭代后"收敛"?

方法:
  随着 Bootstrap 迭代增加, 累计均值的变化:
  sem_i = std(iterations[:i]) / sqrt(i)
  当 sem_i < 0.01|mean 时, 认为收敛

这回答了: "30 次够不够? 是否需要更多?"
```

## 七、总结与思考题

### 7.1 核心收获

1. **Diagnostic.means 和 Extension 是高度稳定的特征** (CV < 0.07)——它们在数据扰动下的一致性确认了"真正的信号"
2. **Code.Profession 是相对不稳定的特征** (CV=0.200)——它的"重要性"受抽样噪声影响大
3. **平均排名稳定性 ρ=0.4114**——整体排序**不稳定**，存在迭代中排名完全相反的情况 (ρ=-0.4286)
4. **Bootstrap 将点估计变为区间估计**——"Diagnostic.means 重要性 0.21 [95% CI 0.19-0.23]" 比 "0.24" 更科学
5. **交叉验证和 Bootstrap 稳定性互补**——一个评估性能稳定性, 一个评估解释稳定性
6. **排名不稳定的教学意义**——在论文中不应只报告"唯一点排名"，应报告排名范围或置信区间
