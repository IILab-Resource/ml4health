# 模块 3：分类特征分析 — 卡方检验

> **核心思想**：当特征是"类别"（如性别、种族、肿瘤类型）而不是"数值"时，T 检验和 Mann-Whitney U 检验都不再适用——你无法计算"性别的均值"。这时我们需要一种**专门针对分类变量×分类变量**的检验方法：**卡方检验（Chi-square test of independence）**。本模块带你一步步完成从列联表构建、稀疏类别过滤、卡方统计量计算，到效应量（Cramér's V）解读的全流程，并讨论"统计显著 ≠ 实际有意义"这一关键陷阱。

## 二、背景回顾

本教程使用的是**巴西癌症登记数据**（Brazilian Cancer Registry）。在模块 1 中，我们过滤掉 `Status.Vital`（生存状态）缺失的样本，剩余 **209,758 条**记录。

目标变量 `target` 是二分类：
- `1 = VIVO`（存活）
- `0 = MORTO`（死亡）

本模块分析的 **18 个分类特征**包括人口学变量（性别、种族、教育）、临床变量（诊断方式、肿瘤扩展、侧性、形态学描述、疾病编码、远处转移）和地理变量（国籍、出生州）等。

> 📌 **为什么不用 T 检验？** T 检验要求**因变量是连续数值**（如年龄），比较两组均值是否相等。但分类特征（如 Gender 取值"男/女/未知"）没有"均值"的概念，所以必须换一种基于**频数**的检验方法——卡方检验。

## 三、代码全貌

下面是 `02_statistical_analysis.py` 第 230–304 行的核心代码，本模块将逐行拆解：

```python
categorical_results = []

for col in categorical_features:
    data = df_model[[col, 'target']].dropna()
    if len(data) < 30:
        continue

    # 统计每个类别中的样本数
    value_counts = data[col].value_counts()
    # 过滤掉太稀疏的类别 (频数 < 5 可能导致卡方检验不稳定)
    valid_categories = value_counts[value_counts >= 5].index
    data_filtered = data[data[col].isin(valid_categories)]

    if len(data_filtered) < 30:
        continue

    # 构建列联表
    contingency = pd.crosstab(data_filtered[col], data_filtered['target'])

    # 如果列联表太小或太大，跳过
    if contingency.shape[0] < 2 or contingency.shape[1] < 2:
        continue

    # 卡方检验
    try:
        chi2_stat, p_value, dof, expected = chi2_contingency(contingency)
    except Exception:
        continue

    # Cramér's V 效应量
    n_total = contingency.values.sum()
    phi2 = chi2_stat / n_total
    k = min(contingency.shape) - 1
    cramer_v = np.sqrt(phi2 / k) if k > 0 else 0

    categorical_results.append({
        'Feature': col,
        'N': len(data_filtered),
        'Categories': contingency.shape[0],
        'Chi2': chi2_stat,
        'P_Value': p_value,
        'Cramér_V': cramer_v,
        'Significant_0.05': 'Yes' if p_value < 0.05 else 'No',
        'Significant_Bonf': 'Yes' if p_value < 0.05 / total_n_features else 'No'
    })
```

整体流程可以概括为 6 步：

| 步骤 | 操作 | 目的 |
|---|---|---|
| 1 | `df[[col, 'target']].dropna()` | 取出该特征和目标变量，丢弃缺失值 |
| 2 | `value_counts()` + 频数过滤 | 剔除频数 < 5 的稀疏类别 |
| 3 | `pd.crosstab()` | 构建列联表 |
| 4 | `chi2_contingency()` | 执行卡方检验 |
| 5 | 计算 Cramér's V | 得到不受样本量影响的效应量 |
| 6 | Bonferroni 校正 | 控制多重比较的家族错误率 |

## 四、第一步：取出特征和目标变量，丢弃缺失值

```python
data = df_model[[col, 'target']].dropna()
if len(data) < 30:
    continue
```

**逐层拆解：**

- `df_model[[col, 'target']]`：从大 DataFrame 中**只取两列**——当前要分析的特征 `col` 和目标变量 `target`。注意这里用的是**双括号 `[[ ]]`**，返回的是一个 DataFrame（两列）；如果用单括号 `[col]` 返回的是 Series（一列）。
- `.dropna()`：丢弃任何含有 `NaN` 的行。这样剩下的 `data` 中，`col` 和 `target` 两列都没有缺失值。
- `if len(data) < 30: continue`：如果有效样本不足 30 个，跳过该特征。30 是统计学中"小样本"的经验下限，样本太少时任何检验都不可靠。

> 💡 **小贴士**：`continue` 是 Python 循环控制语句，作用是"跳过本次循环剩余的代码，直接进入下一次循环"。在这里，如果某特征有效样本太少，我们就放弃分析它，转而处理下一个特征。

## 五、第二步：统计类别频数 + 过滤稀疏类别

这是本模块的**第一个重点**。真实数据中，分类特征经常存在大量"长尾"类别——绝大多数样本集中在少数几个类别，而剩下几十甚至几百个类别各自只有 1–4 个样本。这些稀疏类别会严重干扰卡方检验。

### 5.1 用 `value_counts()` 统计每个类别的频数

```python
value_counts = data[col].value_counts()
```

`Series.value_counts()` 是 pandas 中最常用的频数统计方法。它会对 Series 中的**每个唯一值**计数，并按频数**从高到低排序**，返回一个新的 Series：
- **索引（index）** = 类别值
- **值（values）** = 该类别出现的次数

**示例：** 假设 `data['Gender']` 长这样（前 10 行）：

| 行号 | Gender | target |
|---|---|---|
| 0 | Masculino | 1 |
| 1 | Feminino | 1 |
| 2 | Feminino | 0 |
| 3 | Masculino | 1 |
| 4 | Feminino | 1 |
| ... | ... | ... |

调用 `data['Gender'].value_counts()` 后得到：

```
Feminino     145000
Masculino     64000
Unknown          758
Name: Gender, dtype: int64
```

可以看到，索引是类别名（Feminino、Masculino、Unknown），值是对应的样本数。

### 5.2 稀疏类别过滤：保留频数 ≥ 5 的类别

```python
valid_categories = value_counts[value_counts >= 5].index
data_filtered = data[data[col].isin(valid_categories)]
```

**第一行：`value_counts[value_counts >= 5].index`**

这是一个**布尔索引**操作，逐层拆解：

1. `value_counts >= 5`：对 Series 的每个值做比较，返回一个**布尔 Series**（True/False）。例如：
   ```
   Feminino     True
   Masculino    True
   Unknown      True     # 假设 Unknown 有 758 个
   Name: Gender, dtype: bool
   ```
   但如果某个类别只有 3 个样本，它对应的值就是 `False`。

2. `value_counts[value_counts >= 5]`：用布尔 Series 作为"掩码"筛选原 Series，**只保留 True 对应的行**。频数 < 5 的类别被剔除。

3. `.index`：取出筛选后 Series 的索引，也就是**保留下来的类别名列表**。

**第二行：`data[data[col].isin(valid_categories)]`**

- `data[col].isin(valid_categories)`：`.isin()` 是 pandas 的"成员判断"方法。它对 `data[col]` 的每个元素判断"是否在 `valid_categories` 列表中"，返回一个布尔 Series（True/False）。
- `data[布尔 Series]`：用这个布尔掩码筛选 `data`，**只保留 `col` 取值在 `valid_categories` 中的行**。

最终 `data_filtered` 中，该特征的所有类别都至少有 5 个样本。

### 5.3 为什么频数 < 5 要过滤？

> 📌 **卡方检验的期望频数要求**：卡方检验有一个**核心假设**——列联表中**至少 80% 的单元格期望频数 ≥ 5，且所有单元格期望频数 ≥ 1**。如果某些类别样本极少，对应的期望频数会很小（可能 < 1），导致：
> 1. **χ² 统计量被严重高估**：公式中有 `(O−E)² / E` 这一项，当 E 很小时，分母小，整项被放大，χ² 值虚高。
> 2. **p 值不可靠**：χ² 分布是渐近分布，只有在期望频数足够大时才近似准确。E 太小时，真实分布偏离 χ² 分布，p 值失真。
> 3. **检验不稳定**：增减一两个样本就可能改变结论。
>
> 因此，**在构建列联表之前先过滤稀疏类别**，是卡方分析的标准预处理步骤。注意这里过滤的是**观测频数**（每个类别的样本数），而上面提到的"期望频数 ≥ 5"是检验假设——两者相关但不完全相同。过滤观测频数 ≥ 5 是为了**间接保证期望频数不会太小**。

> 💡 **小贴士**：如果你的数据中某个重要类别天然样本少（如罕见病），可以考虑：
> - **合并相近类别**（如把"印第安原住民"和"黄种人"合并为"其他"）。
> - **使用 Fisher 精确检验**（`scipy.stats.fisher_exact`），它不依赖期望频数假设，但只适用于 2×2 表；大表可用 `scipy.stats.chi2_contingency` 的 Monte Carlo 模拟选项。
> - **增加样本量**（如果可行）。

### 5.4 第二次样本量检查

```python
if len(data_filtered) < 30:
    continue
```

过滤稀疏类别后，可能整列样本数大幅下降（极端情况下，如果几乎所有类别都被过滤掉，`data_filtered` 可能只剩几行）。所以需要**再次检查**样本量是否足够。这是一种"防御性编程"——每一步操作后都验证结果是否合理。

## 六、第三步：构建列联表（Contingency Table）

这是本模块的**第二个重点**。列联表是卡方检验的输入，理解它的结构至关重要。

### 6.1 用 `pd.crosstab()` 构建列联表

```python
contingency = pd.crosstab(data_filtered[col], data_filtered['target'])
```

`pd.crosstab(index, columns)` 是 pandas 构建列联表（也叫交叉频数表，cross tabulation）的函数：
- **第一个参数 `index`**：作为**行**的变量（这里是特征 `col` 的取值）。
- **第二个参数 `columns`**：作为**列**的变量（这里是 `target` 的取值）。
- **返回值**：一个 DataFrame，**行索引是 `col` 的类别，列名是 `target` 的类别，单元格是该组合的频数**。

### 6.2 列联表长什么样？——以 Gender × target 为例

假设 `Gender` 有 3 个类别（Feminino、Masculino、Unknown），`target` 有 2 个类别（0=MORTO 死亡，1=VIVO 存活），那么列联表大致如下：

| Gender <br>（行：特征类别） | 0 (MORTO) <br>（列：死亡） | 1 (VIVO) <br>（列：存活） | **行合计** |
|---|---:|---:|---:|
| Feminino | 52,000 | 93,000 | **145,000** |
| Masculino | 28,000 | 36,000 | **64,000** |
| Unknown | 300 | 458 | **758** |
| **列合计** | **80,300** | **129,458** | **209,758** |

**如何读这张表？**

- **单元格**：表示同时满足"行类别"和"列类别"的样本数。例如左上角的 52,000 表示"性别为女性且死亡"的患者有 52,000 人。
- **行合计**（最右列）：每个性别类别的总人数。例如 Feminino 共 145,000 人。
- **列合计**（最下行）：每个目标类别的总人数。例如死亡共 80,300 人。
- **总合计**（右下角）：所有样本数 209,758。

> 💡 **小贴士**：`pd.crosstab` 还有一个常用参数 `margins=True`，会自动添加"行合计"和"列合计"行/列（即上表中的 All 行和 All 列）：
> ```python
> pd.crosstab(data['Gender'], data['target'], margins=True)
> ```
> 但在卡方检验中我们**不需要** margins，因为 `chi2_contingency` 只接受纯频数表。带合计的表反而会出错。

### 6.3 列联表的维度检查

```python
if contingency.shape[0] < 2 or contingency.shape[1] < 2:
    continue
```

- `contingency.shape` 是一个元组 `(行数, 列数)`。
- `contingency.shape[0]` = 行数 = 特征的类别数。
- `contingency.shape[1]` = 列数 = 目标变量的类别数。

**为什么至少需要 2×2？**

卡方检验的本质是"两个变量是否独立"。如果列联表只有 1 行或 1 列，意味着其中一个变量是**常数**（只有一个取值），那么"独立性"这个问题本身就没有意义——常数变量和任何变量都"独立"。

例如，如果某特征过滤后只剩 1 个类别，那么所有样本都在同一行，列联表只有 1 行，此时无法判断该特征与目标变量是否有关联，必须跳过。

> 📌 **最小表尺寸**：卡方检验要求列联表至少是 **2×2**（2 行 2 列）。1×k 或 k×1 的表没有自由度，无法检验独立性。

## 七、第四步：执行卡方检验

这是本模块的**第三个重点**。

### 7.1 调用 `chi2_contingency`

```python
try:
    chi2_stat, p_value, dof, expected = chi2_contingency(contingency)
except Exception:
    continue
```

`scipy.stats.chi2_contingency(observed)` 是 SciPy 提供的卡方独立性检验函数。它接收一个列联表（二维数组或 DataFrame），返回一个**包含 4 个值的元组**：

| 返回值 | 含义 | 类型 |
|---|---|---|
| `chi2_stat` | 卡方统计量 χ² | float |
| `p_value` | p 值 | float |
| `dof` | 自由度（degrees of freedom） | int |
| `expected` | 期望频数表（在 H₀ 成立时） | ndarray |

### 7.2 卡方检验的原假设与备择假设

- **原假设 H₀**：两个分类变量**独立**（无关联）。即特征 `col` 与 `target` 之间没有关系，知道 `col` 的值不能帮助你预测 `target`。
- **备择假设 H₁**：两个分类变量**不独立**（有关联）。

**p 值的含义：**
- p 值是在 H₀ 成立的前提下，观察到"当前 χ² 统计量或更极端值"的概率。
- p < 0.05 → 拒绝 H₀，认为两个变量**有关联**。
- p ≥ 0.05 → 无法拒绝 H₀，认为两个变量**可能独立**（注意：不能"证明"独立，只能说"没有足够证据说它们有关联"）。

### 7.3 卡方统计量的计算直觉

卡方统计量的公式是：

$$
\chi^2 = \sum_{i} \sum_{j} \frac{(O_{ij} - E_{ij})^2}{E_{ij}}
$$

其中：
- $O_{ij}$ = 第 $i$ 行第 $j$ 列的**观测频数**（observed），即列联表中的实际计数。
- $E_{ij}$ = 第 $i$ 行第 $j$ 列的**期望频数**（expected），即在 H₀（独立）成立时，这个单元格"理论上应该有多少样本"。
- 求和遍历列联表的所有单元格。

**期望频数 $E_{ij}$ 怎么算？**

$$
E_{ij} = \frac{\text{行合计}_i \times \text{列合计}_j}{\text{总样本数}}
$$

**直觉理解：**
- 如果两个变量独立，那么"第 $i$ 行的样本"和"第 $j$ 列的样本"应该"按比例混合"。期望频数就是这种"按比例混合"的结果。
- 例如 Gender × target 表中，Feminino 行合计 145,000，MORTO 列合计 80,300，总样本 209,758。如果 Gender 与 target 独立，那么"女性且死亡"的期望频数 = 145,000 × 80,300 / 209,758 ≈ 55,530。
- 如果实际观测值（如 52,000）和期望值（55,530）相差很大，说明"独立"假设站不住脚，χ² 增大，p 值减小。
- $(O - E)^2$ 保证差值非负且放大极端偏离；除以 $E$ 是为了**标准化**（否则样本多的单元格天然差值大，不公平）。

**自由度 dof：**

$$
\text{dof} = (r - 1) \times (c - 1)
$$

其中 $r$ 是行数，$c$ 是列数。自由度反映了"列联表中能自由取值的单元格数"。例如 3×2 表的 dof = (3−1)×(2−1) = 2。

### 7.4 `try-except` 异常处理

```python
try:
    chi2_stat, p_value, dof, expected = chi2_contingency(contingency)
except Exception:
    continue
```

**为什么要用 `try-except`？**

`chi2_contingency` 在某些极端情况下会抛出异常：
- 列联表中有 0 频数单元格（虽然前面过滤了，但仍可能出现）。
- 输入数据类型异常（如非数值）。
- 数值溢出（极大列联表 + 极大频数）。

`try-except` 的作用是"**尝试执行卡方检验，如果失败就跳过这个特征，不让整个程序崩溃**"。这是处理真实数据时的稳健做法——18 个特征中可能有 1–2 个因数据问题失败，但我们希望其他 16 个仍能正常分析。

- `try:` 块：尝试执行可能出错的代码。
- `except Exception:` 块：捕获**任何类型**的异常（`Exception` 是所有内置异常的基类）。
- `continue`：跳过本次循环，处理下一个特征。

> 💡 **小贴士**：在生产代码中，捕获 `Exception` 被认为过于宽泛（会吞掉所有错误，包括你没想到的 bug）。更严谨的做法是捕获具体异常，如 `except (ValueError, FloatingPointError):`。但教程中用 `Exception` 是为了简洁，且这里即使漏掉某个特征也不会造成严重后果。

## 八、第五步：Cramér's V 效应量

这是本模块的**第四个重点**，也是最容易让初学者困惑的部分。

### 8.1 为什么需要效应量？

> 📌 **Cramér's V 比 χ² 统计量更有意义**：χ² 统计量有两个致命缺陷：
> 1. **受样本量影响极大**：样本量越大，χ² 越大，p 值越小。在 20 万样本的数据集中，即使两个变量只有极其微弱的关联，χ² 也会很大，p 值也会接近 0——但这并不代表关联"强"。
> 2. **没有上限**：χ² 可以是 0 到 +∞，无法直接判断"关联有多强"。
>
> **Cramér's V** 通过对 χ² 做归一化，得到一个 **[0, 1] 之间的效应量**，不受样本量影响，可以直接解读关联强度。这就是为什么我们在看卡方检验结果时，**应该看 V 值而不是 χ² 值**。

### 8.2 代码

```python
n_total = contingency.values.sum()
phi2 = chi2_stat / n_total
k = min(contingency.shape) - 1
cramer_v = np.sqrt(phi2 / k) if k > 0 else 0
```

### 8.3 公式与逐行拆解

Cramér's V 的公式是：

$$
V = \sqrt{\frac{\chi^2}{n \times k}}
$$

其中：
- $\chi^2$ = 卡方统计量
- $n$ = 总样本量
- $k = \min(r, c) - 1$，$r$ 是行数，$c$ 是列数

**逐行拆解：**

**第一行：`n_total = contingency.values.sum()`**

- `contingency.values`：把 DataFrame 转成 NumPy 数组。
- `.sum()`：对所有单元格求和，得到总样本量 $n$（等于 `len(data_filtered)`，但这里从列联表直接算更稳妥）。

**第二行：`phi2 = chi2_stat / n_total`**

- 这是 **φ²（phi 系数的平方）**，即 $\phi^2 = \chi^2 / n$。
- 对于 2×2 表，$\phi = \sqrt{\chi^2 / n}$ 就是经典的 **phi 系数**，本身已经是 [0, 1] 的效应量。
- 但对于大于 2×2 的表，$\phi$ 可能超过 1（因为 χ² 没有上限），所以需要进一步归一化——这就是 Cramér's V 的作用。

**第三行：`k = min(contingency.shape) - 1`**

- `contingency.shape` = (行数, 列数)。
- `min(contingency.shape)` = min(行数, 列数)。
- `k = min(行数, 列数) - 1`。

**为什么取最小维度减 1？**

Cramér's V 的归一化因子 $k$ 取的是**行数和列数中较小者减 1**，原因是：
- χ² 的最大可能值与列联表的维度有关。理论上，$\chi^2_{\max} = n \times (\min(r, c) - 1)$。
- 所以 $\chi^2 / (n \times (\min(r, c) - 1))$ 的最大值恰好是 1，开方后 V ∈ [0, 1]。
- 取 min 而不是 max，是为了让 V 的上界始终是 1（如果取 max，V 可能永远达不到 1，失去归一化意义）。

**第四行：`cramer_v = np.sqrt(phi2 / k) if k > 0 else 0`**

- `np.sqrt(phi2 / k)` = $\sqrt{\phi^2 / k} = \sqrt{\chi^2 / (n \times k)}$ = Cramér's V。
- `if k > 0 else 0`：防御性编程。如果列联表是 1×c 或 r×1（虽然前面已过滤，但保险起见），k = 0，此时除以 0 会报错，所以直接返回 0。

### 8.4 Cramér's V 的解读标准

Cramér's V 的取值范围是 **[0, 1]**：
- **V = 0**：完全独立（无关联）。
- **V = 1**：完全关联（知道一个变量的值就能完全确定另一个变量）。

常用的解读标准（Cohen, 1988）：

| V 值范围 | 关联强度 | 解读 |
|---|---|---|
| V < 0.1 | **微小**（negligible） | 几乎没有关联，即使统计显著也意义不大 |
| 0.1 ≤ V < 0.3 | **小**（small） | 弱关联 |
| 0.3 ≤ V < 0.5 | **中**（medium） | 中等关联 |
| V ≥ 0.5 | **大**（large） | 强关联 |

> 💡 **小贴士**：这些阈值是经验值，不同领域可能略有调整。在医学研究中，V = 0.3 已经算是相当强的关联；而在社会科学中，V = 0.5 才算"大"。解读时要结合领域背景。

### 8.5 为什么 Cramér's V 不受样本量影响？

对比 χ² 和 V 的公式：

$$
\chi^2 = \sum \frac{(O - E)^2}{E}, \quad V = \sqrt{\frac{\chi^2}{n \times k}}
$$

- χ² 公式中，每个单元格贡献 $(O-E)^2/E$。当样本量翻倍时，$O$ 和 $E$ 都翻倍，$(O-E)^2$ 变成 4 倍，$E$ 变成 2 倍，所以每项贡献变成 2 倍——**χ² 随 n 线性增长**。
- V 公式中，分子 χ² 随 n 增长，但分母也有一个 $n$，所以 $n$ 被约掉了——**V 只取决于关联的"强度"（比例关系），与样本量无关**。

这就是为什么在 20 万样本的数据集上，即使 V = 0.017（极弱关联），p 值也可能 < 0.05（统计显著）。**统计显著只说明"有关联"，V 才告诉你"关联有多强"**。

## 九、第六步：Bonferroni 校正

```python
'Significant_Bonf': 'Yes' if p_value < 0.05 / total_n_features else 'No'
```

### 9.1 多重比较问题

本案例一共分析了 **22 个特征**（4 个数值 + 18 个分类）。对每个特征做一次检验，α = 0.05，意味着每次检验有 5% 的概率"误报"（假阳性，即 H₀ 为真却被拒绝）。

做 22 次独立检验，**至少出现一次假阳性的概率**约为：

$$
P(\text{至少一次假阳性}) = 1 - (1 - 0.05)^{22} \approx 1 - 0.32 = 0.68
$$

也就是说，即使所有特征都与目标变量无关，我们仍有 **68% 的概率**至少发现一个"显著"特征——这显然是不可接受的。

### 9.2 Bonferroni 校正的思路

Bonferroni 校正是最简单也最保守的多重比较校正方法：

$$
\alpha_{\text{校正}} = \frac{\alpha}{m}
$$

其中 $m$ 是检验次数（这里是 `total_n_features = 22`）。

- 原本 α = 0.05，校正后 α = 0.05 / 22 ≈ 0.00227。
- 只有 p < 0.00227 才算"Bonferroni 显著"。
- 这样 22 次检验的家族错误率被控制在 0.05 以内。

### 9.3 代码解读

```python
'Significant_Bonf': 'Yes' if p_value < 0.05 / total_n_features else 'No'
```

- `total_n_features` 在代码开头定义为 `len(numeric_features) + len(categorical_features) = 4 + 18 = 22`。
- `0.05 / total_n_features` = 0.05 / 22 ≈ 0.00227。
- 如果 `p_value < 0.00227`，标记为 `'Yes'`（Bonferroni 显著），否则 `'No'`。

> 💡 **小贴士**：Bonferroni 校正虽然简单，但**过于保守**——它会增加假阴性（漏报真实关联）的概率。当检验次数很多时（如几百个特征），可以考虑更强大的方法，如 **Benjamini-Hochberg（FDR）校正**。但在本案例 22 个特征的情况下，Bonferroni 是合理的选择。

## 十一、结果解读

### 11.1 强效应量特征（V ≥ 0.5）——疾病编码类

**Morphology.Description（V = 0.6959）**、**Illness.Code（V = 0.6385）**、**Description.of.Disease（V = 0.6225）**、**Description.of.Topography / Topography.Code（V = 0.6110）** 这一组特征效应量最大，都在 0.6 以上。

- 这些都是**疾病/肿瘤的医学编码**（ICD-O 形态学、拓扑学、疾病描述等），类别数极多（300–400+）。
- **V 值大并不意外**：不同类型的癌症（如乳腺癌 vs 胰腺癌）本身预后差异巨大，肿瘤类型几乎"决定"了生存结局。这符合医学常识——癌症的形态学类型是预后最重要的预测因子之一。
- **注意事项**：这些特征类别数极多（362、419、405、323），V 值可能被高估。Cramér's V 在类别数多时存在轻微偏倚，建议用 **Cramér's V 的偏差校正版本**（Bergsma's correction）做更严谨的估计。但在教学场景下，原始 V 已足够说明问题。

### 11.2 中强效应量特征（0.3 ≤ V < 0.5）

**Distant.metastasis（V = 0.4665）**：远处转移与生存强相关，完全符合医学常识——发生远处转移的癌症患者预后显著变差。这是临床上最重要的预后因子之一。

**Naturality.State（V = 0.3970）**：出生州与生存中度相关。这可能反映了巴西各州**医疗资源差异**（发达州 vs 欠发达州）、**环境因素**或**人口结构**的间接影响。它本身不是"病因"，而是一个**代理变量（proxy variable）**。

### 11.3 弱效应量特征（0.1 ≤ V < 0.3）

**Raca.Color（V = 0.1646）**、**Degree.of.Education（V = 0.1509）**、**State.Civil（V = 0.1090）** 这些社会经济特征效应量较小但非零。

- 种族、教育、婚姻状态对生存有弱影响，可能通过**就医时机、经济条件、社会支持**等中介因素起作用。
- 这种弱关联在公共卫生研究中仍有意义，但在个体预测建模中贡献有限。

### 11.4 微小效应量特征（V < 0.1）——统计显著但实际无意义

**Gender（V = 0.0918）**、**Laterality（V = 0.0587）**、**Nationality（V = 0.0497）**、**Type.of.Death（V = 0.0171）** 这一组是本模块最重要的"反面教材"。

> 📌 **统计显著 ≠ 实际有意义**：以 **Type.of.Death** 为例：
> - p = 2.92e-09（极小，远小于 Bonferroni 阈值 0.00227）→ **统计上极其显著**。
> - V = 0.0171（接近 0）→ **实际上几乎没有关联**。
>
> 为什么会这样？因为样本量 N = 120,445 极大，即使关联强度只有 0.017，卡方检验也能"检测"到它。但这不代表"死亡类型"能帮你预测生存状态——事实上，Type.of.Death 本身就只在死亡患者中有记录，它与 target 的"关联"更多是**数据结构**造成的，而非真正的预测关系。
>
> **结论**：在大样本场景下，**永远要看效应量，不能只看 p 值**。p 值告诉你"有没有"，效应量告诉你"有多强"。

### 11.5 一个有趣的"重复"现象

注意 **Description.of.Topography** 和 **Topography.Code** 的结果**完全相同**（N=209,638，类别数=323，χ²=78250.35，V=0.6110）。这不是巧合——这两个特征是**同一信息的不同表示**：一个是拓扑学部位的文字描述，另一个是对应的 ICD-O 代码。它们一一对应，所以分析结果完全一样。

> 💡 **小贴士**：在特征工程中，这种"重复特征"应该被识别并去除（保留一个即可），否则会在建模时引入**多重共线性**。本教程保留两者是为了教学展示，实际建模时应做去重。

## 十二、常见问题（FAQ）

### Q1：为什么有些特征的样本量 N 远小于 209,758？

A：因为 `data = df_model[[col, 'target']].dropna()` 会丢弃 `col` 列缺失的行。例如：
- `Extension` 只有 82,695 个有效样本 → 说明约 60% 的患者没有"肿瘤扩展"信息。
- `Distant.metastasis` 只有 9,986 → 说明只有约 5% 的患者有"远处转移"记录。
- `Child.Illness.Description` 只有 2,749 → 这是儿童期疾病描述，大多数成年患者没有这个字段。

样本量差异反映了**数据收集的完整性**，也间接提示了哪些特征可能存在**缺失机制**（如"未记录"本身可能就是一种信息）。

### Q2：为什么 `Distant.metastasis` 有 113 个类别？转移部位不是只有几个吗？

A：这个字段记录的是**转移部位的组合**（如"肝"、"肝+肺"、"骨+脑+肝"），所以类别数远多于单一部位。每个组合算一个类别，导致类别数膨胀。在实际建模中，可能需要把它拆分成多个二值特征（是否肝转移、是否骨转移等）。

### Q3：Cramér's V 能大于 1 吗？

A：理论上 V ∈ [0, 1]。但由于偏差，在类别数多、样本量小时，V 可能略超过 1（如 1.02）。如果出现这种情况，说明需要用偏差校正版本，或检查数据是否有问题。本案例中所有 V 都在 [0, 1] 内。

### Q4：卡方检验和 T 检验有什么区别？

A：
| | T 检验 | 卡方检验 |
|---|---|---|
| 适用场景 | 数值特征 × 二分类目标 | 分类特征 × 分类目标 |
| 比较内容 | 两组均值是否相等 | 两个分类变量是否独立 |
| 检验统计量 | t 统计量 | χ² 统计量 |
| 假设 | 正态性、方差齐性 | 期望频数 ≥ 5 |
| 效应量 | Cohen's d | Cramér's V |

### Q5：如果 p 值显示为 0.0，是真的等于 0 吗？

A：不是。p 值是浮点数，显示为 0.0 是因为**太小**，超出了打印精度（如 p < 1e-300）。在 SciPy 中，p 值的下限约为 1e-308（浮点数下溢）。对于本案例的 20 万样本，几乎所有"真实有关联"的特征 p 值都会下溢为 0。这也是为什么**不能只看 p 值**——它已经"饱和"了，无法区分"强关联"和"弱关联"。

### Q6：Bonferroni 校正后所有特征仍显著，是不是校正没用？

A：不是。Bonferroni 校正把阈值从 0.05 降到 0.00227，但本案例中所有 p 值都远小于 0.00227（多数为 0 或 1e-78 级别），所以全部通过。这说明关联是**真实且强烈**的，而非假阳性。Bonferroni 的价值体现在**样本量较小**或**关联较弱**的场景下，它能有效控制假阳性。在大样本场景下，效应量比校正更关键。

### Q7：为什么 `Description.of.Topography` 和 `Topography.Code` 结果完全一样？

A：因为这两个特征是**同一信息的两种编码**——一个是文字描述（如"C50.1 乳房外上象限"），另一个是对应的 ICD-O 代码（如"C50.1"）。它们一一对应，所以列联表、χ²、V 完全相同。在建模时应去掉一个，避免共线性。

## 十三、小贴士汇总

> 💡 **小贴士 1**：`value_counts()` 默认按频数降序排序。如果你想按类别名排序，用 `value_counts().sort_index()`；想看占比，用 `value_counts(normalize=True)`。

> 💡 **小贴士 2**：`.isin()` 方法比写多个 `==` 条件更简洁。例如 `df[df['city'].isin(['北京', '上海', '广州'])]` 比 `(df['city'] == '北京') | (df['city'] == '上海') | ...` 简洁得多。

> 💡 **小贴士 3**：`pd.crosstab` 还支持 `normalize` 参数计算百分比：`pd.crosstab(a, b, normalize='index')` 按行归一化（每行加起来=1），`normalize='columns'` 按列归一化，`normalize='all'` 全局归一化。

> 💡 **小贴士 4**：卡方检验只能检测"有关联"，**不能说明因果方向**。即使 V 很大，也不能说"A 导致 B"——可能存在混杂变量。因果推断需要额外设计（如随机对照试验、工具变量、DML 等）。

> 💡 **小贴士 5**：对于 2×2 列联表，Cramér's V 等价于 phi 系数，也等价于 Pearson 相关系数的绝对值。但对于大表，只有 Cramér's V 是归一化的。

> 💡 **小贴士 6**：**永远不要**只看 p 值下结论。大样本下 p 值几乎总是显著，必须结合效应量（Cramér's V、Cohen's d 等）判断实际意义。这是统计学和机器学习的核心思维差异之一。

> 💡 **小贴士 7**：当类别数极多（如 Morphology.Description 有 362 类）时，Cramér's V 会有轻微正向偏倚。严谨分析可用 **Cramér's V 偏差校正**（Bergsma, 2013）或交叉验证效应量的稳定性。

> 💡 **小贴士 8**：稀疏类别过滤的阈值（频数 ≥ 5）是经验值，不是铁律。对于 2×2 表，更严格的要求是"所有期望频数 ≥ 10"；对于大表，"80% 期望频数 ≥ 5"即可。根据数据特点灵活调整。

## 十四、本模块小结

本模块我们完成了：

1. **理解卡方检验的核心思想**：通过比较"观测频数"与"期望频数"的差异，判断两个分类变量是否独立。
2. **掌握列联表的构建**：用 `pd.crosstab()` 构建行=特征类别、列=目标类别、单元格=频数的交叉表。
3. **学会稀疏类别过滤**：用 `value_counts()` 统计频数，用 `value_counts >= 5` 筛选，用 `.isin()` 过滤——保证卡方检验的期望频数假设成立。
4. **调用 `chi2_contingency`**：获取 χ² 统计量、p 值、自由度、期望频数表，并理解每个返回值的含义。
5. **计算 Cramér's V 效应量**：用 $V = \sqrt{\chi^2 / (n \times k)}$ 归一化，得到 [0, 1] 之间、不受样本量影响的关联强度指标。
6. **应用 Bonferroni 校正**：将 α 除以检验次数，控制多重比较的家族错误率。
7. **识别"统计显著 ≠ 实际有意义"陷阱**：Type.of.Death 虽然 p = 2.92e-09 极其显著，但 V = 0.0171 几乎无关联——大样本下 p 值会"饱和"，必须看效应量。
8. **解读 18 个特征的实际结果**：疾病编码类特征（Morphology.Description、Illness.Code 等）V > 0.6，与生存强相关；远处转移（V = 0.47）符合医学常识；人口学特征（性别、国籍）V < 0.1，实际意义有限。

**关键结论**：
- 在 20 万大样本下，**所有 18 个分类特征都统计显著**（p < Bonferroni 阈值）。
- 但**效应量差异巨大**（V 从 0.017 到 0.70），只有 V ≥ 0.3 的特征才对个体预测有实质贡献。
- 这印证了本案例教程的核心主题：**统计学显著性是建模的必要条件，但不是充分条件**。后续模块会进一步用机器学习方法（特征选择、SHAP 等）验证哪些特征真正有预测价值。
