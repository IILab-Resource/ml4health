## 一、前序知识要求

### 1.1 数学与几何基础

| 概念                 | 说明                 | 与本教程的关系           |
| ------------------ | ------------------ | ----------------- |
| **欧几里得距离**         | 高维空间中两点之间的直线距离     | KNN 在高维失效的原因之一    |
| **维度（Dimension）**  | 数据的特征数量，每个特征构成一个维度 | 特征越多，数据空间的几何结构越复杂 |
| **投影（Projection）** | 将高维数据映射到低维空间       | PCA 的核心操作         |
| **方差（Variance）**   | 数据在某个方向上的离散程度      | PCA 寻找"最大方差方向"    |
| **线性变换**           | 用矩阵乘法改变数据的坐标表示     | PCA 就是一组线性变换      |
| **簇（Cluster）**     | 数据中自然聚集的群体         | 聚类分析的目标           |
| **距离度量**           | 欧氏距离、曼哈顿距离、余弦相似度等  | K-Means 和层次聚类的核心  |

### 1.2 机器学习基础

| 概念                      | 说明                             |
| ----------------------- | ------------------------------ |
| **K 近邻（KNN）**           | 基于距离度量的非参数分类器（模块 1 用于演示维度灾难）   |
| **过拟合**                 | 模型学到噪声信号，在训练集好测试集差（维度越高越容易过拟合） |
| **特征选择 vs 特征变换**        | 选择是保留子集，变换是创建新组合（本教程的核心对比）     |
| **可解释性**                | 能否理解"为什么"模型做出某个预测（PCA 降低可解释性）  |
| **Logistic Regression** | 线性分类器，对特征共线性敏感（模块 8/9 的评估模型）   |

## 三、各模块详解

### 模块 1：高维数据的挑战（维度灾难）

#### 核心问题

> 假设你有 1 个特征（Age），值域 0-120，数据在这个一维线上均匀分布。你需要多少个点才能保证相邻点的距离 ≤ 1？

- 1 维：121 个点
- 2 维：121² = 14,641 个点（网格填充）
- 22 维（本数据集）：121²² ≈ 10⁴⁶ 个点

这就是**维度灾难**——随着维度增加，数据空间呈指数膨胀，无论多少数据都显得"稀疏"。

#### 本实验的关键发现

| 维度     | KNN (k=5) AUC | 解读            |
| ------ | ------------- | ------------- |
| 2      | 0.7591        | 信息太少，无法区分     |
| 6      | 0.8836        | 逐步增加维度，信息量增大  |
| **10** | **0.9025**    | **接近峰值**      |
| 14     | 0.9068        | 趋于平稳          |
| 22     | 0.9072        | 平台期——额外维度收益递减 |

**教学重点**：KNN 的性能随维度增加先快速上升后趋于平稳。前 10 个维度提供了大部分有效信息，超过 14 个维度后，**额外维度带来的收益递减**。这说明特征确实存在冗余——降维可以在几乎不损失性能的前提下减少计算量。

#### 讨论：为什么 KNN 在高维空间效果会下降？

```
低维空间:                         高维空间:
采样密集, 近邻有意义             采样稀疏, "近邻"其实很远
    ●                               ●   ●
    ● ●                                  ●
    ●   ●                                  ●
    近邻 ≈ 相似样本                  近邻 ≈ 不相似的样本

KNN 在高维的基本假设被打破: "近" ≠ "相似"
因为距离度量在高维空间趋于均匀化
```

### 模块 2：PCA（主成分分析）

#### PCA 不是特征选择，而是特征变换

这是教学中最重要的概念区分：

| 对比       | 特征选择（Filter/LASSO/Boruta） | PCA                                  |
| -------- | ------------------------- | ------------------------------------ |
| **输出**   | 原始特征的子集                   | 原始特征的线性组合                            |
| **可解释性** | "Age 是重要特征"               | "PC1 是 Age、year、Code.Profession 的组合" |
| **信息保留** | 丢弃不重要的特征                  | 保留方差最大的方向                            |
| **数学本质** | 子集选择                      | 基变换 + 数据投影                           |
| **维度可调** | 离散（保留哪些）                  | 连续（选择几个 PC）                          |

#### PCA 的核心数学思想

```
第 1 步: 数据中心化 (减去均值)
第 2 步: 计算协方差矩阵 S = XᵀX / (n-1)
第 3 步: 对 S 做特征值分解 S = VΛVᵀ
第 4 步: 特征向量 = 主成分方向 (PC1, PC2, ...)
         特征值 = 该方向上的方差
第 5 步: 数据投影到前 k 个主成分上
```

#### 本实验的方差解释率

| 主成分      | 解释方差       | 累积         | 解读                      |
| -------- | ---------- | ---------- | ----------------------- |
| **PC1**  | **27.53%** | 27.53%     | 最大方差方向，约 1/4 的信息        |
| **PC2**  | **15.40%** | 42.93%     | 与 PC1 正交的第二大方向          |
| PC5      | 6.94%      | 68.52%     | 前 5 个 PC 解释了约 2/3 的信息   |
| **PC10** | **3.48%**  | **91.19%** | **10 个 PC 解释了 91% 的方差** |
| PC15     | 0.37%      | 99.14%     | 15 个 PC 到 99%           |

**关键发现**：

- PC1 就已经解释了 27.53% 的信息——说明数据在某个方向上高度结构化
- 10 个 PC 解释 91%，但最后 5 个 PC 只贡献了 8%——**长尾效应**
- 这验证了特征选择阶段的发现：特征确实存在冗余

#### 课堂讨论：为什么有时候降维后性能反而提高？

| 原因        | 说明                          |
| --------- | --------------------------- |
| **去除噪声**  | 方差最小的 PC 通常是噪声，丢弃它们相当于去噪    |
| **降低过拟合** | 减少参数数量，降低模型对训练集的过度拟合        |
| **打破共线性** | PCA 生成的 PC 之间正交，完美解决多重共线性问题 |

### 模块 3：PCA 可视化

#### 教学效果最好的部分

PCA 将 22 维数据投影到 PC1-PC2 平面上，让我们用肉眼观察数据的**宏观结构**。

**本实验观察到的结果**：

- PC1-PC2 投影中，MORTO（蓝色）和 VIVO（红色）有部分分离趋势
- 但分离并不彻底——两类在中心区域高度重叠
- 说明**存活和死亡患者在宏观特征上不是天然可分的**

#### 讨论：如果两类明显分开意味着什么？

- 如果能被 PC1-PC2 清晰分开 → 数据在全局线性结构下就可分 → 线性模型（如逻辑回归）效果会很好
- 如果完全重叠 → 线性模型无法区分 → 需要非线性模型或更好的特征

本数据集的情况介于两者之间：PC1-PC2 可见分离趋势但不彻底，逻辑回归 AUC ≈ 0.90 也印证了这个判断。

### 模块 4：t-SNE

#### 为什么 PCA 有时看不到结构？

PCA 是**线性**的——它只能捕捉数据中的**全局线性结构**。如果数据中的聚类结构是**非线性**的（如环形、螺旋形），PCA 投影会完全丢失这些信息。

t-SNE 解决了这个问题：

```
PCA 假设:         t-SNE 假设:
数据在全局      数据在局部
是线性结构      是邻域结构
    █               █
    █ █            ███
    █   █          █ █ █
```

#### t-SNE 的核心思想

| 步骤        | 高维空间                           | 低维空间          |
| --------- | ------------------------------ | ------------- |
| **计算相似度** | 高斯分布 → P(ij) = 点 j 是点 i 的邻居的概率 | t 分布 → Q(ij)  |
| **优化目标**  | 最小化 P 和 Q 之间的 KL 散度            | 同左            |
| **结果**    | —                              | 低维嵌入保留了局部邻域结构 |

#### 本实验观察

与 PCA 相比，t-SNE 的二维投影中两组（VIVO vs MORTO）的分离 **更加明显**。但这不意味着"t-SNE 更好"，而是因为 t-SNE 将局部邻域结构聚集在一起，创造了更强的视觉分离效果。

**不同 perplexity 的影响**：

| Perplexity | 效果      | 解读              |
| ---------- | ------- | --------------- |
| 5          | 很多小簇    | 过于关注局部，分解了全局结构  |
| **30**     | **适中**  | **默认值，平衡局部和全局** |
| 80         | 更大、更少的簇 | 更关注全局，丢失细节      |

### 模块 5：t-SNE 常见误区

#### 这是最重要的科研训练内容

很多学生看到 t-SNE 图中两类明显分开，会兴奋地说："t-SNE 分得很开，模型一定很好！"

**这是完全错误的。**

#### 为什么 t-SNE 不能用于性能评价？

| 原因               | 说明                             |
| ---------------- | ------------------------------ |
| **t-SNE 随机性**    | 不同随机种子给出不同结果（实验显示了 3 种完全不同的布局） |
| **t-SNE 不保留距离**  | 只保留邻居关系，两点之间的距离没有定量意义          |
| **t-SNE 不保留密度**  | 簇的大小不代表原始空间的密度                 |
| **t-SNE 是可视化工具** | 它优化的是"视觉分离"而不是"分类准确率"          |

#### t-SNE 的正确使用场景

```
✅ 可以做:                        ❌ 不能做:
探索数据结构                    评估模型性能
发现潜在子群                    比较不同模型的优劣
辅助提出假设                    作为特征选择的依据
生成可视化报告                  报告准确率/精确率
```

### 模块 6：UMAP（高级）

#### 为什么 UMAP 在医学 AI 论文中越来越常见？

| 对比        | t-SNE           | UMAP                |
| --------- | --------------- | ------------------- |
| **速度**    | O(n²)，慢         | O(n log n)，快 2-10 倍 |
| **全局结构**  | 只保留局部           | 保留局部 + 全局           |
| **数学框架**  | 概率方法            | 拓扑学（单纯复形）           |
| **可复现性**  | 对随机种子敏感         | 更稳定                 |
| **参数敏感性** | perplexity 影响巨大 | n\_neighbors 更稳健    |

**本实验结果**：UMAP 的二维投影在保持局部邻域的同时，也展现了更多的全局结构，这让它比 t-SNE 更适合探索数据的真实拓扑。

### 模块 7：聚类分析

#### 核心问题：不看标签，能否自动发现良性/恶性群体？

这是一个特别适合科研训练的问题——**"无监督学习能否发现有监督学习的分类"**。

**本实验结果**：

| 方法            | 指标         | 值          | 解读          |
| ------------- | ---------- | ---------- | ----------- |
| K-Means (k=2) | ARI        | **0.0081** | 与真实标签几乎无关系  |
| K-Means (k=3) | ARI        | 0.0135     | 略有改善但仍极低    |
| K-Means (k=5) | ARI        | 0.0179     | 无实质性改善      |
| K-Means (k=2) | Silhouette | 0.1588     | 结构存在, 但非球形簇 |

**教学结论**：ARI ≈ 0 意味着**聚类结果与真实标签几乎没有重合**。这告诉我们一个重要的事实：

> 存活和死亡患者的特征差异不足以在无监督环境下被自动发现。有监督学习（使用标签）是关键。

这与 PCA 和 t-SNE 的可视化结果完全一致——虽然有分离趋势，但重叠度很高。

#### 讨论：数据天然可分吗？

```
PCA 投影中可见分离趋势 ≠ 天然可分
t-SNE 中视觉分离明显   ≠ 天然可分
K-Means ARI ≈ 0        = 天然不可分 (在欧几里得距离下)
```

### 模块 8：PCA 用于机器学习

#### PCA 不只是画图工具

很多学生认为 PCA 只是"画图的"，实际上 PCA 可以用于建模——将高维数据压缩后训练模型。

**本实验结果**：

| 方法                     | AUC        | Recall | Brier  | 解读                     |
| ---------------------- | ---------- | ------ | ------ | ---------------------- |
| **Original (22 dims)** | **0.9024** | 0.8843 | 0.1250 | 全特征基线                  |
| PCA-5                  | 0.8222     | 0.7727 | 0.1734 | 压缩到 5 维→信息损失较大         |
| PCA-10                 | 0.8935     | 0.8763 | 0.1312 | 10 维 ≈ 91% 方差→AUC 接近恢复 |
| PCA-15                 | 0.9018     | 0.8845 | 0.1255 | 15 维→接近原始性能            |
| PCA-20                 | 0.9023     | 0.8842 | 0.1250 | 完全恢复→最后几维含少量有用信息       |

**关键发现**：

- PCA-10（仅 10 个主成分）已经达到全特征 AUC 的 99.0%（0.8935 / 0.9024）
- PCA-15 几乎完全恢复到全特征性能
- PCA-5 损失较大（AUC 0.8222）——5 个 PC 只解释了 68.5% 的方差

#### 什么时候 PCA 有帮助？

```
✅ PCA 有帮助的场景:
  - 特征高度共线 → PCA 生成正交主成分
  - 需要压缩存储 → 用少量 PC 替代多特征
  - 后续模型对共线性敏感 → 线性回归、逻辑回归
  - 去噪 → 丢弃方差最小的 PC（通常含噪声）

❌ PCA 没帮助的场景:
  - 需要可解释性 → 医生需要知道"为什么"
  - 特征是二值分类变量 → PCA 假设连续变量线性关系
  - 树模型 → 不关心特征的共线性
```

### 模块 9：PCA vs 特征选择比较

#### 课程中的经典讨论题

| 对比维度        | PCA           | 特征选择（Filter/LASSO/Boruta） |
| ----------- | ------------- | ------------------------- |
| **输出类型**    | 新特征（线性组合）     | 原始特征子集                    |
| **可解释性**    | 低（PC 没有物理含义）  | 高（保留原始特征）                 |
| **信息保留**    | 渐进式（选择 PC 数量） | 离散式（选择/不选择）               |
| **对模型影响**   | 压缩去噪，提升稳定     | 简化模型，提升泛化                 |
| **医疗领域的偏好** | 探索性分析         | **临床可解释建模**               |

**本实验的定量对比**：

| 方法             | AUC    | 可解释性 | 适用场景         |
| -------------- | ------ | ---- | ------------ |
| Full (22 dims) | 0.9024 | ★★★  | 全部信息可用       |
| Filter Top 10  | 0.8902 | ★★★  | 需简化但保留特征含义   |
| RF Top 10      | 0.8919 | ★★★  | 树模型重要性排序     |
| PCA-10         | 0.8935 | ★    | 不在意特征含义，只求低维 |

**教学结论**：

- 如果目标是**可解释性**（临床决策）：Filter 或 RF 优于 PCA（虽然 AUC 略低，但保留了原始特征含义）
- 如果目标是**压缩信息**（降维可视化、去噪）：PCA 优于特征选择（AUC 最高且去相关）
- 四者的 AUC 差距不到 0.012——可解释性和信息保留不是非此即彼

## 四、扩展

### 4.1 更多降维方法

| 方法                       | 分类    | 特点       | 适用场景      |
| ------------------------ | ----- | -------- | --------- |
| **PCA**                  | 线性    | 最大化方差    | 基线降维、去相关  |
| **Factor Analysis (FA)** | 线性    | 建模隐变量    | 心理测量、社会经济 |
| **LDA**                  | 线性有监督 | 最大化类间分离  | 分类的可视化    |
| **Isomap**               | 非线性   | 保持测地距离   | 流形数据      |
| **LLE**                  | 非线性   | 保持局部线性嵌入 | 非线性流形     |
| **Autoencoder**          | 神经网络  | 非线性压缩    | 复杂高维数据    |

### 4.2 t-SNE vs UMAP 的选择指南

```
┌──────────────────────────────────────┐
│        选择 t-SNE 还是 UMAP?         │
└──────────────────────────────────────┘
                 │
     ┌───────────┴───────────┐
     │                       │
t-SNE 更适合:            UMAP 更适合:
- 数据量 < 10,000        - 数据量 > 10,000
- 需要经典基准对比        - 需要速度
- 教学演示                - 需要全局+局部结构
- 论文中的标准可视化      - 当前论文趋势
```

### 4.3 聚类分析的扩展

| 方法                      | 特点            | 缺点             |
| ----------------------- | ------------- | -------------- |
| **K-Means**             | 简单快速，假设球形簇    | 需预设 K 值，对初始值敏感 |
| **层次聚类**                | 树状图直观展示层次结构   | 计算量大，O(n²)     |
| **DBSCAN**              | 基于密度，无需预设 K 值 | 对距离阈值敏感        |
| **Gaussian Mixture**    | 软聚类，概率分配      | 需预设组件数         |
| **Spectral Clustering** | 基于图论，可处理非球簇   | 计算量大           |

## 五、总结与思考题

### 5.1 核心收获

1. **高维数据难以直接观察** → 降维是将数据结构展示给人的桥梁
2. **PCA 是线性特征变换** → 保留最大方差方向，但丢失非线性结构和可解释性
3. **t-SNE 擅长局部结构** → 可视化效果好，但不能用于模型评价
4. **UMAP 保留全局+局部结构** → 医学 AI 论文中使用频率越来越高
5. **聚类分析揭示数据天然结构** → ARI ≈ 0 说明本数据集中存活/死亡不是天然分离的群体
6. **PCA 可用于建模** → 10 个主成分 ≈ 22 个全特征的 99% 性能
7. **PCA vs 特征选择** → 不矛盾，取决于目标：信息压缩用 PCA，可解释性用特征选择

### 5.2 思考题

> 以下问题覆盖从理解到批判性思维的全层次：

1. **（基础）** 维度灾难实验中，KNN 的 AUC 随维度增加先快速上升（2→10 维）后趋于平稳（14→22 维）。为什么 2 维时 AUC 远低于 10 维？为什么超过 14 维后 AUC 几乎不再提升？
2. **（进阶）** PCA 的 PC1 解释了 27.53% 的方差。你觉得 PC1 主要捕捉了 Age 还是 year 的信息？如何用 PCA 的载荷向量（loadings）验证你的猜想？
3. **（进阶）** PCA 降维到 10 维时 AUC = 0.8935，而 Filter 选择 Top 10 特征时 AUC = 0.8902。为什么相同维度下 PCA 反而略优于 Filter？（提示：PCA 生成的正交主成分打破了共线性，而 Filter 保留的原始特征可能存在冗余）
4. **（拓展）** K-Means 聚类的 ARI ≈ 0，说明无监督聚类无法发现存活和死亡群体。如果换用 Spectral Clustering，结果会不同吗？为什么？
5. **（拓展）** t-SNE 的三个不同随机种子给出了三种完全不同的布局。这对你在论文中使用 t-SNE 有什么启示？该怎么办才能让结果更可靠？
6. **（开放）** 有人说"PCA 只是画图的，对机器学习没有实际帮助"，本教程模块 8 的结果能否反驳这个观点？请用数据说话。
7. **（开放）** 本教程中 PCA-5 → PCA-10 的 AUC 提升了 0.0713，但 PCA-5 只有 5 个维度（比 10 个维度少一半）。如果计算效率是你的首要考虑，你会选择 5 个还是 10 个主成分？你的决策依据是什么？
8. **（实践）** 在 t-SNE 中使用不同的 perplexity（5, 30, 50, 100），观察可视化结果的变化。你认哪个 perplexity 最适合本数据集？为什么？
